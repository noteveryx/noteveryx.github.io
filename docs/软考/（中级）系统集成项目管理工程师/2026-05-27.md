## 今日核心概念

### 1. 关键链法（CCM）
定义：在关键路径法基础上，考虑资源约束和不确定性，通过插入缓冲来管理进度的一种网络分析技术。

关键要点：
- 关键链 = 考虑资源约束后最长的活动路径，不同于关键路径
- 三种缓冲：项目缓冲（PB，放在关键链末尾）、接驳缓冲（FB，放在非关键链汇入关键链处）、资源缓冲（RB，仅预警不占时间，用于提醒资源到位）
- 核心思想：消除活动中的安全时间（削减50%工期估算），将安全时间集中放在缓冲区统一管理
- 与关键路径法最大区别：关键路径法不考虑资源约束，关键链法必须考虑
- 适用范围：资源受限、工期紧张的多项目并行环境

### 2. 质量保证（QA）与质量控制（QC）
定义：质量保证关注过程合规和标准执行（"做正确的事"），质量控制关注交付物缺陷检测和纠正（"事做得正确"）。

关键要点：
- QA面向过程，QC面向结果；QA是预防为主，QC是检测为主
- QA审计质量要求和质量控制测量结果；QC输入包含质量管理计划和质量测量指标
- QA常用工具：质量审计、过程分析（七大管理工具：亲和图、关联图、树形图、优先矩阵、矩阵图、过程决策程序图PDPC、网络图）
- QC常用工具：检查、七种基本质量工具（因果图、控制图、流程图、直方图、帕累托图、散点图、核查表）
- 区别：QA关注"如何做才不出错"，QC关注"做出来的东西有没有错"

### 3. 风险应对策略
定义：针对已识别风险制定具体行动方案，分为消极风险/威胁和积极风险/机会两大类。

关键要点：
- 消极风险（威胁）五项策略：规避（消除风险根源）、转移（如买保险、转包给第三方）、减轻（降低发生概率或影响）、接受（主动接受=预留应急储备，被动接受=不采取行动）、上报（超出项目经理权限）
- 积极风险（机会）五项策略：开拓（确保机会发生）、分享（与第三方合作）、提高（增加概率或影响）、接受、上报
- 次生风险（规划风险应对后产生的新风险）和残余风险（采取应对措施后仍遗留的风险）属于风险登记册需要持续跟踪的内容
- 应急储备用于"已知-未知"风险（如预留10%工期），管理储备用于"未知-未知"风险（如预留5%总预算）
- 弹回计划：当主应对策略无效时的备用方案

### 4. 冲突解决方法
定义：项目团队管理中的冲突处理方式，按"双赢→双输"从优到劣排序。

关键要点：
- 合作/解决问题（最佳）：面对面协商，找到令各方满意的方案，双赢
- 妥协/调解（次佳）：各方各退一步，没有人完全满意，双输
- 强制/命令（优劣参半）：牺牲一方满足另一方，一方赢一方输，紧急情况下可用
- 缓和/包容（优劣参半）：强调共同点、淡化分歧，维持和谐但问题未解决
- 撤退/回避（最差）：暂时搁置冲突，拖延处理，双输，问题可能恶化
- 考试高频考点：合作是唯一双赢策略；强制只在紧急时使用；撤退永远不推荐

### 5. 项目管理过程组与知识领域矩阵
定义：项目管理知识体系中，五大过程组（启动、规划、执行、监控、收尾）与十大知识领域交叉形成的项目管理框架。

关键要点：
- 规划过程组涉及的知识领域最多（除整体管理收尾外，几乎所有知识领域都有规划活动）
- 整体管理横跨全部五个过程组（制定项目章程→制定项目管理计划→指导与管理项目工作→监控项目工作→结束项目或阶段）
- 收尾过程组仅涉及整体管理和采购管理两个知识领域
- 监控过程组：整体管理有"监控项目工作"和"实施整体变更控制"，范围管理有"确认范围"和"控制范围"，进度/成本管理有对应的控制过程
- 各知识领域在过程组中并非均匀分布，建议按过程组维度纵向梳理、按知识领域横向记忆，形成网状理解

## 案例分析题

**题目**：

某市财政局委托A公司建设"财政大数据分析平台"，项目预算500万元，工期8个月。项目经理小王采用关键路径法编制了进度计划，识别出关键路径工期为7.5个月，计划工期裕量不足。

项目进入第5个月末，绩效数据如下：
PV=320万元，EV=280万元，AC=340万元。

同时发生了以下情况：
（1）测试组发现财务管理模块存在3个严重缺陷，测试经理认为是开发组代码规范性不足导致，开发组长认为是需求文档描述模糊导致，双方争执不下，测试工作停滞。
（2）项目发起人提出新增"绩效评价"模块，涉及预算约80万元，小王预估会增加工期2个月。发起人要求小王"想办法搞进去，不能超工期"。
（3）小王发现核心数据接口供应商近一个月未按SLA响应问题，存在延期风险，但供应商是唯一可选方。

**问题1**（8分）：计算第5月末的CV、SV、CPI、SPI，并分析当前项目状态。

**问题2**（6分）：针对情况（1），请说明冲突的本质是什么，并推荐一种最合适的冲突解决方法及理由。

**问题3**（8分）：针对情况（2），小王应如何按照整体变更控制流程处理？请写出关键步骤。

**问题4**（3分）：针对情况（3），请为小王推荐风险应对策略并说明理由。

**参考答案**：

**问题1**：
CV = EV - AC = 280 - 340 = -60万元，费用超支。
SV = EV - PV = 280 - 320 = -40万元，进度滞后。
CPI = EV / AC = 280 / 340 ≈ 0.82，每花费1元仅产生0.82元价值，费用绩效较差。
SPI = EV / PV = 280 / 320 = 0.875，完成计划进度的87.5%，进度滞后。
综合判断：项目处于费用超支、进度滞后的不良状态，需立即采取纠正措施。

**问题2**：
冲突本质是问题归属不清引发的人际冲突，测试组和开发组互相推诿责任。
推荐采用合作/解决问题方法。理由：双方需要面对面共同分析缺陷根因（是代码问题还是需求问题），找到双方都能接受的解决方案，而非简单判定谁对谁错。合作是唯一的双赢策略，适合此类非紧急但需根本解决的冲突。

**问题3**：
（1）发起人提交正式变更请求（书面形式）；
（2）小王作为项目经理首先评估变更对范围、进度、成本、质量、风险等方面的影响；
（3）将变更请求和影响评估提交CCB（变更控制委员会）审批；
（4）CCB审批通过后，更新项目管理计划、范围基准、进度基准、成本基准等项目文件；
（5）通知相关干系人变更结果，按更新后的计划执行；
（6）跟踪变更执行情况，确保变更可控。
关键提示：新增模块涉及范围、成本、进度三重基准变更，必须走正式变更控制流程，不能口头答应"想办法"。

**问题4**：
推荐采用减轻策略。
理由：供应商是唯一可选方，无法规避或转移；可以通过以下措施降低风险概率和影响——与供应商高层召开紧急会议明确SLA违约责任、派驻技术人员到供应商现场协助、制定备用技术方案（如自研接口替代方案），同时申请应急储备用于加班赶工。在接受唯一供应商约束的前提下，积极降低风险发生的可能性和影响程度。

**踩分要点**：
- 问题1：公式正确且计算准确各1分，状态描述2分（超支1分+滞后1分）
- 问题2：冲突本质2分（责任推诿/问题归属不清），推荐方法2分（合作），理由2分（双赢+根本解决）
- 问题3：提交变更请求2分，影响评估2分，CCB审批2分，更新计划+通知干系人+跟踪各1分；"不要口头答应"是隐形加分点
- 问题4：策略1分（减轻），理由2分（唯一供应商不可换+降低概率和影响的具体措施）

## 记忆口诀

**关键链缓冲诀**：
关键链末项目缓冲，非关键接接驳缓冲。资源缓冲不占时，安全时间集中管。

**质量保证与控制区别诀**：
保证看过程管规矩，控制测结果抓缺陷。七管工具帮保证，七基工具助控制。

**风险应对策略诀**：
消极五招记心间：规避转移和减轻，接受上报排两边。积极五招对应看：开拓分享和提高，接受上报同样来。

**冲突解决优先诀**：
合作双赢最优先，妥协各退居中间。强制紧急一把用，缓和和谐问题悬。撤退搁置最次选，考场上直接排除先。

**变更控制六步诀**：
提变更→评影响→交CCB→审决策→更新计划→通知跟。基准变更必过CCB，口头答应是大忌。
