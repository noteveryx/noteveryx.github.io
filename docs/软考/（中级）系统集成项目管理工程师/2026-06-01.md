# （中级）系统集成项目管理工程师 · 2026-06-01

## 今日核心概念

### 1. 项目进度控制（进度压缩技术）

**定义：** 在不改变项目范围的前提下，通过赶工或快速跟进等手段缩短项目工期的技术方法。

**关键要点：**
- **赶工（Crashing）**：向关键路径上的活动投入更多资源（加人、加班）来压缩工期；代价是直接成本上升，必须进行成本/收益权衡；只对可以用增加资源来缩短工期的活动有效
- **快速跟进（Fast Tracking）**：将原本顺序执行的活动并行执行；优点是不增加资源，缺点是返工风险增加，且只适用于可并行化的活动
- **选择原则**：优先对关键路径上"单位时间成本增加最小"的活动赶工；两种方法可以组合使用
- **区别记忆**：赶工→花钱买时间；快速跟进→冒险抢时间
- **注意**：压缩工期不等于减少范围，减少范围属于范围变更

---

### 2. 采购计划与采购文件类型

**定义：** 采购管理规划过程中确定项目自制还是外购，并编制采购相关文件的过程。

**关键要点：**
- **自制与外购分析**：需考虑成本（直接+间接）、能力、保密性、时间等因素
- **采购文件类型区分**：
  - **信息邀请书（RFI）**：仅收集信息，不作为正式采购
  - **报价邀请书（RFQ）**：适用于标准化、价格为主要选择标准的采购
  - **建议邀请书（RFP）**：适用于复杂采购，需要供应商提出解决方案
  - **投标邀请书（IFB）**：建设工程常用，价格最低且满足要求即中标
- **供方选择标准**：总拥有成本（TCO）、技术能力、管理能力、财务稳定性、参考案例
- **集中采购 vs 分散采购**：集中采购统一管理、折扣力度大；分散采购响应速度快、适应项目个性化需求

---

### 3. 项目绩效报告与沟通管理

**定义：** 按照沟通管理计划，收集并向干系人分发项目绩效信息的过程，包括状态报告、进展报告和预测报告。

**关键要点：**
- **报告类型**：
  - 状态报告：描述项目在某一特定时间点所处的状态（当前状态）
  - 进展报告：描述项目团队在某一时间段内完成了哪些工作（过去做了什么）
  - 预测报告：基于已有绩效信息预测项目未来状态（未来会怎样）
- **里程碑报告**：对特定里程碑节点的完成情况进行汇报
- **沟通方式选择**：
  - 正式文字：合同、变更请求、重大决策
  - 非正式文字：邮件、日常通知
  - 正式口头：演示、会议汇报
  - 非正式口头：日常交流、澄清说明
- **沟通渠道数计算**：n个人，沟通渠道数 = n×(n-1)/2
- **踩分点**：报告类型的定义区分，以及沟通方式的选择依据是考点

---

### 4. 问题日志与经验教训登记册

**定义：** 问题日志是记录和追踪项目执行过程中发生的问题、其状态和解决情况的文件；经验教训登记册记录整个项目中积累的知识，以便未来项目参考。

**关键要点：**
- **问题日志内容**：问题类型、提出人、提出日期、描述、优先级、分配责任人、目标解决日期、状态、解决方案
- **问题 vs 风险**：风险是尚未发生的不确定事件；问题是已经发生的需要处理的实际事件
- **经验教训登记册**：
  - 贯穿项目全生命周期，在收尾时最终更新
  - 内容：情境描述、已识别的问题或成功点、采取的行动、行动的结果、建议
  - 是组织过程资产的重要组成部分
- **更新时机**：项目全过程持续更新，不仅限于项目收尾阶段
- **与收尾的关系**：项目收尾时将经验教训登记册转入组织过程资产库

---

### 5. 虚拟团队与资源管理

**定义：** 虚拟团队是指具有共同目标、在完成角色任务时很少或没有面对面时间的人员组合，通常依托信息技术协作。

**关键要点：**
- **虚拟团队优势**：可利用地理上分散的专业技能、减少差旅成本、纳入有行动障碍的成员、跨时区工作可延长工作时间
- **虚拟团队挑战**：沟通障碍（时区/语言/文化差异）、孤立感、信任建立困难、协作工具依赖
- **管理措施**：明确的沟通计划（含时区差）、定期视频会议、建立团队规范和期望、使用协作平台
- **团队建设方法**：集中办公（战情室）、虚拟团队工具（Zoom/Teams/企业微信）
- **激励理论考点**：
  - 马斯洛需求层次：生理→安全→社交→尊重→自我实现
  - 赫茨伯格双因素：保健因素（薪资/环境）消除不满；激励因素（成就/认可）激发积极性
  - 麦格雷戈X/Y理论：X理论→员工懒惰需严格管控；Y理论→员工有责任心可授权
  - 期望理论（弗鲁姆）：激励力=效价×期望值×工具性

## 案例分析题

**题目：**

某智慧园区管理系统项目，合同总价2800万元，计划工期18个月。项目经理小张在第10个月末进行绩效分析，获得以下数据：

- 计划价值（PV）= 1680万元
- 挣值（EV）= 1400万元
- 实际成本（AC）= 1540万元

此时项目发现一批采购的网络设备存在质量问题，供应商坚持认为设备符合合同规格，而项目方认为设备无法满足实际安装要求。与此同时，由于进度落后，项目经理决定对3项关键路径活动同时采取"赶工"和"快速跟进"措施，并提交了一份紧急变更申请。

**问题一（10分）：** 请计算该项目第10个月末的SPI、CPI、EAC（以ETC=剩余工作按当前CPI效率完成为假设计算），并分析项目当前状态。

**问题二（10分）：** 针对与供应商的争议，项目经理应按照什么流程处理？常见的合同争议解决方式有哪些？各自适用场景是什么？

**问题三（10分）：** 项目经理提交的紧急变更申请应通过什么流程处理？赶工和快速跟进分别有什么优缺点？请说明在关键路径上如何选择进度压缩策略。

---

**参考答案：**

**问题一：**

- SPI = EV / PV = 1400 / 1680 ≈ **0.83**（进度落后，每花1元计划价值只完成0.83元工作）
- CPI = EV / AC = 1400 / 1540 ≈ **0.91**（成本超支，每花1元只产生0.91元价值）
- EAC = BAC / CPI = 2800 / 0.91 ≈ **3077万元**（预计完工成本）
- ETC = EAC - AC = 3077 - 1540 ≈ **1537万元**（预计剩余成本）
- 项目状态分析：**进度落后且成本超支**。SPI < 1说明进度偏慢，CPI < 1说明成本效率低。项目需要采取纠正措施，并及时向干系人汇报偏差情况。

**问题二：**

合同争议处理流程：
1. 双方协商谈判（最优先，成本最低）
2. 提交采购方（业主）仲裁或项目协调小组调解
3. 调解（第三方中立调解员）
4. 仲裁（申请仲裁机构裁决，一裁终局，不可再诉）
5. 诉讼（司法途径，周期长、成本高，最后手段）

各方式适用场景：
- **协商**：争议金额小、双方关系良好、希望继续合作
- **调解**：双方有调解意愿，争议较复杂但不想上仲裁/诉讼
- **仲裁**：合同中有仲裁条款，争议较大，需要专业裁决但不想诉讼
- **诉讼**：仲裁无法解决或一方不履行仲裁裁决，需要司法强制力

**问题三：**

变更处理流程（整体变更控制）：
1. 提出变更申请（项目经理提交）
2. 变更影响评估（对范围、进度、成本、质量、风险的影响）
3. CCB（变更控制委员会）审批
4. 批准后更新项目管理计划（基准）
5. 实施变更，监督执行
6. 关闭变更，记录结果

赶工 vs 快速跟进对比：

| 维度 | 赶工 | 快速跟进 |
|------|------|----------|
| 核心手段 | 增加资源 | 活动并行 |
| 直接成本 | 增加 | 不增加 |
| 主要风险 | 沟通协调难度增加 | 返工风险增加 |
| 适用条件 | 活动可通过增加资源缩短工期 | 活动之间存在可并行化空间 |

选择策略：
- 优先分析关键路径上各活动的"赶工成本斜率"（单位时间压缩成本），选择成本斜率最小的活动优先赶工
- 若活动间存在逻辑上可并行的前后依赖关系（非强制依赖），可考虑快速跟进，但需充分评估返工风险
- 两种方法可结合使用

---

**踩分要点：**

1. **挣值公式必须准确**：SPI=EV/PV，CPI=EV/AC，EAC=BAC/CPI；常见错误是把PV和EV搞反，或用AC/EV计算CPI
2. **合同争议解决方式的顺序**：协商→调解→仲裁→诉讼，答题时必须体现优先级，不能只列方式不说顺序
3. **变更流程必须包含CCB审批**：很多考生漏写"CCB审批"步骤，直接丢分
4. **赶工只对关键路径有效**：非关键路径活动赶工不能缩短总工期，这是常见陷阱
5. **快速跟进的前提**：活动之间是非强制依赖（非硬逻辑），才能并行；强制依赖（如必须先完成A才能开始B的物理约束）不能快速跟进

## 记忆口诀

**进度压缩两法诀**
赶工花钱买时间，资源投入关键链；
快跟冒险抢进度，并行活动返工险；
成本斜率选赶工，逻辑并行选快跟；
两法结合效果好，CCB审批不能忘。

**挣值三指标诀**
SPI看进度，EV除PV；
CPI看成本，EV除AC；
大于一是好，小于一要找；
EAC预完工，BAC除CPI。

**合同争议处理诀**
协商优先不伤和，调解中立来斡旋；
仲裁专业一裁终，诉讼耗时最后关；
条款写明仲裁好，避免诉讼两败还。

**采购文件四字诀**
RFI问信息，RFQ问价钱；
RFP问方案，IFB建设专；
复杂采购用RFP，标准品质用RFQ。

**沟通渠道公式诀**
n个人来沟通，n乘(n-1)除以2；
人越多道越多，项目经理管得苦；
沟通计划提前做，渠道爆炸烦恼去。
