## 今日核心概念

### 1. 项目整体变更控制
定义：贯穿项目全生命周期的综合性管理过程，通过对变更请求的审查、评估、批准或否决，确保所有变更都经过正式的审批流程，并协调各知识领域受变更影响后的联动调整。

关键要点：
- 变更控制委员会（CCB）是变更审批的核心决策机构，CCB的组成应在项目管理计划中明确
- 变更请求必须以书面形式提出，包含变更内容、原因、影响分析
- 整体变更控制的输出包括：批准的变更请求、变更日志、更新的项目管理计划、更新的项目文件
- 变更控制流程：提出变更→记录变更请求→评估影响→CCB审批→通知相关方→执行变更→验证变更→归档
- 紧急变更也须走审批流程，可以先批准后补齐手续，但不得跳过审批环节

### 2. 项目范围确认（确认范围）
定义：正式验收已完成的项目可交付成果的过程，由客户或发起人审查可交付成果是否满足项目范围说明书中定义的验收标准和范围要求。

关键要点：
- 范围确认的主体是客户、发起人或他们的授权代表，而非项目团队
- 范围确认的依据是项目管理计划（范围基准）、核实的可交付成果、工作绩效数据
- 确认范围与质量控制不同：质量控制关注可交付成果是否正确（过程检查），范围确认关注可交付成果是否可被接受（结果验收）
- 范围确认的输出：验收的可交付成果、变更请求、工作绩效信息、项目文件更新
- 每个可交付成果完成后都应进行范围确认，确认结果将影响项目收尾

### 3. 质量成本（COQ）
定义：为确保产品或服务满足质量要求而开展的所有活动的总成本，分为一致性成本（预防成本和评估成本）与非一致性成本（内部失败成本和外部失败成本）两大类。

关键要点：
- 预防成本：培训、流程设计、质量规划、供应商评估——投入预防，成本最低
- 评估成本：测试、检查、审计、评审——发现问题的成本
- 内部失败成本：返工、废品、缺陷修复——交付前发现问题的成本
- 外部失败成本：保修、退货、客户投诉、法律责任——交付后发现问题的成本，代价最高
- 考试常考：最优质量水平不是零缺陷，而是在一致性成本与非一致性成本之和最小时的平衡点

### 4. 项目沟通管理
定义：确保项目信息及时、准确、恰当地在项目干系人之间收集、生成、分发、存储、检索和最终处置的过程体系。

关键要点：
- 沟通模型包含：编码→传递信息→解码→确认→反馈，噪声是干扰沟通的因素
- 沟通方式分为：交互式沟通（会议、电话）、推式沟通（邮件、报告）、拉式沟通（知识库、内网门户）
- 沟通渠道数公式：n(n-1)/2，其中n为干系人数量，沟通渠道数随人数快速增长
- 沟通管理计划包含：沟通频率、方式、责任人、信息格式、干系人偏好、升级机制
- 项目经理应花70%~90%的时间进行沟通，沟通不畅是项目失败的首要原因之一

### 5. 合同管理要点
定义：在信息系统集成项目中，对合同从签订、执行到终止的全过程进行管理，确保双方按照合同约定履行各自的权利和义务。

关键要点：
- 合同类型按风险分担：固定总价（FFP）卖方风险最大、成本补偿（CPFF/CPF）买方风险最大、工料合同（T&M）风险居中
- 合同变更须经双方书面同意，变更后的内容成为合同的一部分
- 索赔是合同执行中一方因对方未履行义务而提出补偿要求，索赔程序：索赔意向→索赔报告→索赔审核→协商解决
- 争议解决途径：协商→调解→仲裁→诉讼，仲裁是一裁终局
- 合同收尾时需进行产品验证和管理收尾，合同档案归档保存

## 案例分析题

**题目**：

某市交通管理部门委托某信息系统集成公司建设"智慧交通综合管理平台"，合同金额800万元，工期10个月，采用固定总价合同。项目启动后，项目经理老张组建了12人的开发团队并制定了详细的项目管理计划。

项目执行到第6个月时，发生如下情况：
（1）客户方新增了"车辆违章自动识别"功能模块，老张评估后认为工作量不大，直接安排团队成员在原有任务中穿插实施。
（2）老张每两周向客户发送一封项目进展邮件，但客户在第6个月检查时表示"完全不知道项目做到哪里了"，对项目进展非常不满。
（3）项目组内部开发人员与测试人员因缺陷认定标准不同多次发生争吵，老张采取回避态度，未做任何处理。
（4）第6个月末，老张发现已完成工作的实际成本为520万元，但按计划应完成的工作预算为480万元，实际挣值为400万元。
（5）一名外包开发人员未签署保密协议就参与了核心模块开发，项目结束后该人员离职，客户对此提出安全质疑。

**问题1**：计算第6个月末的CV、SV、CPI、SPI（保留两位小数），并判断项目绩效状态。

**问题2**：请指出老张在上述情况（1）~（5）中分别存在哪些管理问题，并给出正确的做法。

**问题3**：结合固定总价合同的特点，分析本项目面临的主要风险，并提出应对措施。

**参考答案**：

**问题1**：
CV = EV - AC = 400 - 520 = -120万元（成本超支120万元）
SV = EV - PV = 400 - 480 = -80万元（进度落后相当于80万元的计划工作量）
CPI = EV / AC = 400 / 520 ≈ 0.77
SPI = EV / PV = 400 / 480 ≈ 0.83

绩效判断：项目处于成本严重超支、进度明显滞后的不利状态（CPI<1，SPI<1），须立即采取纠正措施。

**问题2**：

情况（1）：
存在问题：未经正式变更审批流程就实施新增功能，属于范围蔓延。
正确做法：应要求客户提交书面变更申请，组织评估变更对范围、进度、成本、质量的影响，提交CCB审批后再执行。

情况（2）：
存在问题：沟通管理不到位，仅使用单向的推式沟通，未与客户确认信息是否被理解和接受。
正确做法：制定沟通管理计划，根据客户偏好采用交互式沟通（如定期现场汇报会），设置沟通频率和方式，关键里程碑节点进行面对面确认。

情况（3）：
存在问题：冲突管理采用回避策略，未积极解决团队内部矛盾。
正确做法：应尽早介入调解，组织开发和测试双方共同制定缺陷认定标准，统一质量判定依据，必要时采取合作/解决问题策略。

情况（4）：
存在问题：挣值监控不足，到第6个月才发现严重偏差，说明日常监控频率不够。
正确做法：应每月或每两周进行挣值分析，发现偏差趋势时及时预警并启动纠正措施，如赶工或快速跟进。

情况（5）：
存在问题：未对外包人员进行安全管理，未签署保密协议即安排核心模块开发。
正确做法：所有参与项目的人员（含外包）必须签署保密协议和知识产权协议，核心模块开发应进行权限控制，项目结束后及时回收权限和资料。

**问题3**：
主要风险：
①固定总价合同下卖方承担全部成本风险，范围蔓延将直接侵蚀利润甚至造成亏损；
②客户对项目进展不满意可能导致验收困难和尾款回收风险；
③团队冲突和核心人员流失影响项目交付质量；
④信息安全合规风险可能引发法律责任和合同纠纷。

应对措施：
①严格变更控制流程，新增功能必须通过CCB审批并签署合同补充协议；
②建立多渠道沟通机制，定期召开项目汇报会，确保客户对项目进展的知情权；
③加强团队建设与冲突管理，统一质量标准，关注核心成员的激励与保留；
④完善信息安全管理，所有人员签署保密协议，实施代码权限控制和版本管理。

**踩分要点**：
- 问题1：四个指标计算正确（每个1分），绩效判断须明确"成本超支""进度滞后"（2分），合计6分
- 问题2：五个情况逐一作答，每个"存在问题"1分+"正确做法"1分，合计10分
- 问题3：风险分析至少答出3点（每点1分），应对措施对应（每点1分），合计6分
- 总分22分，17分以上为合格

## 记忆口诀

**整体变更控制五步诀**

一提申请写书面
二记日志不拖延
三评影响四审批
五通干系六执行
七验成果八归档
紧急变更先审批后补单
无审批不干活是铁律

**质量成本四分法**

预防评估是一致（投在前面不出事）
内失外失非一致（出了问题花代价）
预防最省钱，外失最费钱
考试记住：预防<评估<内失<外失

**沟通管理三问诀**

多少人——沟通渠道n(n-1)/2算
怎么通——交互推拉分清三
通什么——计划定好不瞎忙
项目经理七到九成时间在沟通
沟通不畅项目黄

**合同类型风险诀**

固定总价卖方扛（风险最大）
成本补偿买方慌（风险最大）
工料合同居中央（各有风险好商量）
索赔程序：意向→报告→审核→协商
争议解决：协商调解仲裁诉讼四级上
仲裁一裁终局跑不掉
