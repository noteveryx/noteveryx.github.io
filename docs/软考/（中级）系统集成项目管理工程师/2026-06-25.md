## 今日核心概念

### 1. 软件质量度量模型（McCall模型与ISO/IEC 25010）

定义：软件质量度量模型是评估软件产品质量的结构化框架。McCall模型将质量要素分为三类11个要素，ISO/IEC 25010则定义了8个质量特性，是当前国际标准。

关键要点：
- McCall模型三大类：产品运行（正确性/可靠性/效率/完整性/可用性）、产品修正（可维护性/灵活性/可测试性）、产品转移（可移植性/可重用性/互操作性）
- ISO/IEC 25010八维：功能适用性、性能效率、兼容性、易用性、可靠性、信息安全性、可维护性、可移植性
- 两者关系：McCall模型是早期的经典框架，ISO/IEC 25010是国际现行标准，两者维度有重叠但分类方式不同
- 考试高频点：给定场景判断属于哪个质量维度（如"用户操作三步内完成"→易用性；"系统故障后2小时内恢复"→可靠性）
- 区分要点：McCall的"完整性"对应信息安全，"互操作性"对应兼容性，"可重用性"在ISO 25010中无直接对应

### 2. 网络安全法律法规体系（三法一条例）

定义：《网络安全法》《数据安全法》《个人信息保护法》构成我国网络安全领域法律体系的三大支柱，辅以《关键信息基础设施安全保护条例》等配套法规，形成"三法一条例"的监管格局。

关键要点：
- 《网络安全法》（2017）：确立网络安全等级保护制度（等保2.0的法律依据），要求网络运营者履行安全保护义务，实施实名制、日志留存、安全事件报告等制度。罚款上限：对企业最高100万元、对直接责任人最高10万元
- 《数据安全法》（2021）：建立数据分类分级保护制度，区分一般数据/重要数据/核心数据三级，重要数据出境须经安全评估，核心数据实行更加严格的管理制度。罚款上限：对企业最高1000万元
- 《个人信息保护法》（2021）：确立"告知-同意"为核心的个人信息处理规则，赋予个人查阅权/更正权/删除权/可携带权等七项权利。自动化决策需保证透明和公平，罚款上限：对企业最高5000万元或上年营业额5%
- 三条法律处罚标准对比：个保法罚款最重（5000万/5%）> 数安法（1000万）> 网安法（100万），考试常考处罚档次区分
- 项目实践关联：系统集成项目中涉及用户数据的采集、存储、传输、处理均需遵守三法要求，未履行义务可导致项目被责令停用或罚款

### 3. 企业信息化核心系统（ERP/CRM/SCM/MES）

定义：企业信息化核心系统是企业管理和运营的数字基础，ERP（企业资源计划）、CRM（客户关系管理）、SCM（供应链管理）、MES（制造执行系统）构成企业信息化的四大支柱系统。

关键要点：
- ERP：整合企业全部资源（财务/人力/生产/采购/库存/销售），实现物流、资金流、信息流三流合一。典型厂商：SAP、Oracle、用友、金蝶。实施原则：先固化再优化，一把手工程
- CRM：管理与客户的全部交互，覆盖营销/销售/服务三大模块。核心价值：360度客户视图，提高客户留存率和客单价。运营型CRM（业务流程自动化）、分析型CRM（客户数据分析挖掘）、协作型CRM（多渠道客户互动）
- SCM：管理从供应商到客户的端到端供应链流程，覆盖计划/采购/制造/交付/退货五大环节。核心目标：降低库存成本、提升交付准时率、实现供需协同
- MES：位于ERP与车间自动化之间的执行层，实时采集生产数据（设备状态/产量/质量），实现生产过程的透明化和可追溯。MES向上接ERP的计划指令，向下连PLC/SCADA等设备控制系统
- 考试高频：给定业务场景判断应部署哪类系统；系统之间的关系（ERP是中枢，CRM面向客户，SCM面向供应商，MES面向车间）

### 4. 知识管理与组织过程资产

定义：知识管理是将项目实践中积累的经验、教训、最佳实践进行系统化收集、存储、共享和再利用的管理活动。组织过程资产是知识管理在项目管理中的具体体现。

关键要点：
- 显性知识：可以文字化、编码化、文档化的知识（规范/手册/模板/标准/代码库），易于传播和共享
- 隐性知识：存在于个人头脑中的经验、直觉、判断力（专家经验/项目教训/技术诀窍），难以编码化，需要通过交流/导师制/复盘来传承
- 知识管理三过程：知识获取（经验教训总结/外部引入）→ 知识存储（知识库/文档系统/wiki/组织过程资产库）→ 知识应用（培训/模板重用/案例分析/决策支持）
- 组织过程资产的两大类：流程与程序类（标准/模板/流程/指南/工作授权程序）；共享知识库类（经验教训登记册/度量数据库/风险数据库/以往项目档案）
- 经验教训登记册（Lessons Learned Register）：记录项目中遇到的问题、解决方案、成功实践、避免的错误，项目收尾时更新为最终版并归档为组织过程资产
- 区分要点：经验教训登记册属于项目文件（当前项目使用），归档后成为组织过程资产（未来项目可用）

### 5. 商业智能BI与数据仓库

定义：商业智能（BI）是将企业数据转化为决策信息的技术和工具集合。数据仓库是BI的基础，通过ETL过程将分散的源数据整合为统一的分析数据平台。

关键要点：
- 数据仓库四大特征（Inmon定义）：面向主题（按业务主题组织数据/非按应用）、集成的（统一命名/编码/度量/消除不一致）、随时间变化（保存历史数据/时间键）、非易失的（只读/不修改/批更新）
- ETL三阶段：Extract（从多个异构源系统抽取数据）→ Transform（数据清洗/转换/标准化/去重）→ Load（加载到数据仓库目标表）
- OLAP（联机分析处理）五大操作：切片（固定一个维度值）→ 切块（固定多个维度值）→ 钻取（上钻下钻改变粒度）→ 旋转（行列交换视角）→ 钻透（从汇总钻到明细源数据）
- 数据挖掘四大任务：分类（决策树/朴素贝叶斯，预测离散类别）→ 聚类（K-Means/DBSCAN，无监督分群）→ 关联规则（Apriori，发现购物篮"啤酒+尿布"模式）→ 回归（预测连续数值）
- 星型模式 vs 雪花模式：星型模式（事实表+多个非规范化的维度表，查询快，冗余多）；雪花模式（维度表进一步规范化，节约空间但查询需更多JOIN）
- 考试高频：给定分析需求判断OLAP操作类型；给定数据特征判断适合哪种挖掘方法；区分OLTP（事务处理/增删改/当前数据）与OLAP（分析处理/读为主/历史数据）

## 案例分析题

**题目**：

某市政府启动"智慧行政审批服务系统"建设项目，预算800万元，工期10个月。项目经理小王带领团队进行需求调研后，系统进入开发阶段。

第5个月末，项目状态如下：
- 计划完成工作量价值 PV = 400万元
- 实际完成工作量价值 EV = 310万元  
- 实际花费成本 AC = 420万元
- 项目总预算 BAC = 800万元

在项目执行过程中，出现以下问题：

**问题1**：系统上线试运行期间，用户投诉系统响应缓慢（页面加载超过8秒），且操作流程复杂（办理一项审批需跳转5个页面）。小王的团队分析后认为"只要功能正确就算质量合格"。请从软件质量度量角度分析小王的观点是否正确，并指出系统在ISO/IEC 25010质量模型中存在哪些维度的缺陷，给出改进建议。

**问题2**：系统采集了大量办事群众的个人信息（身份证号、手机号、住址、人脸照片等）。小王认为"政务系统不需要遵守商业网站那么严格的规定"，未采取数据加密和访问控制措施。请从网络安全法律法规（三法一条例）角度指出：
（1）该项目违反了哪部法律的哪些规定？
（2）可能面临何种处罚？
（3）项目经理应在项目中采取哪些安全措施？

**问题3**：该审批系统需要与已有的工商注册系统（自建）、税务系统（上级统建）、不动产登记系统（第三方）进行数据交换。请问：
（1）应采用哪种信息系统集成模式（EAI模式）？
（2）设计数据交换方案时应考虑哪些关键技术要素？

**问题4**：请计算第5月末的CV、SV、CPI、SPI，评估成本和进度绩效，并计算典型偏差下完工尚需估算ETC和完工估算EAC。

**参考答案**：

**问题1**：
小王观点错误。软件质量不仅指功能正确性，还包括性能效率、易用性等多个维度。

ISO/IEC 25010质量模型缺陷分析：
- 性能效率缺陷：页面加载超过8秒，未满足用户响应时间要求（一般要求3秒以内）
- 易用性缺陷：办理一项审批需跳转5个页面，操作效率低下，用户学习成本高
- 功能适用性：功能逻辑正确，该维度基本满足

改进建议：
- 性能优化：数据库索引优化、缓存机制、CDN加速、前端懒加载、代码压缩
- 易用性优化：减少跳转步骤（合并相关页面）、提供"一表式"申报、引入向导式引导、优化导航设计
- 建立性能基线和易用性测试标准，纳入非功能需求规格（SRS中明确性能指标和可用性指标）

**问题2**：
（1）违反的法律及规定：
- 《个人信息保护法》：未取得用户"告知-同意"即大规模采集敏感个人信息（身份证号/人脸照片）；未对个人信息采取加密等安全保护措施；违反"最小必要原则"，采集范围超出业务必需
- 《网络安全法》：未履行网络安全保护义务（第21条），未采取数据加密/访问控制等技术措施
- 《数据安全法》：未建立数据分类分级保护制度，未对个人信息实施相应安全保护

（2）可能面临的处罚：
- 《个人信息保护法》第66条：情节严重的，处5000万元以下或上年营业额5%以下罚款，并可责令暂停相关业务、停业整顿、吊销相关业务许可或营业执照；对直接责任人处10万-100万元罚款，并可禁止一定期限内担任相关职务
- 《网络安全法》：拒不改正或导致危害的，处1万-10万元罚款，对直接主管人员处5000元-5万元罚款

（3）应采取的安全措施：
- 数据加密：个人信息传输和存储必须加密（HTTPS+AES256）
- 访问控制：实施基于角色的访问控制（RBAC），最小权限原则
- 数据脱敏：非必须展示的敏感字段（身份证号/手机号）进行脱敏处理（如显示为320***********1234）
- 安全审计：记录所有数据访问日志，定期审计
- 合规流程：上线前完成个人信息保护影响评估（PIA），制定并公示隐私政策

**问题3**：
（1）应采用企业应用集成（EAI）的混合模式：
- 与工商注册系统（自建系统）→ 数据层集成（直接数据库对接/中间件）
- 与税务系统（上级统建）→ 表示层集成/接口层集成（通过Web Service/SOA/API网关调用，因无法控制上级系统内部）
- 与不动产登记系统（第三方）→ 接口层集成（RESTful API/Web Service，通过标准协议对接）

（2）数据交换方案关键技术要素：
- 数据交换标准：定义统一的数据交换格式（XML/JSON）和数据字典
- 接口规范：制定API接口文档，明确请求/响应格式、认证方式、错误码
- 安全机制：HTTPS加密传输、Token/OAuth2.0鉴权、数字签名防篡改
- 消息队列：引入MQ实现异步解耦，保证数据最终一致性
- 数据映射与转换：不同系统编码标准不一（如区域编码/事项编码），需要建立映射表和转换规则
- 异常处理：接口超时/失败时的重试机制和补偿方案

**问题4**：
- CV = EV - AC = 310 - 420 = -110万元（成本超支110万元）
- SV = EV - PV = 310 - 400 = -90万元（进度滞后90万元）
- CPI = EV / AC = 310 / 420 ≈ 0.738（每花1元仅完成0.738元的价值，成本效率低）
- SPI = EV / PV = 310 / 400 = 0.775（进度效率为计划的77.5%）

绩效评估：CPI < 1 且 SPI < 1，项目成本超支、进度滞后，情况较差，需立即采取纠正措施。

典型偏差下（假定当前偏差将持续）：
- ETC = (BAC - EV) / CPI = (800 - 310) / 0.738 ≈ 663.96万元
- EAC = AC + ETC = 420 + 663.96 ≈ 1083.96万元

预测：按照当前效率继续执行，项目最终将花费约1084万元，超出预算284万元，需申请追加预算或采取成本压缩措施。

**踩分要点**：
- 问题1：明确指出"功能正确不等于质量合格"（2分）；正确识别性能效率和易用性两个缺陷维度（各1分）；给出具体改进建议（2分）
- 问题2：准确指出违反个保法（未告知同意/安全措施缺失，2分）、网安法（未履行安全保护义务，1分）；处罚力度正确（2分）；安全措施至少列出3项（3分）
- 问题3：EAI模式选择要有理由（不同系统用不同模式，2分）；数据交换技术要素至少列出4项（4分）
- 问题4：四个指标计算正确各1分（共4分）；绩效评估判断正确（1分）；ETC和EAC公式使用正确各2分（共4分）
