## 今日核心概念

### 1. 关键路径法（CPM）
定义：在不考虑任何资源限制的情况下，通过计算项目网络路径上所有活动的最早开始（ES）、最早完成（EF）、最晚开始（LS）、最晚完成（LF）时间，确定项目中耗时最长的活动路径即为关键路径。

关键要点：
- 关键路径上的总浮动时间（TF）恒为0，活动延迟将直接拖后项目总工期
- 一个项目网络可能存在多条关键路径，关键路径越多风险越大
- 总浮动时间 TF = LS - ES = LF - EF，自由浮动时间 FF = 下一活动ES - 本活动EF
- 进度压缩中的"赶工"会增加成本缩短工期，"快速跟进"将串行改并行但增加返工风险
- 正推法确定ES/EF，逆推法确定LS/LF

### 2. 挣值管理（EVM）
定义：将范围、时间、成本三个维度综合为统一的量化指标体系，通过计划值（PV）、挣值（EV）、实际成本（AC）三个核心参数衡量项目的实际绩效与进度偏差。

关键要点：
- CV = EV - AC（成本偏差），SV = EV - PV（进度偏差）—— 正值有利
- CPI = EV / AC（成本绩效指数），SPI = EV / PV（进度绩效指数）—— >1有利
- ETC = (BAC - EV) / CPI（典型偏差预测完工尚需），ETC = BAC - EV（非典型偏差）
- EAC = AC + ETC（完工估算），TCPI = (BAC - EV) / (BAC - AC)（完工尚需绩效指数）
- BAC为完工预算，EVM适用于所有项目的绩效监控和控制过程

### 3. 工作分解结构（WBS）
定义：以可交付成果为导向，对项目团队为实现项目目标、创建所需可交付成果而需要实施的全部工作范围的层级分解。

关键要点：
- WBS最低层的组件称为"工作包"，工作包可进一步分解为"活动"
- WBS是范围基准的核心组成部分，100%规则：WBS必须100%覆盖项目工作范围
- 创建WBS的过程是"创建工作分解结构"，属于项目范围管理知识领域
- WBS词典对每个WBS组件进行详细说明，包含工作描述、负责组织、里程碑、资源需求等
- 分解的原则：上层按可交付成果、下层按工作内容，同层采用相同分解方法

### 4. 配置管理
定义：对项目产品、成果、服务或组件进行识别、记录、控制、核实的技术和管理活动，确保项目各阶段的产出物状态可追溯、版本可控。

关键要点：
- 配置管理活动包括：配置识别、配置状态记录、配置核实与审计
- 配置项（CI）是纳入配置管理的对象的最小单元，变更必须经过配置控制流程
- 配置管理计划是项目管理计划的组成部分，定义配置项识别规则和控制流程
- 版本控制是配置状态记录的核心手段，基线是正式评审通过的配置项集合
- 区别：配置管理侧重产品成果的技术规范，变更控制侧重项目变更的审批流程

### 5. 风险管理计划与风险登记册
定义：风险管理计划是描述如何安排与实施项目风险管理活动的纲领性文件；风险登记册是识别风险、分析风险、规划应对的全过程记录。

关键要点：
- 风险管理计划包含：方法论、角色职责、预算、时间安排、风险类别、风险概率与影响矩阵
- 风险登记册包含：已识别风险清单、潜在风险责任人、潜在风险应对措施清单
- 定性风险分析：概率影响矩阵法、风险数据质量评估；定量风险分析：蒙特卡洛模拟、决策树分析
- 消极风险应对策略：规避、转移、减轻、接受；积极风险应对策略：开拓、分享、提高、接受
- 风险再评估和风险审计是风险控制过程中的两项重要活动

## 案例分析题

**题目**：

某信息系统集成公司承接了一个电子政务平台建设项目，合同金额500万元，工期8个月。项目经理小王制定了项目管理计划后即投入执行。项目进行到第4个月末时，小王收集了如下数据：
- 计划完成工作预算（PV）：250万元
- 已完成工作实际成本（AC）：300万元
- 已完成工作预算（EV）：220万元

此时，客户提出一项功能变更需求，小王认为变更不大，口头同意了客户的要求并安排开发人员实施。第5个月结束时，该变更导致额外支出20万元工时。项目团队成员因连续加班士气低落，有两名骨干成员提出离职。

**问题1**：计算第4个月末的CV、SV、CPI、SPI，并分析项目当前绩效状况。

**问题2**：假设后续月份不再发生新变更且绩效按典型偏差延续，分别计算进度执行指数不变和成本绩效保持不变两种情形下的EAC（完工估算）。

**问题3**：指出小王在项目管理过程中存在哪些主要问题，并给出改进建议。

**参考答案**：

**问题1**：
CV = EV - AC = 220 - 300 = -80万元（成本超支）
SV = EV - PV = 220 - 250 = -30万元（进度滞后）
CPI = EV / AC = 220 / 300 ≈ 0.73（每花1元仅完成0.73元的预算工作）
SPI = EV / PV = 220 / 250 = 0.88（进度完成88%）

绩效分析：项目成本和进度均偏离基准，CPI < 1 且 SPI < 1，表明项目处于成本超支、进度滞后的不利状态，须立即采取纠正措施。

**问题2**：
情形一（典型偏差，CPI=0.73）：EAC = BAC / CPI = 500 / 0.73 ≈ 684.93万元
情形二（含变更后重估，BAC实际=520万，CPI=0.73）：按新BAC计算 EAC = 520 / 0.73 ≈ 712.33万元

**问题3**：
存在问题：
①未建立变更控制流程，口头同意变更导致范围蔓延，未进行变更影响评估；
②未进行挣值管理分析，第4个月才发现严重偏差，监控不及时；
③忽视团队管理，连续加班导致核心成员离职，未做好团队建设和冲突管理；
④合同金额500万但未进行成本风险管控，变更未评估费用影响。

改进建议：
①建立正式的整体变更控制流程，所有变更须提交变更申请并经CCB审批；
②增加挣值分析频率（建议双周或月度），发现偏差及时启动纠正措施；
③实施团队建设活动，评估工作量并适当增补人力，关注成员心理状态；
④变更实施前必须评估对范围、进度、成本、质量的影响，形成书面记录。

**踩分要点**：
- 问题1：四个指标计算正确（每个1分），绩效分析须点出"成本超支"和"进度滞后"（2分），合计6分
- 问题2：正确使用典型偏差公式（2分），分清是否含变更成本（2分），合计4分
- 问题3：至少答出变更控制、挣值监控、团队管理三个维度（每个2分），改进建议对应三个维度（每个1分），合计9分
- 总分19分，15分以上为合格

## 记忆口诀

**挣值公式速记诀**

PV计划值、EV是挣值、AC实际成本记心里
CV成本差，SV进度差 —— EV减去PV和AC就对了
CPI和SPI都是除，EV是分子不动摇
完工估算EAC，BAC除以CPI，或者AC加ETC
完工尚需TCPI，（BAC减EV）除（BAC减AC）不能忘！

**配置变更区分诀**

配置管产品（技术规格、版本号）
变更管过程（谁审批、怎么走流程）
基线是铁门，变更要申请
CCB来决策，实施要记录
没有审批不动工，一刀切守底线

**风险管理策略顺口溜**

消极风险四项法：规避（躲开）、转移（外包、买保险）、减轻（降低概率或影响）、接受（认了）
积极风险四项法：开拓（确保发生）、分享（拉合伙）、提高（加大机会）、接受（顺其自然）
风险登记册跟着走，识别分析到应对
控制过程再评估，新风险要补录
