## 今日核心概念

### 1. 自制与外购分析
定义：用于确定某项工作或可交付成果最好由项目团队自行完成，还是从外部采购。是采购管理规划过程的核心工具。
关键要点：
- 自制：项目团队内部完成，控制力强但占用内部资源
- 外购：从外部获取，释放内部资源但增加管理成本
- 分析时需考虑直接成本（人工、材料、设备）和间接成本（管理费用、行政费用）
- 不仅要考虑财务成本，还需考虑技术能力、交付周期、供应商稳定性、战略保密等因素
- 教材中的"租赁还是购买"分析可视为自制外购分析的特殊情形

### 2. 工作绩效数据、工作绩效信息、工作绩效报告
定义：三者是监控过程组中从原始数据到决策报告的信息层次递进关系。
关键要点：
- 工作绩效数据：执行过程中收集的原始观察结果和测量值（如实际开始日期、实际成本、缺陷数量），未经任何加工
- 工作绩效信息：对工作绩效数据进行综合分析后得到的结果（如偏差分析、挣值数据、预测的完工估算），在各控制过程中产生
- 工作绩效报告：对工作绩效信息进行汇总编制的实体或电子文档，用于制定决策、提出问题、采取行动（如状态报告、备忘录、论证报告）
- 考试陷阱：数据→信息→报告，这是单向递进，不能混淆
- 监控项目工作过程输入"工作绩效信息"，输出"工作绩效报告"；整体变更控制的关键输入之一

### 3. SWOT分析
定义：从优势、劣势、机会、威胁四个维度对项目或组织进行全面审视的风险识别技术。
关键要点：
- 优势与劣势：属于内部因素（技术能力、团队经验、资金资源、管理制度）
- 机会与威胁：属于外部因素（市场趋势、政策变化、竞争对手、技术变革）
- 识别风险时需逐项考察，将SO（优势+机会）作为积极风险、WT（劣势+威胁）作为消极风险
- 可与头脑风暴、德尔菲技术结合使用，提升风险识别的全面性
- SWOT分析的输出结果直接纳入风险登记册

### 4. 项目工作说明书
定义：对项目需交付的产品、服务或成果的叙述性说明，由发起人或客户提供，是制定项目章程的核心输入文件。
关键要点：
- 内容包括：业务需要（为什么做）、产品范围描述（做什么）、战略计划（和公司战略如何对齐）
- 内部项目由项目发起人基于业务需要提供SOW
- 外部项目（招投标场景）SOW即招标文件的一部分，由客户提供
- SOW与项目范围说明书的区别：SOW是项目启动前的粗略描述，范围说明书是规划后的详细定义
- SOW是制定项目章程的输入，项目章程是制定项目管理计划的输入——要牢记输入输出顺序

### 5. 信息安全CIA三元组
定义：信息安全的三个核心属性——机密性、完整性、可用性，是信息系统安全的基本框架。
关键要点：
- 机密性：确保信息仅被授权者访问（如加密、访问控制、身份认证）
- 完整性：确保信息在存储和传输过程中不被未授权篡改（如数字签名、哈希校验、版本控制）
- 可用性：确保授权用户在需要时可以访问信息（如冗余设计、容灾备份、DDoS防护）
- 中级考试常见场景：某系统遭SQL注入攻击导致数据泄露→机密性受损；日志被篡改→完整性受损；服务器宕机无法访问→可用性受损
- 信息安全等级保护（等保）是中国特有的安全管理制度，中级考试有一定出题概率

## 案例分析题

**题目**：

某市人力资源和社会保障局委托A公司建设"智慧人社综合服务平台"，项目预算1200万元，工期12个月。项目启动后，项目经理小王组织了项目启动会议，由发起人签发了项目章程。在规划阶段，小王发现平台中的"电子档案管理模块"技术难度较高，团队内部缺乏相关经验。小王进行了自制与外购分析：自制方案预估成本350万元（含新增3名开发人员的人力成本），外购方案（采购成熟的第三方电子档案系统并定制开发）预估成本280万元。

小王还组织了风险识别会议，采用SWOT分析得出以下结果：
- 优势：公司有丰富的政务系统开发经验，团队熟悉人社业务流程
- 劣势：缺乏电子档案系统开发经验，原定技术架构与电子档案模块不兼容
- 机会：国家正大力推进政务服务数字化转型，市场空间大
- 威胁：竞争对手已推出同类产品，项目进度压力大

项目执行到第6个月末，相关数据如下：
- 计划价值：PV=620万元
- 挣值：EV=530万元
- 实际成本：AC=680万元
- 已完成工作占总工作量的45%

项目此时还暴露出两个问题：
（1）信息中心反馈：平台在一次渗透测试中发现SQL注入漏洞，攻击者可获取参保人员个人信息；同时文件上传接口缺少校验，可能被植入恶意文件。
（2）项目发起人要求每周提交工作绩效报告，但小王提交的报告内容过于简略，只列出了本周完成的几项任务名称，缺乏数据支撑和趋势分析。

**问题1**（6分）：请计算项目第6个月末的CV、SV、CPI、SPI，并分析当前项目绩效情况。
**问题2**（6分）：基于自制与外购分析结果，你建议小王选择哪种方案？请说明理由（至少回答3点）。
**问题3**（4分）：结合案例，SWOT分析的"劣势"对应了什么具体风险？请指出至少2项针对性的风险应对措施。
**问题4**（4分）：案例中信息中心反馈的2个安全问题分别违反了CIA三元组中的哪些属性？各应如何修复？
**问题5**（5分）：小王提交的工作绩效报告存在哪些不足？一份合格的工作绩效报告应包含哪些内容？

**参考答案**：

**问题1**：
CV = EV - AC = 530 - 680 = -150万元（成本超支150万）
SV = EV - PV = 530 - 620 = -90万元（进度落后90万）
CPI = EV / AC = 530 / 680 ≈ 0.78（每花1元只完成0.78元的工作）
SPI = EV / PV = 530 / 620 ≈ 0.85（进度效率为计划的85%）

当前绩效：成本超支且进度落后，CPI<1且SPI<1，项目处于双偏差状态，需立即采取纠正措施。

**问题2**：
建议选择外购方案。理由：
（1）成本优势：外购280万低于自制350万，节省70万元
（2）技术风险低：团队缺乏电子档案系统开发经验，自制可能产生大量试错成本，且工期不可控
（3）快速交付：成熟产品经过市场验证，定制开发周期短于从零开发，有助于缓解进度压力
（4）安全可靠：成熟产品通常已完成安全测试，而团队自行开发可能留下安全隐患

**问题3**：
具体风险："缺乏电子档案系统开发经验"→可能导致模块质量不达标、交付延期、成本超支；"技术架构不兼容"→可能导致系统集成失败、返工风险。

应对措施：
（1）技术架构不兼容：采用回避策略，通过外购方案从根本上避免架构兼容性问题（若必须自制：采用减轻策略，提前进行架构评估与技术预研）
（2）缺乏开发经验：采用转移策略，将电子档案模块外包给有经验的团队；或采用减轻策略，引进专家顾问+团队培训

**问题4**：
（1）SQL注入漏洞可获取个人信息→违反机密性。修复：参数化查询、输入过滤、Web应用防火墙（WAF）
（2）文件上传接口缺少校验→同时违反完整性和可用性。修复：文件类型白名单校验、文件大小限制、病毒扫描、上传目录设置不可执行权限

**问题5**：
不足：
（1）只有任务名称列表，缺乏量化数据（如挣值、偏差分析）
（2）缺少趋势分析和预测（如EAC、ETC、进度预测）
（3）未体现问题和风险的当前状态及应对进展
（4）未与基准计划对比，无法判断项目健康度

合格的工作绩效报告应包含：
（1）本期完成工作的量化数据和与计划的偏差分析
（2）挣值分析数据（PV、EV、AC、CV、SV、CPI、SPI）
（3）风险与问题的最新状态及应对措施
（4）下期工作计划和资源需求
（5）需要管理层决策的事项

**踩分要点**：
- 问题1：CV/SV各1分，CPI/SPI各1分，绩效分析2分（必须点出"双偏差"或"成本超支且进度落后"）
- 问题2：明确给出结论1分，每条合理理由各1-2分，必须结合案例具体情况（不能只背理论）
- 问题3：具体风险识别2分（对应SWOT劣势），应对措施每条1分（必须区分策略类型）
- 问题4：每个安全问题违规属性1分，每个修复措施1分（共4分）
- 问题5：不足2分（至少2点），报告内容2分（至少2点），整体表述1分

## 记忆口诀

**自制外购分析诀**
自制省钱控得住，外购省心风险输
算钱别忘间接费，技术周期也得顾
自有能力比三方，战略保密不含糊

**工作绩效三层次诀**
原始数据是基础，分析处理变信息
汇总成报告上报，层层递进不倒挂
数据→信息→报告，监控链条不能差

**SWOT分析决**
内部优劣自可控，外部机会夹威胁
SO联手是良机，WT合伙成危机
优势放大机会抓，劣势补上威胁避

**CIA三元组诀**
机密不让偷着看，完整防篡改不变
可用想用就要在，三性缺一都算瘫
配加密、加签名、做容灾，三招保安全

**工作绩效报告编写决**
光列任务太敷衍，数据偏差要呈现
挣值图表加分析，风险问题跟进展
趋势预测给建议，管理决策有支撑
