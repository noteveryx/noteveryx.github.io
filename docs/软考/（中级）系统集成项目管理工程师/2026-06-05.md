## 今日核心概念

### 1. PERT三点估算与持续时间估算
定义：计划评审技术对活动持续时间进行三种估算——最乐观时间、最可能时间、最悲观时间，通过加权平均计算期望工期。

关键要点：
（1）期望工期计算公式：期望值 = （最乐观时间 + 4×最可能时间 + 最悲观时间）/ 6
（2）标准差计算公式：标准差 = （最悲观时间 - 最乐观时间）/ 6
（3）方差 = 标准差的平方 = [（最悲观 - 最乐观）/ 6]²
（4）项目总工期的标准差 = 各关键路径活动方差之和的平方根
（5）在±1个标准差范围内完成的概率为68.26%，±2个标准差为95.46%，±3个标准差为99.73%
（6）与CPM关键路径法的区别：CPM用单一估算值，PERT考虑不确定性；PERT更适用于研发类、不确定性较高的项目
（7）考试高频考点：给定三个估算值，先算期望值和标准差，再算某一工期范围内的完成概率

### 2. 决策树分析与期望货币价值
定义：决策树是一种图形化的决策分析工具，将不同方案及其可能的结果以树状结构呈现，通过计算各路径的期望货币价值来辅助决策。

关键要点：
（1）决策树节点类型：决策节点（方框）、机会节点（圆圈）、结果节点（三角/终点）
（2）EMV计算公式：EMV = 各结果概率 × 对应收益/损失，求和
（3）决策准则：选择EMV最大的方案（收益型）或EMV最小的方案（成本型）
（4）决策树与EMV主要用于定量风险分析，回答"该选择哪个方案"的问题
（5）与SWOT的区别：SWOT是定性识别，决策树+EMV是定量比较
（6）考试典型出法：给出两种技术方案的成败概率和收益/损失，画决策树算EMV选最优方案
（7）常见误区：只看结果值不看概率导致错误决策；忽略机会成本

### 3. 项目财务评价指标（NPV、投资回收期、ROI）
定义：项目财务评价是从经济效益角度评估项目是否值得投资的一组定量指标，常用指标包括净现值、投资回收期、投资回报率。

关键要点：
（1）净现值（NPV）：将项目各期净现金流量按一定折现率折算到当前的价值总和。NPV > 0则项目可行；NPV越大越优
（2）NPV计算公式：NPV = Σ（第t年净现金流量 / （1+折现率）^t） - 初始投资
（3）投资回收期：项目累计净收益抵偿全部投资所需的时间。分为静态回收期（不考虑资金时间价值）和动态回收期（考虑折现）
（4）投资回报率（ROI）：年均净收益 / 投资总额 × 100%，反映投资效率
（5）NPV > 0是项目立项的必要条件之一，在可行性研究（经济可行性）中重点考察
（6）中级考试重点：给定现金流量表，计算静态/动态回收期和NPV，判断项目是否可行
（7）与挣值管理的区别：财务评价用于立项决策（项目启动前），挣值管理用于项目执行监控（项目进行中）

### 4. ISO9000质量管理体系与CMMI
定义：ISO9000是国际标准化组织制定的质量管理体系标准族，CMMI是能力成熟度模型集成，两者是系统集成企业进行质量管理的核心参考框架。

关键要点：
（1）ISO9000核心理念：以顾客为关注焦点、领导作用、全员参与、过程方法、持续改进、循证决策、关系管理（七项质量管理原则）
（2）ISO9001是认证标准，规定质量管理体系要求；ISO9000是基础和术语；ISO9004是追求持续成功指南
（3）CMMI五个成熟度等级：初始级（1级）、已管理级（2级）、已定义级（3级）、量化管理级（4级）、优化级（5级）
（4）CMMI阶段式模型从1级逐步提升到5级，每个等级包含若干过程域
（5）CMMI 2级核心过程域：需求管理、项目策划、项目监控、供应商协议管理、度量分析、过程和产品质量保证、配置管理
（6）考试重点：ISO9000八项原则（2015版改为七项）、CMMI五级名称和特征、CMMI各等级对应的管理特征
（7）与质量管理工具的区别：ISO9000/CMMI是体系框架（管整体），质量管理工具（因果图、控制图等）是具体方法（管细节）

### 5. 知识产权与标准化
定义：知识产权是法律赋予权利人对其智力创造成果的专有权利，标准是对重复性事物所做的统一规定。系统集成项目涉及的软件著作权、专利权、国家标准等均在此范畴。

关键要点：
（1）计算机软件著作权：自软件开发完成之日起自动产生，保护期50年（自然人终生加死后50年，法人/组织为首次发表后50年）
（2）专利权：发明（保护期20年）、实用新型（10年）、外观设计（15年），均自申请日起算；授予专利需满足新颖性、创造性、实用性
（3）委托开发软件著作权归属：有合同约定从约定，无约定归受托人（开发者）；合作开发著作权由合作方共同享有
（4）国家标准（GB/GB/T）：强制性标准（GB）必须执行；推荐性标准（GB/T）鼓励采用；企业标准不得低于国家标准
（5）项目招标文件中引用国家标准时，若为强制标准则必须满足，若为推荐标准则应明确是否作为实质性要求
（6）投标人资质要求中不得以不合理的条件限制或排斥潜在投标人，不得对潜在投标人实行歧视待遇
（7）考试高频考点：知识产权归属判断、专利类型与保护期、标准的分类与适用

## 案例分析题

**题目**：

某市大数据管理局启动"智慧政务数据共享交换平台"建设项目，预算500万元，工期10个月。甲公司中标，项目经理小陈负责实施。

项目启动阶段，小陈组织团队估算活动持续时间。其中"数据接口开发"模块有三名资深工程师估算——最乐观20天、最可能30天、最悲观46天；"数据清洗引擎"模块有两名新入职员工估算——最乐观15天、最可能25天、最悲观65天。

项目执行到第5个月末，发生以下情况：
（1）数据接口开发实际耗时35天完成，但数据清洗引擎因技术难度高发生返工，实际耗时55天
（2）供应商提供的数据库一体机出现兼容性问题，导致系统集成测试延误两周
（3）项目经理发现部分非功能性需求（系统并发处理能力、数据加密等级）在需求规格说明书中未明确，甲乙双方产生争议；甲方认为应当满足政务云三级等保要求和国家标准GB/T 22239，乙方认为合同中未明确约定

项目财务数据显示：初始投资500万元，预计项目交付后年均运维净收益80万元（折现率8%），项目运维期5年。

**问题1**：请用PERT三点估算法计算"数据接口开发"和"数据清洗引擎"两个模块的期望工期和标准差，并分析哪个模块的估算风险更大。（8分）

**问题2**：分析第5个月末出现的问题分别属于哪些项目管理知识领域，并给出处理建议。（8分）

**问题3**：关于非功能性需求争议，国家标准GB/T 22239的性质是什么？甲方要求是否合理？项目经理应如何处理？（7分）

**问题4**：计算该项目的净现值（NPV），判断项目是否可行。如回收期超过4年，是否需要重新论证？（7分）

**参考答案**：

**问题1**：
数据接口开发：期望工期 = （20+4×30+46）/6 = 186/6 = 31天；标准差 = （46-20）/6 = 4.33天
数据清洗引擎：期望工期 = （15+4×25+65）/6 = 180/6 = 30天；标准差 = （65-15）/6 = 8.33天

虽然两个模块的期望工期相近（31天和30天），但数据清洗引擎的标准差为8.33天，是数据接口开发（4.33天）的近2倍。标准差越大说明估算的不确定性越高、风险越大，因此数据清洗引擎的估算风险明显更大。这也与实际情况吻合——新入职员工估算经验不足，导致悲观与乐观估算跨度过大（跨度50天），返工风险高。

**问题2**：
（1）兼容性问题属于采购管理和风险管理领域。处理建议：首先依据采购合同向供应商提出索赔或要求更换/修复设备；同时启动风险应对预案（如临时租用兼容设备）；更新风险登记册和问题日志。
（2）需求不明确争议属于范围管理和沟通管理领域。处理建议：立即召开干系人会议明确需求；补充编制需求规格说明书；评估需求变更对成本和进度的影响；执行整体变更控制流程；更新项目文件。

**问题3**：
GB/T 22239《信息安全技术 网络安全等级保护基本要求》是国家推荐性标准（"T"表示推荐性）。甲方要求满足政务云三级等保——三级等保是强制性要求（《网络安全法》明确规定），不是推荐性标准，甲方要求完全合理。即使合同中未明确约定，满足法律法规的强制性要求是合同履行的法定义务，乙方不能以"合同未约定"为由拒绝。

项目经理处理方式：确认三级等保属于法定强制要求；组织技术团队评估实现方案及额外成本；向甲方说明增加的工作量和成本，协商合同补充协议；更新需求规格说明书和项目计划；执行整体变更控制流程。

**问题4**：
每年净现金流入 = 80万元，折现率8%，运营期5年
NPV = 80/（1+8%）+ 80/（1+8%）² + 80/（1+8%）³ + 80/（1+8%）⁴ + 80/（1+8%）⁵ - 500
= 80×0.9259 + 80×0.8573 + 80×0.7938 + 80×0.7350 + 80×0.6806 - 500
= 74.07 + 68.58 + 63.50 + 58.80 + 54.45 - 500
= 319.40 - 500 = -180.60万元

NPV < 0，项目在经济上不可行。静态投资回收期约6.25年（500÷80），远超4年。需要重新论证项目的必要性和经济合理性，建议从三个方面重新评估：（1）核实年均净收益估算是否过于保守；（2）考虑项目的社会效益和战略价值（智慧政务平台属于公共服务，不能仅以经济效益衡量）；（3）寻找降低成本或增加收益的优化方案。

**踩分要点**：
- 问题1：公式正确（2分），结果正确（2分），比较分析（2分），与实际关联（2分）
- 问题2：正确识别知识领域（4分），建议具体可行（4分）
- 问题3：指出T为推荐性标准（1分），指出三级等保属法律强制要求（2分），合同未约定不构成免责理由（2分），处理步骤完整（2分）
- 问题4：NPV计算过程和结果正确（3分），判断项目不可行（1分），提出多维论证建议且逻辑清晰（3分）

## 记忆口诀

**三点估算公式诀**
乐观加悲观，四倍最可能，六分之一除，期望自然来。
标准差公式：悲减去乐，除以六来，方差再平方，求和再开方。

**NPV判定诀**
净现值看正负，正可行来负否决。折现率折未来，钱越远来越不值。
回收期看快慢，越短越好风险小。回收期超标准，重新论证有必要。

**ISO9000与CMMI区分诀**
ISO看体系，原则七项记心里：顾客焦点第一位，领导作用紧跟随，全员参与不可少，过程方法来管理，持续改进不能停，循证决策有依据，关系管理聚合力。
CMMI看等级，一二三四五升级：初始级乱哄哄，已管理级有规程，已定义级标准化，量化管理靠数据，优化级持续改。

**知识产权归属诀**
著作权自动生，发表之日五十年。
专利权需申请，发明二十实新十，外观设计十五年。
委托开发按约定，没有约定归受托。
合作开发共同享，单独转让需协商。

**等保与标准适用诀**
GB强制必须做，GB/T推荐鼓励用。
等保三级是法定，合同没写也得行。
招标要求引标准，强制不满足直接否，推荐不满足可说明。
