## 今日核心概念

### 1. 信息系统总体规划方法——诺兰阶段模型
定义：诺兰（Nolan）模型将组织信息系统发展划分为六个阶段，描述信息化从引入到成熟的演进规律，是信息系统总体规划的重要理论依据。

关键要点：
- 六个阶段：初装（单机应用）→ 蔓延（部门级独立系统激增）→ 控制（加强管理、统一规划）→ 集成（从分散到整合、建设集中式系统）→ 数据管理（以数据为核心、统一数据标准）→ 成熟（信息资源全面支撑战略决策）
- 前三阶段（初装/蔓延/控制）属于"计算机时代"，关注技术扩散与管控
- 后三阶段（集成/数据管理/成熟）属于"数据时代"，关注数据整合与价值挖掘
- 不可跳跃：每个阶段必须依次经历，跳跃会导致信息化建设根基不稳
- 考试要点：判断某组织当前处于哪个阶段、给出下一阶段的建设建议

### 2. 信息系统总体规划方法——BSP法（企业系统规划法）
定义：BSP（Business Systems Planning）是IBM提出的自上而下的信息系统规划方法，通过分析企业目标和业务过程，确定信息系统的总体结构。

关键要点：
- 核心思想：先分析业务过程，再导出信息需求，最后形成信息系统结构
- 四步流程：定义企业目标 → 定义业务过程 → 定义数据类 → 定义信息系统总体结构
- 输出产物：BSP研究最终产出U/C矩阵（Use/Create矩阵），用于识别子系统边界
- U/C矩阵规则：每个数据类有且仅有一个C（创建者），但可有多个U（使用者）；同一子系统内U和C应尽量集中在对角线附近
- 优点：自上而下规划、自下而上实现，确保信息系统与业务战略对齐

### 3. 信息系统总体规划方法——CSF法（关键成功因素法）与SST法（战略目标集转化法）
定义：
- CSF（Critical Success Factors）：识别管理者最关注的关键成功因素，围绕这些因素规划信息系统
- SST（Strategy Set Transformation）：将组织战略目标集合转化为信息系统战略目标集合

关键要点：
- CSF法四步：识别组织目标 → 识别CSF → 识别性能指标 → 定义数据字典与信息系统需求
- CSF适用场景：高层管理者需求明确、业务目标清晰的场景（如银行风控系统）
- SST法两步：识别组织战略集（使命/目标/战略/其他战略变量）→ 将组织战略集转化为信息系统战略集（系统目标/系统约束/系统设计战略）
- SST优点：确保信息系统战略与组织战略严格对齐
- 三方法对比记忆：BSP重"业务过程→数据类"、CSF重"管理者关注点→关键指标"、SST重"组织战略→系统战略"

### 4. 构件技术与软件复用
定义：构件（Component）是系统中可独立部署、具有明确接口和契约的软件单元。软件复用是将已有软件资产用于新系统开发的过程，降低开发成本并提高质量。

关键要点：
- 构件三大特征：可独立部署、具有明确接口（契约式设计）、可组合
- 常见构件模型：COM/DCOM（微软）、CORBA（OMG组织/跨平台跨语言）、EJB（Java企业级）、CCM（CORBA构件模型）
- 软件复用三个级别：代码复用（最低级/函数库/类库）→ 设计复用（设计模式/架构模式）→ 分析复用（需求模型/领域模型）
- 构件组装方式：源代码组装、二进制组装、服务组装（SOA方式）
- 构件库管理：构件描述、分类、检索、版本管理；构件获取方式包括自开发、商业采购、开源获取、遗留系统再工程
- 适用条件：需求相对稳定、领域知识成熟、存在可复用的标准化构件

### 5. 项目进度管理综合计算考点——总浮动时间与自由浮动时间
定义：
- 总浮动时间（TF）：活动可以延误而不影响项目总工期的最大时间
- 自由浮动时间（FF）：活动可以延误而不影响任一紧后活动最早开始时间的最大时间

关键要点：
- 计算公式：TF = LS - ES = LF - EF，FF = min(紧后活动ES) - 本活动EF
- TF可能被后续活动"继承"，但FF是独立的——一个活动消耗了FF不会影响任何其他活动
- 关键路径上所有活动的TF=0且FF=0
- 考试陷阱：某活动"延误了X天"要先判断X是否超过TF（超过会影响总工期），再判断X是否超过FF（超过会影响紧后活动）
- 进度压缩与浮动：赶工（加班加资源加速关键路径）vs 快速跟进（将串行活动改为并行，可能增加返工风险）——两种方法都作用于关键路径
- 计算题模板：给出网络图 → 计算每个活动的ES/EF/LS/LF → 计算TF/FF → 确定关键路径 → 判断延误影响

## 案例分析题

**题目**：

某市政务服务中心启动"智慧政务一体化平台"项目，预算1200万元，工期18个月。项目经理小赵在项目规划阶段进行了如下工作：

（1）采用BSP法对政务服务中心的业务过程进行分析，识别出"行政审批""便民服务""数据共享""效能监察"四个核心业务过程，绘制了U/C矩阵。但在U/C矩阵中，小赵将"审批数据"数据类标记了三个C（创建者），分别是"行政审批"业务过程、"数据共享"业务过程和"效能监察"业务过程。

（2）进度管理方面，项目某阶段包含6个活动，网络关系如下表：

| 活动 | 紧前活动 | 工期(周) |
|------|----------|----------|
| A    | -        | 4        |
| B    | A        | 3        |
| C    | A        | 5        |
| D    | B        | 6        |
| E    | C        | 4        |
| F    | D,E      | 3        |

（3）项目在进行到第8周时，活动D已经完成90%，活动E已经完成80%，活动F尚未开始。与项目计划相比，发现活动E由于需求变更额外增加了2周工期。

**问题1**：分析小赵在U/C矩阵设计中的错误，并给出纠正建议。（8分）

**问题2**：根据网络关系表，画出单代号网络图，计算各活动的ES/EF/LS/LF，确定关键路径和总工期。（10分）

**问题3**：计算活动D和活动E的总浮动时间和自由浮动时间。活动E增加2周工期后，会对项目产生什么影响？（7分）

**参考答案**：

**问题1**：
U/C矩阵的核心规则：每个数据类只能有一个创建者（C），但可以有多个使用者（U）。小赵的错误在于"审批数据"数据类被三个业务过程同时标记为C，违反了数据唯一创建者原则。

纠正建议：
- 分析三个业务过程的实际职责：行政审批过程负责产生审批数据，应为唯一的C
- 数据共享过程读取审批数据进行共享分发，应标记为U
- 效能监察过程读取审批数据进行效能分析，应标记为U
- 最终U/C矩阵：审批数据 → C(行政审批)、U(数据共享)、U(效能监察)

**问题2**：
单代号网络图：

```
A(4) → B(3) → D(6) → F(3)
  ↘ C(5) → E(4) ↗
```

正推计算ES/EF：
- A: ES=0, EF=4
- B: ES=4, EF=7
- C: ES=4, EF=9
- D: ES=7, EF=13
- E: ES=9, EF=13
- F: ES=13, EF=16

逆推计算LS/LF（总工期=16）：
- F: LS=13, LF=16
- D: LS=7, LF=13
- E: LS=9, LF=13
- B: LS=4, LF=7
- C: LS=4, LF=9
- A: LS=0, LF=4

关键路径：A→B→D→F 和 A→C→E→F，两条路径工期均为16周。

**问题3**：
活动D：TF = LS-ES = 7-7 = 0, FF = min(紧后F的ES) - EF = 13-13 = 0
活动E：TF = LS-ES = 9-9 = 0, FF = min(紧后F的ES) - EF = 13-13 = 0

活动E原工期4周，增加2周后变为6周。E的EF从13变为15，导致F的ES变为max(13, 15) = 15，F的EF变为18，项目总工期从16周延长至18周，延误2周。因此活动E的延误直接导致总工期延误，需要采取赶工或快速跟进等措施进行纠正。

**踩分要点**：
- 问题1：明确指出"每个数据类只能有一个C"（3分），正确分析三个过程的职责分配（3分），给出具体纠正方案（2分）
- 问题2：网络图正确（2分），ES/EF/LS/LF计算每项0.5分（共3分），关键路径识别正确（2分），总工期计算正确（1分），需注意两条关键路径都要写出（2分）
- 问题3：TF/FF计算正确（各1分，共4分），延误影响分析正确"总工期增加2周"（2分），提到纠正措施如赶工/快速跟进（1分）

## 记忆口诀

**诺兰六阶段口诀**：
初装蔓延控不住，集成数据才成熟。
前三技术后三数，不可跳越步步走。

**BSP规划法口诀**：
目标过程数据类，U/C矩阵划边界。
每个数据一创建，多用少创才规范。

**CSF与SST口诀**：
CSF找关键因素，高层关注定指标；
SST战略转系统，组织使命变系统。

**构件复用口诀**：
构件独立有接口，代码设计分析三复用。
COM/EJB/CORBA，组装复用降成本。

**浮动时间口诀**：
总浮不误总工期，自由不误紧后始。
关键路径两个零，延误超TF要赶工。
