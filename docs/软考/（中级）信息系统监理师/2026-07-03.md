# 信息系统监理师 2026-07-03

## 今日核心概念

### 1. 信息系统工程监理的信息管理与知识沉淀

定义：监理方在项目全过程中对信息进行收集、传递、加工、存储、检索、利用与归档的系统性管理活动，并通过知识沉淀形成组织过程资产，是监理工作"可追溯、可复用、可审计"的基础设施。

关键要点：
- **信息管理四原则**：及时性（24小时内形成记录）、准确性（与现场实际一致）、完整性（要素齐全签章完整）、可追溯性（事项-时间-责任-依据四要素闭环）
- **监理信息分类**：项目基础信息（合同/章程/批复）、过程信息（日志/纪要/通知单）、技术信息（方案/图纸/测试报告）、管理信息（计划/制度/人员）、外部信息（政策/标准/气象），每类信息须明确来源、责任人、存储路径
- **知识沉淀三层级**：个人级（监理工程师工作笔记/案例复盘）→项目级（项目监理总结/典型问题清单）→组织级（监理单位知识库/最佳实践库），逐级上升避免知识流失
- **信息系统辅助**：采用监理项目管理软件（PMIS）实现文档电子化、流程化、可检索，关键文档（合同/通知单/评估报告）须同时保留纸质原件
- **保密与脱敏**：涉及国家秘密、商业秘密、个人信息的信息须分级管理（绝密/机密/秘密/内部/公开），传递采用加密通道，存储采用加密介质，销毁采用不可恢复方式

### 2. 信息系统工程验收与移交监理

定义：信息系统工程项目竣工验收是项目建设全过程的最终质量把关环节，监理方对验收方案、验收测试、问题整改、资产移交、文档归档进行全过程监督与确认的活动，是"建设"向"使用"责任转移的法定节点。

关键要点：
- **三阶段验收流程**：分项验收（子系统/单站/单设备）→子系统验收（功能模块/集成系统）→项目竣工验收（整体系统），前一阶段未通过不得进入下一阶段
- **竣工验收八项条件**：完成设计全部内容→通过系统测试（含性能/安全/压力/容灾）→完成试运行（通常3-6个月）→完成等保测评与密评→完成第三方测评→技术资料齐全→培训完成→运维体系建立
- **验收测试监理要点**：审核验收测试方案（含用例/环境/数据/通过标准）→见证关键测试用例执行→审查测试报告与缺陷闭环清单→核对合同技术规格逐项确认
- **资产与文档移交**：硬件设备（开箱清单/合格证/序列号）、软件系统（许可证/介质/源码/部署包）、技术文档（设计/测试/运维/用户手册）、管理文档（合同/变更/培训记录）四大类须分类造册
- **缺陷责任期**：自竣工验收之日起计算，通常为12个月，承建单位在缺陷责任期内对系统质量问题承担修复责任，监理负责跟踪闭环

### 3. 信息系统工程监理的工程造价全过程控制

定义：监理方依据合同约定和工程量清单，运用科学的方法对信息系统工程项目从投资决策、设计、招标、施工到竣工的造价进行全过程监督、控制与纠偏，确保项目实际投资不超过批准概算。

关键要点：
- **造价控制四个阶段**：投资估算（立项决策，可行性研究阶段）→设计概算（初步设计限额）→施工图预算（施工准备）→竣工决算（实际发生），前者逐级控制后者，概算超估算10%须重新报批
- **监理造价控制三角色**：设计阶段审核概算是否突破估算（价值工程/限额设计）→招标阶段审核工程量清单准确性→施工阶段审核变更与签证的造价影响
- **工程变更造价影响审核**：工程量变化→综合单价是否执行合同→套用定额是否准确→取费标准是否符合约定→材料设备价差是否符合合同调差条款
- **四类费用结算审核**：工程款（按合同节点支付/累计不超过合同价90%）→变更款（凭变更签证/补充协议）→索赔款（凭书面证据/责任认定）→质保金（预留3%-5%/缺陷责任期满无质量问题退还）
- **造价偏差纠偏措施**：发现偏差→分析原因（设计/施工/市场/管理）→签发造价控制通知单→要求承建单位提交纠偏方案→跟踪落实→闭环报告

### 4. 信息系统工程监理的伦理与职业道德

定义：监理工程师在执业过程中应当遵循的行为规范、价值取向和职业操守，是监理工作独立性、公正性、科学性的根本保障，也是监理行业健康发展的制度基础。

关键要点：
- **监理职业道德六准则**：独立公正（不偏袒任何一方）→守法诚信（不伪造/不隐瞒/不欺骗）→科学严谨（依据标准/数据/事实判断）→廉洁自律（不收受贿赂/不谋取私利）→保守秘密（项目信息/商业秘密/个人隐私）→持续学习（更新知识/提升能力）
- **"五不准"行为准则**：不准在施工单位兼职→不准向建设单位推荐设备/材料供应商→不准收受承建单位的礼金礼品→不准泄露项目技术和商务秘密→准许同时承担两个以上项目监理但须确保资源投入
- **利益冲突识别与回避**：与承建单位存在关联关系（亲属/经济利益/前雇主）时应主动申报并回避，监理单位须建立利益冲突登记与回避制度
- **违规处罚体系**：监理工程师违规→警告/通报/暂停执业/吊销证书→构成犯罪的依法追究刑事责任；监理单位违规→罚款/降低资质/吊销资质证书
- **独立性保障机制**：监理费用由建设单位支付但应在合同中约定独立条款→监理报告直送建设单位不经过承建单位→监理例会三方独立议事规则→监理档案独立归档

### 5. 信息系统工程监理的新技术应用与挑战

定义：随着云计算、大数据、人工智能、区块链、物联网等新技术在政务信息化项目中的广泛应用，监理工作面临新技术认知更新、监理方法创新、质量评价标准重构等多维挑战，监理人员须主动适应并形成新方法论。

关键要点：
- **云平台监理要点**：IaaS/PaaS/SaaS分层架构审查→云服务等级协议（SLA）审核（可用性≥99.9%/响应时间/数据持久性）→多租户隔离与数据安全→云迁移方案与回退机制→云资源使用效率与成本监控
- **大数据平台监理要点**：Hadoop/Spark/Flink等技术选型合理性→数据分层架构（ODS/DWD/DWS/ADS）→数据治理与元数据管理→实时/离线处理能力→数据质量监控与告警
- **AI应用监理要点**：算法公平性（避免偏见）→模型可解释性（黑箱风险）→数据合规性（训练数据来源/授权）→伦理审查（涉及人脸识别/决策辅助）→模型持续评估与版本管理
- **区块链应用监理要点**：共识机制选型（PoW/PoB/DPoS）→智能合约安全性（漏洞审计）→链上数据隐私保护→节点部署与运维→与现有系统集成
- **监理能力升级路径**：技术培训（云/大/物/智/链基础）→案例库建设（典型项目复盘）→专家咨询（外聘技术专家）→工具研发（自动化测试/智能审查）→标准制定（参与行业新标准编制）

## 案例分析题

**题目**：

某省"政务服务一网通办"升级改造项目，总投资1.2亿元，建设单位为省政务服务办公室，承建单位为F科技公司，监理单位为G公司。项目采用"云原生"架构（基于政务云IaaS层自建PaaS，部署12个业务子系统和1个大数据分析平台），合同工期24个月。在项目实施过程中，发生以下四个事件：

**事件一（信息管理与文档）**：项目进行到第10个月时，省审计厅开展专项审计，发现监理单位G公司的监理档案存在以下问题：（1）监理月报与现场实际进度存在明显偏差（月报显示"进度正常"但实际已滞后2个月）；（2）多份监理通知单无签收记录（仅监理方单方签字，承建单位未签收）；（3）涉及项目核心算法的技术方案评审记录缺失。审计厅要求监理单位G公司书面说明情况并限期整改。

**事件二（竣工验收准备）**：项目进入竣工验收准备阶段，承建单位F公司提交《竣工验收方案》，计划在2026年12月组织竣工验收。监理单位G公司审查发现：（1）试运行时间仅2个月（合同约定不少于6个月）；（2）等保三级测评报告未取得（测评机构刚进场）；（3）缺陷责任期条款未在合同中明确约定；（4）大数据分析平台尚未完成数据迁移（约80%数据待迁）。监理单位应如何处理？

**事件三（造价控制与变更）**：项目实施过程中，发生以下造价相关事项：（1）建设单位提出新增"营商环境评估"功能，承建单位报价增加320万元（含开发费180万+云资源租赁费140万）；（2）因政务云IaaS层服务费涨价，承建单位主张追加云服务费85万元；（3）原合同中"数据治理服务费200万元"未明确具体服务范围，承建单位提交结算时主张按200万元全额结算。监理单位G公司应如何审核这三项费用主张？

**事件四（新技术监理与伦理）**：项目中的"智能审批"子系统采用AI算法自动审批行政许可事项（涉及企业开办、资质申请等）。系统上线前，监理审查发现：（1）AI模型的训练数据中某类企业（个体工商户）样本量不足5%，可能导致审批结果偏见；（2）模型决策结果无法解释（典型黑箱模型）；（3）涉及个人信息的训练数据未做匿名化处理。监理单位G公司应如何处理这些新挑战？

**问题**：
1. 针对事件一，监理单位G公司在信息管理方面存在哪些问题？审计厅要求的整改应从哪些方面展开？监理单位和监理工程师应分别承担什么责任？
2. 针对事件二，监理单位应如何处置这四项竣工验收条件不具备的问题？缺陷责任期未约定对监理工作有什么影响？
3. 针对事件三，三项费用主张是否成立？监理审核工程变更与结算应遵循哪些原则？
4. 针对事件四，从新技术应用与伦理的角度，监理应如何处理AI算法的三个问题？这反映了监理工作哪些新的能力要求？

**参考答案**：

**问题1参考答案**：

信息管理问题：
- 监理月报与实际不符，违反"真实性"原则，监理工作未如实反映现场情况，存在"报喜不报忧"问题。
- 监理通知单无承建单位签收记录，违反"完整性"和"可追溯性"原则，监理指令无法证明已送达承建单位，监理工作法律效力存疑。
- 技术方案评审记录缺失，违反监理文档管理要求，关键技术决策无书面记录，一旦发生争议无法追溯。

整改方向：
- 真实补编：按现场实际进度重新编制第8-10个月监理月报，注明"补编"并由总监理工程师签字确认。
- 通知单补签：核实已发出通知单的实际内容，联系承建单位补签，必要时由建设单位出具确认函。
- 评审记录补全：组织对核心算法技术方案的补充评审，形成评审记录归档。
- 责任追究：对相关监理工程师进行批评教育，情节严重的依据内部制度处理。
- 制度补强：建立"监理月报三级审核制"（监理工程师编制→专业监理工程师审核→总监理工程师签发），建立"通知单签收回执制"。

责任划分：
- 监理单位G公司：负管理责任，未建立有效的文档审核和监督机制，须向审计厅提交整改报告并接受相应行政处罚（如通报/罚款）。
- 总监理工程师：负直接领导责任，对监理文档的真实性、完整性负总责，应被约谈并可能面临暂停执业。
- 责任监理工程师：负直接责任，月报编制人或通知单签发人应被批评教育或通报。

**问题2参考答案**：

四项验收条件问题的处置：

①试运行时间不足：监理应书面通知F公司，竣工验收必须在试运行满6个月后方可组织，目前不具备验收条件。要求F公司继续试运行并完善试运行记录（含问题清单/整改记录/运行报告）。在试运行期满前监理不签发《竣工验收监理评估报告》。

②等保测评未取得：等保三级测评是政务系统上线和验收的强制性前置条件，未取得测评报告不得组织竣工验收。监理应要求F公司协调测评机构加快进度，监理跟踪测评过程。若测评中发现重大安全问题，须整改闭环后重新测评。

③缺陷责任期未约定：监理应建议建设单位与承建单位签订补充协议，明确缺陷责任期（建议12个月）、质保金预留比例（3%-5%）、缺陷责任期内的责任划分。在补充协议签订前，监理不签发竣工验收报告。

④数据迁移未完成：80%数据未迁移意味着大数据分析平台不具备验收条件。监理应要求F公司制定数据迁移专项计划（含迁移方案/回滚方案/数据校验/完整性与一致性验证），监理审核签认后实施。迁移完成并经监理验证前，不得组织平台验收。

缺陷责任期未约定的影响：
- 监理无法可依地跟踪承建单位在系统交付后的质量责任
- 系统质量问题在交付后出现时易引发争议
- 质保金预留与退还缺乏依据
- 监理在缺陷责任期内的跟踪监督无合同依据
- 易形成"建用脱节"——建设单位承担了应由承建单位承担的质量责任

**问题3参考答案**：

费用主张逐项审核：

①新增"营商环境评估"功能320万元：
- **功能新增部分（180万）**：成立。属于合同外新增需求，应按变更处理。要求F公司提供建设单位书面变更申请→监理组织影响分析（工期/费用/技术）→三方协商→CCB审批→签补充协议。变更审批后180万元方可纳入结算。
- **云资源租赁费（140万）**：需具体审核。如属于新增功能必需的云资源且有详细使用计划，成立；如仅笼统报价且无具体使用量数据，监理应要求F公司拆分报价（计算资源/存储资源/网络资源单价×数量×使用时长），按云平台SLA实际使用量结算。

②云服务费涨价85万元：
- **不成立（原则上）**。固定总价合同下，云服务费属于承建单位投标时即应充分预估的风险，不应因市场价格波动而调整。除非合同明确约定"云服务费按实际使用量结算"或"价格上涨超过5%时按比例调整"，否则该85万元不予追加。
- 监理应核查合同条款和投标报价清单，若F公司在投标时已充分预见云服务费风险，该主张不予支持。

③数据治理服务费200万元：
- **部分成立**。合同未明确具体服务范围属合同缺陷。监理应组织双方协商确定服务范围，按"实际完成工作量+有效交付物"结算。
- 建议处理：要求F公司提供数据治理服务交付物清单（数据标准/数据质量报告/元数据管理文档/数据安全方案等）→监理依据合同原则和交付物价值协商结算金额→签订补充协议明确数据治理服务范围和结算依据。

造价审核基本原则：
- 合同范围内工程量不调（固定总价合同基本属性）
- 工程变更须有书面签证（建设单位/承建单位/监理三方签字）
- 综合单价按合同约定执行
- 套用定额/取费标准须符合合同约定的计价规范
- 材料设备价差按合同调差条款执行
- 监理审核须有充分依据（变更文件/签证单/工程量清单/材料核价单）
- 总监负责制：审核结论须经总监理工程师签字并加盖监理单位公章

**问题4参考答案**：

AI算法三个问题的处理：

①训练数据样本不均衡：
- 监理应要求F公司补充样本采集（特别是少数类企业样本），重新训练模型，并通过公平性评估（如不同群体的通过率差异不超过5%）。
- 在未解决公平性问题前，监理不签认"智能审批"子系统上线。
- 同时建议建设单位考虑人工审核兜底机制，对AI低置信度结果转人工审批。

②模型黑箱不可解释：
- 监理应要求F公司采用可解释AI技术（如LIME/SHAP/注意力机制）或规则化重构模型，使关键决策可解释。
- 政务AI应用涉及公民权利，必须可解释、可追溯、可申诉。不可解释的模型不能用于审批决策。
- 要求F公司配套建设"AI审批结果说明"和"人工申诉/复核"机制。

③个人信息未匿名化：
- 严重违反《个人信息保护法》。监理应要求F公司立即停止使用未脱敏数据进行训练，对训练数据实施匿名化/去标识化处理。
- 同时审查数据来源合法性，是否取得个人信息主体的授权同意。
- 必要时监理应建议建设单位启动个人信息影响评估（PIPIA）。

新技术监理能力要求：
- **技术认知能力**：监理人员须持续学习云/大/物/智/链等新技术的基本原理、技术架构和典型风险。
- **新方法构建能力**：传统监理方法（旁站/巡视/平行检验）须与新技术特点结合，如AI监理需关注数据质量/算法公平/伦理审查。
- **跨学科协同能力**：监理须与建设单位、行业专家、法律顾问建立协同机制，共同应对技术、伦理、法律复合问题。
- **伦理审查能力**：监理须建立"技术-伦理-法律"三维审查框架，特别是涉及公民权益的AI应用。
- **持续学习能力**：监理人员每年须完成新技术培训学时（建议不少于40学时），监理单位须建立新技术知识库。
- **工具研发能力**：监理单位应研发或引入自动化测试工具（如AI模型公平性测试工具、算法审计工具）提升监理效能。

**踩分要点**：
- 问题1：三大信息管理问题（真实性/完整性/缺失）各1分，补编/补签/补评整改各1分，总监+工程师+单位三级责任划分2分
- 问题2：四项验收条件处置各1.5分（共6分），缺陷责任期影响须答出"无据可循/质保金依据/责任划分争议"等3个以上要点得3分
- 问题3：三项费用判定各1.5分（共4.5分），各项理由论述2分（共6分），审核原则3.5分
- 问题4：AI三个问题处置各2分（共6分），新技术能力要求答出"技术认知/新方法/伦理审查/持续学习"得4分

## 记忆口诀

**信息管理四原则**：及时准确完整可追溯，24小时形成记录

**监理档案三级审**：工程师编、专监审、总监理签，三级把关不失真

**通知单签收回执制**：单方签字不算送达，承建签收才生效

**竣工验收八条件**：设计完工+测试通过+试运行+等保密评+第三方+资料全+培训完+运维立

**三阶段验收序**：分项验、子系统验、整体竣工验，前不过后不签

**造价控制四阶段**：估算→概算→预算→决算，逐级控制不突破，超概10%须重批

**四类费用结算**：工程款（90%封顶）、变更款（凭签证）、索赔款（凭证据）、质保金（3-5%）

**监理伦理六准则**：独立公正、守法诚信、科学严谨、廉洁自律、保守秘密、持续学习

**监理五不准**：不兼职、不推荐、不收礼、不泄密、不投入不足（同时承担多项目须保证资源）

**新技术监理五力**：技术认知力、新方法构建力、跨学科协同力、伦理审查力、持续学习力

**AI监理三必须**：数据公平必保、模型可解释必达、个人信息必脱敏

**缺陷责任期意义**：12个月质保期，遗留问题整改期，质量责任追溯期
