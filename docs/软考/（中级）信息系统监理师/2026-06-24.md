# （中级）信息系统监理师 — 2026-06-24 学习内容

## 今日核心概念

### 1. 监理会议制度
定义：监理会议是监理单位在项目实施过程中行使监理职责、协调各方关系、解决工程问题的重要组织形式。按《建设工程监理规范》和信息系统工程监理相关标准，监理会议主要包括第一次工地会议、监理例会和专题会议三类。

关键要点：
- 第一次工地会议：开工前由建设单位主持，监理单位、承建单位参加，明确监理程序、相互通报组织机构与人员分工，监理单位就监理规划主要内容作交底说明
- 监理例会（工地例会）：由总监理工程师主持，定期召开，按周/双周/月频次，会议纪要由监理单位整理、各方会签，是监理过程控制的核心制度
- 专题会议：针对质量、进度、投资、变更、安全等专项问题，由总监理工程师或专业监理工程师主持，根据需要随时召开
- 会议纪要的法律效力：经各方签字确认的会议纪要构成施工合同的补充文件，具有合同约束力

### 2. 信息系统工程初步验收与试运行监理
定义：初步验收是信息系统工程竣工后、正式验收前的重要环节，由建设单位组织，监理单位监督实施，旨在验证系统是否满足合同约定的功能、性能要求。试运行阶段是初步验收后系统在实际业务环境中运行的检验期。

关键要点：
- 监理在初步验收阶段的职责：审核承建单位提交的竣工文档完整性、审查初验方案、监督功能与性能测试过程、审查初验报告
- 试运行条件：承建单位完成所有合同内开发任务并通过内部测试、竣工文档齐备、运行环境就绪、运维人员培训完成
- 试运行期限：通常为1-3个月，合同另有约定除外
- 试运行监理重点：系统稳定性监控、功能符合性验证、数据正确性校验、用户反馈收集与处理跟踪
- 试运行结束标志：试运行期满且系统运行稳定，缺陷修复完毕，形成试运行总结报告

### 3. BIM与数字化技术在监理中的应用
定义：BIM（建筑信息模型）技术在监理工作中，将传统监理手段与数字化信息管理相结合，通过三维可视化模型对信息系统工程的基础设施部分进行进度、质量、投资的综合管控。与广义的数字化监理平台共同构成监理信息化管理工具。

关键要点：
- BIM在监理中的五大应用：碰撞检查（管线综合）、施工进度模拟（4D）、工程量自动统计（5D）、质量安全巡检定位、竣工模型交付
- 数字化监理平台功能模块：文档管理、进度跟踪、质量巡检、变更审批流程、投资控制台账、监理日志电子化
- 监理单位BIM能力要求：配备BIM协调员、制定BIM实施方案、明确BIM交付标准
- 数字化监理与传统监理融合要点：线下旁站不可替代、线上数据辅助分析、电子签章合规管理、数据安全与隐私保护

### 4. 监理检测工具与设备管理
定义：监理单位在信息系统工程实施过程中，依据合同和质量标准，使用专业检测仪器对设备、材料、实体质量进行检验检测，并建立检测设备台账和管理制度。

关键要点：
- 常见监理检测工具：网络测试仪（链路通断/衰减/串扰）、光纤测试仪（OTDR光时域反射仪）、示波器/频谱分析仪、万用表/接地电阻测试仪、温湿度计/风速仪（机房环境）
- 监理检测设备管理要求：建立设备台账（编号/校准周期/检定证书/保管人）、定期送检（国家法定计量检定机构）、严禁使用超期未检设备
- 平行检验时监理自备设备的独立性：不得借用承建单位设备，确保检测结果公正
- 见证取样送检：对关键材料（如网线/光纤/桥架/接地材料），监理见证取样后在建设单位或监理人员监督下送有资质的实验室检测

### 5. 监理表格体系与签发流程
定义：监理表格是监理工作标准化的书面载体，按《建设工程监理规范》分为A类（施工单位报审/报验用）、B类（监理单位用）、C类（通用）三大类。信息系统工程监理可参照使用，并结合信息化特点补充专用表格。

关键要点：
- A类表（承建单位→监理）：施工组织设计/方案报审表、工程开工/复工报审表、分包单位资格报审表、工程材料/设备/构配件报审表、工程报验申请表、工程款支付申请表、工程临时/最终延期申请表
- B类表（监理→承建单位）：监理通知单、工程暂停令、工程款支付证书、监理报告、监理工作联系单
- C类表（通用）：会议纪要、往来函件
- 签发流程要点：签认主体正确（总监/专监权限分清）、时限明确（一般3-7日回复）、编号连续、归档完整
- 信息系统工程专用表格：需求规格说明书审查表、设计说明书审查表、测试计划/报告审查表、系统验收审查表

## 案例分析题

**题目**：

某省"智慧医保一体化平台"建设项目，总投资8500万元，建设内容包括医保业务经办系统、基金监管系统、数据共享交换平台和基础网络与机房改造四部分。甲监理公司中标承担全过程监理。

事件一：开工前，承建单位提交施工组织设计方案，专业监理工程师审查后认为方案中"基础网络布线从中心机房到各楼层配线间全程走吊顶桥架"的路径设计存在消防隐患，在方案审查表上签署"修改后重新报审"。承建单位项目经理表示口头已与建设单位沟通同意，催促监理放行开工。

事件二：施工过程中，专业监理工程师发现综合布线采用的六类网线外观与报验样品颜色不一致，要求暂停使用。经查，该批网线来自承建单位另一工地调拨，报验材料未重新申报。

事件三：软件系统试运行第二周，核心业务处理响应时间超过合同约定的3秒，高峰时段达到12秒。承建单位称系数据库未做分区优化，承诺正式验收前解决。建设单位要求监理对此出具处理意见。

事件四：竣工结算阶段，承建单位提交投资额中"监理取费基数"按"合同总额+变更增加额"计算，而建设单位认为应按"合同总额"计算。合同监理取费条款为"按项目实际结算建设工程费的2.5%计取"。

**问题**：
1. 事件一中，监理工程师的做法是否正确？承建单位口头沟通能否替代方案变更审批？请说明依据。
2. 事件二中，监理工程师暂停使用的依据是什么？调拨材料重新进场的程序是什么？
3. 事件三中，作为总监理工程师，应如何出具处理意见？试运行阶段监理的关键控制点是什么？
4. 事件四中，监理取费基数应如何确定？请给出判断并说明理由。

**参考答案**：

1. 监理工程师做法正确。依据：
   - 施工组织设计方案必须经书面报审、监理审查同意后方可实施，口头沟通不具备法律效力
   - 承建单位与建设单位的"口头沟通"违反合同管理程序和监理规范，监理有权也应当坚持书面审批制度
   - 如确需变更施工方案，承建单位应履行变更程序：提交变更申请→监理审核技术方案与影响评估→建设单位审批→下达变更指令
   - "修改后重新报审"是监理规范规定的正常审查结论，并非否定方案整体，承建单位应按要求修改后正式重新报审

2. 监理工程师暂停使用的依据：
   - 材料/设备进场报验制度：所有进场材料必须经监理审查并签认后方可使用
   - 调拨材料属于"重新进场"行为，原报验审批已失效，必须重新报验
   - 外观与封样不一致已构成质量疑虑，监理有权要求暂停使用并重新检验
   - 重新进场程序：承建单位提交材料报审表→附合格证/检测报告/出厂检验文件→监理现场核验外观规格→必要时见证取样送检→审查合格签认后方可使用

3. 总监处理意见：
   - 要求承建单位7日内提交性能问题分析报告和整改方案，明确根因（数据库分区优化计划/代码优化方案/硬件扩容方案）、整改时限和验证方法
   - 试运行期间性能不达标属承建单位违约，不得以"承诺后续解决"回避当前违约责任
   - 试运行期可根据整改情况酌情顺延，但需建设单位书面确认
   - 如试运行期满性能仍未达标，不得提请正式验收
   - 试运行阶段监理关键控制点：系统稳定性（无重大故障/无数据丢失）、功能完整性（合同约定功能全部执行通过）、性能符合性（响应时间/并发数/吞吐量达标）、数据正确性（业务数据与源系统一致）、用户满意度（操作便利性/培训到位）

4. 监理取费基数判定：
   - 合同约定"按项目实际结算建设工程费的2.5%计取"，则取费基数应为实际结算价
   - "实际结算建设工程费"=合同约定工程费+经审批的变更增加/减少费用
   - 变更增加额若经法定程序审批（监理审核+建设单位确认），应纳入取费基数
   - 未经审批的自发变更/索赔性增加不计入取费基数
   - 最终监理费=(合同工程费+审批变更净增加额)×2.5%

**踩分要点**：
- 事件一：必须点明"口头沟通无效""书面审批不可替代""修改后重新报审合规"
- 事件二：必须点明"调拨=重新进场""原报验失效""重新报验程序完整"
- 事件三：必须有"不得以承诺替代当前违约""试运行不达标不得提请正式验收""试运行关键控制点至少列三项"
- 事件四：必须分析"实际结算"≠"合同总额"，区分"审批变更"与"自发变更"的计费差异

## 记忆口诀

1. **监理会议三类分**：一工（第一次工地会议）建方主，例会总监定周期，专题随叫随到人，纪要签了有法依。

2. **初验试运四步走**：文档全→功能测→试运行→总结过。试运期看四件事：稳定、准确、够快、用户乐。

3. **BIM监理五大用**：碰（碰撞检查）→模拟（4D进度）→统计（5D工程量）→巡检（质量安全）→归档（竣工模型）。

4. **检测设备四必须**：必须建台账、必须定期检、必须独立用（不借承建）、必须在有效期内。

5. **表格三大家**：A类承建报我审，B类我发指令文，C类通用大家填。不全问题"修改后再报"，不批问题"不得进行下道工序"。

6. **调拨材料两不行**：原报验不行、先用在补不行。重新报验四步齐：报审→附证→核验→签认。
