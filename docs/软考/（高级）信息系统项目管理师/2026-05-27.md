## 今日核心概念

### 1. 项目组合管理
定义：对项目组合、项目集和项目进行集中管理，以实现战略目标的过程。项目组合是战略投资的集合，通过优先级排序确保资源投向与组织战略一致。
关键要点：
- 项目组合管理的核心是**战略对齐**，关注"做正确的事"
- 项目组合经理负责在组织层面统筹资源分配
- 项目组合中的项目/项目集不一定相互依赖，但共享有限资源
- 组合管理的生命周期比单个项目更长，是持续性的管理活动
- 每个项目应通过**业务论证**证明其战略价值

### 2. 项目集管理
定义：对一组相互关联且被协调管理的项目进行管理，以获得分别管理无法获得的收益。项目集关注项目间的**依赖关系**和**协同效应**。
关键要点：
- 项目集管理的核心是**收益实现**，关注项目的协同价值
- 项目集经理负责协调各子项目间的依赖和接口
- 项目集组件可以包含项目、子项目集和项目集级别的活动
- 项目集收益管理包含收益识别、分析、规划、实现和过渡五个阶段
- 当各组件间存在强依赖时，应升级为项目集管理

### 3. 项目管理办公室（PMO）
定义：组织内对项目、项目集和项目组合进行集中管理的部门。PMO不是项目管理者的替代者，而是提供支持、指导和标准化的组织结构。
关键要点：
- PMO的三种类型：**支持型**（低控制度，仅提供模板和经验库）、**控制型**（中度控制，要求合规和采用特定方法论）、**指令型**（高控制度，直接管理和控制项目）
- PMO的核心职责包括：制定标准流程和方法论、培训项目经理、管理多项目资源冲突、维护组织过程资产
- PMO是项目经理的上级管理者，但不一定是项目发起人
- 高级信息系统项目管理师考试中，PMO类型区分是高频考点

### 4. 组织过程资产与事业环境因素
定义：组织过程资产是组织内可用于影响项目成功的资产，包括过程资产库、模板、历史信息和经验教训。事业环境因素是项目团队无法控制的、将对项目产生影响的各种条件。
关键要点：
- 组织过程资产分两类：**过程、政策和程序**（如模板、指南）和**组织知识库**（如历史数据、经验教训）
- 事业环境因素包括：组织文化和结构、政府或行业标准、市场条件、基础设施等
- **关键区分**：能修改的是组织过程资产，不能修改的是事业环境因素
- 两者都是制定项目管理计划和大多数过程的输入
- 组织过程资产是项目收尾时需要更新的关键输出

### 5. 项目章程与项目工作说明书（SOW）
定义：项目章程是由项目发起人签发的正式批准文件，正式确立项目并授权项目经理使用组织资源。SOW是对项目范围、交付物和工作要求的书面描述。
关键要点：
- 项目章程的签发标志着项目**正式立项**，在项目启动阶段编制
- 项目章程包含：项目目的、可衡量的目标、高层级需求、总体里程碑、总体预算、关键干系人清单、项目经理及其权限
- SOW由项目发起人或客户提供，是项目章程的重要输入
- SOW与项目范围说明书的区别：SOW是外部输入（客户需求），范围说明书是项目管理计划的一部分
- 项目经理有权修改项目章程的内容，但修改后需重新审批

## 案例分析题

**题目**：

某市政府"智慧交通"信息化建设项目，总投资预算800万元，工期12个月。项目发起人为交通局副局长。项目启动阶段，仅由副局长口头指定了项目经理张某，未编制项目章程。张某编制了一份简单的项目计划后即进入执行阶段。

项目执行到第6个月时发生以下问题：
（1）因交警部门需求变更，新增了"电子围栏"模块，张某直接安排开发人员实施，未走变更流程。
（2）因开发组与交警部门沟通不畅，交警部门对中间交付物不满意，提出投诉。
（3）项目经理张某发现项目进度已落后约3周，但未向高层报告。

**问题1**：请指出项目启动阶段存在的主要问题（不少于3点）。（5分）

**问题2**：项目执行阶段存在哪些管理问题？应如何改进？（10分）

**问题3**：假设项目总预算PV=800万元，第6个月末检查时：EV=340万元，AC=380万元。请计算CV、SV、CPI、SPI，并判断项目的成本和进度绩效。（5分）

**参考答案**：

**问题1（5分）**：
1. **未正式编制项目章程**：项目启动必须由发起人签发正式的项目章程，仅有口头指定不具备合法性（2分）
2. **项目经理任命不规范**：应通过项目章程正式任命项目经理，并明确其权责，而非口头指定（1.5分）
3. **缺少正式的启动会**：应召开项目启动会，向关键干系人宣贯项目目标、范围和计划，确保各方理解一致（1.5分）

**问题2（10分）**：

存在的问题及改进措施：
1. **变更管理缺失**：张某直接安排新增模块开发，未走整体变更控制流程
   - 改进：所有变更须提交变更请求，经变更控制委员会（CCB）评估审批后实施（2.5分）

2. **干系人沟通管理不到位**：开发组与交警部门沟通不畅，交付物不满足需求
   - 改进：建立有效的沟通管理计划，定期与交警部门召开需求确认会，设置里程碑评审（2.5分）

3. **问题升级和上报机制缺失**：进度落后3周未向高层汇报
   - 改进：建立问题上报机制，当进度偏差超过阈值时，及时向项目发起人和管理层报告，并制定纠正措施（2.5分）

4. **项目管理计划不完善**：仅编制了简单计划，缺少变更管理计划、沟通管理计划、干系人管理计划等子计划
   - 改进：完善项目管理计划的各子计划，并经审批后作为项目执行的基准（2.5分）

**问题3（5分）**：
- CV = EV - AC = 340 - 380 = **-40万元**（成本超支40万）（1分）
- SV = EV - PV = 340 - 800×(6/12) = 340 - 400 = **-60万元**（进度落后60万）（1分）
- CPI = EV / AC = 340 / 380 ≈ **0.89**（每花1元只完成0.89元的工作）（1.5分）
- SPI = EV / PV = 340 / 400 = **0.85**（进度效率仅达85%）（1.5分）
- **判断**：CV<0且CPI<1，项目**成本超支**；SV<0且SPI<1，项目**进度落后**。项目整体绩效不理想，需采取赶工或快速跟进等纠正措施。

**踩分要点**：
- 问题1关键踩分词：**项目章程、正式任命、启动会**——三个启动阶段核心动作缺一不可
- 问题2关键踩分词：**变更控制流程、CCB审批、沟通管理计划、问题上报机制、管理计划完整性**
- 问题3计算务必写公式和单位，CPI/SPI需给出绩效判断结论
- 综合题的"改进措施"必须与"问题"一一对应，不能空泛回答

## 记忆口诀

**项目与项目集/组合区分**
```
组合管战略，项目集管收益，项目管交付
组合看方向，项目集看协同，项目看目标
```

**PMO三类型**
```
支持型——帮你不替你（给模板、给经验）
控制型——逼你照着来（要合规、要标准）
指令型——替你说了算（直接管、直接控）
```

**组织过程资产 vs 事业环境因素**
```
能改的是资产，不能改的是因素
资产归自己管，因素由环境定
资产要更新（收尾时），因素只能适应
```

**项目章程必备八要素**
```
目的目标高层需求，里程碑预算和风险
关键干系人名单，经理名字和权限
```

**挣值四大公式**
```
偏差看减法：CV=EV-AC，SV=EV-PV
指数看除法：CPI=EV/AC，SPI=EV/PV
大于1是好事，小于1要警惕
负数超支又落后，正数节约又提前
```
