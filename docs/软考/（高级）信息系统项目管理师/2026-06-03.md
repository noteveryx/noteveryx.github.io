## 今日核心概念

### 1. 质量管理体系三层次
定义：项目质量管理分为质量规划、质量保证、质量控制三个层次，形成完整的质量管理体系。质量规划确定质量标准与检验方法，质量保证通过过程审计确保过程合规，质量控制通过检测发现缺陷并纠正。

关键要点：
- 质量规划输入：范围基准、干系人登记册、风险登记册、组织过程资产
- 质量保证核心活动：过程审计、质量审计、过程分析
- 质量控制核心工具：因果图（鱼骨图）、控制图、帕累托图、直方图、散点图、核对表、流程图
- 七大工具口诀：「因帕控流核直散」——因果图、帕累托图、控制图、流程图、核对表、直方图、散点图
- 质量成本（COQ）：一致性成本（预防+评估）与非一致性成本（内部失败+外部失败）

### 2. 成本基准与储备管理
定义：成本基准是经批准的按时间段分配的项目预算，不包括管理储备。成本基准 = 各活动估算成本之和 + 应急储备。总预算 = 成本基准 + 管理储备。

关键要点：
- 应急储备：已知-未知风险，由项目经理支配，纳入成本基准
- 管理储备：未知-未知风险，由管理层支配，不纳入成本基准
- 应急储备属于EVM计算范围，管理储备不属于
- 项目经理有权使用应急储备但无权使用管理储备
- 成本基准变更需走整体变更控制流程

### 3. 确认范围与质量控制的关系
定义：确认范围是正式验收已完成可交付成果的过程，质量控制是监督并记录执行结果以评估绩效并推荐必要变更的过程。两者相辅相成但职责不同。

关键要点：
- 确认范围关注可交付成果是否符合需求（验收标准），质量控制关注可交付成果是否正确（质量标准）
- 确认范围由客户或发起人执行，质量控制由项目团队或质量部门执行
- 通常质量控制先于确认范围，但两者可并行
- 确认范围不合格→需要缺陷修复（质量控制范畴）
- 确认范围通过→更新范围基准→可交付成果正式验收

### 4. 项目的组织结构四类型
定义：项目组织结构分为职能型、矩阵型（弱/平衡/强）、项目型、复合型四类，不同结构对项目经理的权力、资源可用性、沟通效率影响差异显著。

关键要点：
- 职能型：PM权力最低，沟通效率低，资源利用率高，适合稳定运营
- 弱矩阵型：PM权力有限，协调员角色，职能经理主导
- 平衡矩阵型：PM与职能经理权力相当，资源共享
- 强矩阵型：PM权力较高，接近项目型
- 项目型：PM权力最高，沟通效率高，资源专属，成本最高

### 5. 项目章程与项目工作说明书（SOW）
定义：项目章程是由项目发起人或发起人代表签发的正式文件，正式批准项目成立并授权项目经理动用组织资源。SOW是外部客户或内部发起人对产品范围的描述。

关键要点：
- 章程核心内容：项目目的、可衡量的成功标准、高层级需求、里程碑、总体预算、PM任命及权责、干系人清单、审批要求
- SOW核心内容：业务需要、产品范围描述、战略对齐
- 章程是项目启动的核心输出，没有章程就没有正式项目
- 商业论证是制定章程的重要输入
- 章程变更不影响项目管理计划，需走单独的变更流程

## 案例分析题

**题目**：

某市政府投资建设「智慧水务综合管理平台」项目，总投资800万元，工期18个月。项目采用强矩阵组织结构，任命张某为项目经理。

项目进入执行阶段6个月后，张经理发现以下问题：
（1）项目累计计划价值（PV）为360万元，实际成本（AC）为400万元，挣值（EV）为300万元。
（2）质量检验发现系统模块间接口缺陷率高达15%，数据库查询响应时间超标严重。
（3）客户提出新增水资源实时监测功能，张经理直接口头答应了客户，安排开发人员加班实施，未经过任何评审流程。
（4）项目团队有4名开发人员因薪资问题提出离职，张经理认为人员可随时补充，未采取任何挽留措施。
（5）项目即将进入验收阶段，张经理计划直接将系统交给甲方运维部门，未准备任何验收文档。

**问题1**：请根据题干给出的挣值数据，计算项目的CV、SV、CPI、SPI，并分析项目当前的绩效状况。

**问题2**：请指出张经理在质量管理、变更管理、人力资源管理、收尾管理方面存在的问题，并给出改进措施。

**参考答案**：

问题1：
- CV = EV - AC = 300 - 400 = -100万元（成本超支100万元）
- SV = EV - PV = 300 - 360 = -60万元（进度落后60万元）
- CPI = EV / AC = 300 / 400 = 0.75（每花费1元只完成0.75元的工作）
- SPI = EV / PV = 300 / 360 = 0.83（进度效率仅达计划的83%）
- 绩效分析：项目成本严重超支，进度明显落后，处于不健康状态，需立即采取纠正措施。

问题2：

质量管理问题：
- 接口缺陷率高、响应时间超标说明质量规划不足，未制定有效的质量标准和检验标准
- 未使用质量七工具进行根因分析，缺乏缺陷追踪机制
- 改进措施：使用因果图分析缺陷根因；建立质量核对表；实施过程审计；加强代码评审和接口测试

变更管理问题：
- 客户提出需求变更后直接口头答应，未经变更控制流程
- 未评估变更影响（范围/进度/成本），未提交变更请求
- 改进措施：执行整体变更控制流程（提出变更请求→CCB评审→批准后更新基准→通知干系人）

人力资源管理问题：
- 对核心人员离职风险不重视，未识别团队士气与留存问题
- 未运用激励理论（马斯洛/赫茨伯格）改善团队积极性
- 改进措施：建立人才梯队，识别关键岗位备份人员；运用适当激励手段；关注团队建设

收尾管理问题：
- 未准备验收文档即交付，缺乏系统验收流程
- 未进行确认范围和质量控制就准备移交
- 改进措施：准备完整的验收文档（需求跟踪矩阵、测试报告、用户手册等）；经正式验收后再移交；做好合同收尾和行政收尾

**踩分要点**：
- CV/SV/CPI/SPI公式正确各1分，计算结果正确各1分（共8分）
- 四个管理领域各指出问题+改进措施（各3分，共12分）
- 关键踩分词：CCB、变更请求、因果图、质量七工具、激励理论、确认范围、验收文档、行政收尾、合同收尾

## 记忆口诀

**质量三层次速记**：
规划定标准→保证审过程→控制抓缺陷
（规划是方向盘，保证是GPS，控制是刹车）

**储备管理口诀**：
应急基准内，项目经理用；
管理基准外，高层来拍板；
已知归应急，未知归管理；
基准含应急，预算含管理

**确认范围vs质量控制口诀**：
质控查对不对，确认查要不要；
质控团队做，确认客户批；
质控先来后确认，双管齐下保质量

**组织结构权力梯度**：
职能（最弱）→弱矩阵→平衡矩阵→强矩阵→项目型（最强）
记忆：职弱平衡强项，权力依次递增

**挣值四大公式口诀**：
偏差永远用减法：CV=EV减AC，SV=EV减PV
指数永远用除法：CPI=EV除AC，SPI=EV除PV
EV永远在前面，记住这一点就不会搞反
负号代表不好，低于一也不妙
