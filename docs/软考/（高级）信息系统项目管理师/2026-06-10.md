## 今日核心概念

### 1. 项目立项管理——财务评价四指标
定义：项目立项阶段通过净现值（NPV）、内部收益率（IRR）、投资回收期和投资回报率（ROI）四个核心财务指标评估项目经济可行性，决定是否立项。

关键要点：
- 净现值NPV = Σ(各年净现金流量 / (1+折现率)^t) - 初始投资。NPV>0项目可行，NPV越大越优
- 内部收益率IRR：使NPV=0时的折现率。IRR>基准收益率时项目可行，IRR越大越优
- 投资回收期 = 初始投资 / 年均净现金流量（静态）；需考虑货币时间价值时为动态回收期
- 投资回报率ROI = 年均利润 / 投资总额 × 100%，ROI越高项目越优
- 多项目比选：NPV与IRR冲突时（互斥项目），以NPV为准，因为NPV假设再投资回报率为折现率更合理

### 2. CMMI成熟度模型与能力等级
定义：能力成熟度模型集成（CMMI）是衡量组织过程能力的框架，包含5个成熟度等级和4个能力等级，用于指导组织过程改进。

关键要点：
- 成熟度五级：①初始级（过程无序）→②已管理级（项目级管理）→③已定义级（组织级标准过程）→④量化管理级（统计过程控制）→⑤优化级（持续改进创新）
- 能力四级：0级不完整→1级已执行→2级已管理→3级已定义
- 过程域四大类：过程管理、项目管理、工程、支持
- CMMI两种表示法：阶段式（成熟度等级）和连续式（能力等级），阶段式适合组织整体评估，连续式适合单过程域改进
- 高成熟度（4-5级）的核心：量化管理用统计技术，优化级关注根本原因分析和缺陷预防

### 3. 新一代信息技术及应用
定义：以人工智能、区块链、物联网、云计算、大数据、5G/6G移动通信为代表的新一代信息技术，是信息系统项目管理的技术基础，论文和案例中高频出现。

关键要点：
- 人工智能三大要素：数据（燃料）+ 算法（引擎）+ 算力（动力）。机器学习分为监督学习（有标签）、无监督学习（无标签）、强化学习（试错反馈）
- 区块链核心特征：去中心化、不可篡改、可追溯、智能合约。共识机制包括PoW（工作量证明）、PoS（权益证明）、PBFT（实用拜占庭容错）
- 物联网三层架构：感知层（传感器/RFID）→网络层（5G/NB-IoT）→应用层（平台/场景）。边缘计算解决云端延迟问题
- 云计算三服务模型（IaaS/PaaS/SaaS）×四部署模式（公有/私有/混合/社区云）。容器技术Docker+K8s实现微服务编排
- 大数据5V特征（Volume/Velocity/Variety/Value/Veracity）+ 处理流程（采集→存储→清洗→分析→可视化）

### 4. 业务流程管理与重组
定义：业务流程管理（BPM）是对业务流程进行建模、分析、改进和自动化的持续管理方法；业务流程重组（BPR）是对业务流程进行根本性的再思考和彻底的再设计，以实现绩效的飞跃。

关键要点：
- BPM生命周期：设计→建模→执行→监控→优化（五个阶段循环迭代）
- BPR核心原则：以流程为中心而非职能为中心、以客户为导向、以信息技术为手段、追求根本性变革而非渐进式改进
- BPR三大变革类型：流程再造（激进）、流程优化（渐进）、流程外包（转移）
- BPM常用工具：流程图/泳道图、价值流图VSM、SIPOC模型（供方→输入→过程→输出→客户）
- 流程管理成熟度模型：从职能驱动→流程认知→流程管理→流程卓越→流程智能的五个阶段

### 5. 六西格玛管理方法
定义：六西格玛（Six Sigma）是一套以数据为驱动、追求近乎完美质量的管理方法，目标是将过程缺陷率控制在3.4ppm（每百万次机会3.4个缺陷）以内。

关键要点：
- DMAIC改进流程：定义（Define）→测量（Measure）→分析（Analyze）→改进（Improve）→控制（Control）
- DMADV设计流程（用于新产品/新过程）：定义→测量→分析→设计→验证
- 组织结构：倡导者（高层支持）→黑带大师（培训指导）→黑带（全职项目负责人）→绿带（兼职参与）→黄带（基础认知）
- 核心工具：SIPOC图（界定范围）、鱼骨图（根因分析）、假设检验（验证因子）、控制图SPC（持续监控）
- 与CMMI的关系：六西格玛侧重过程质量改进，CMMI侧重过程能力成熟度，两者可互补使用

## 案例分析题

**题目**：

某省"智慧应急指挥平台"项目总投资800万元，计划工期12个月。项目团队完成了立项阶段的财务评价，各年预计净现金流量如下：第0年-800万元（投资），第1年200万元，第2年350万元，第3年400万元，第4年300万元。设定折现率为10%。

项目经理张某将项目划分为5个子系统并行开发。在第6个月末（计划完成50%工作量），实际支出450万元，实际完成工作量40%。团队采用瀑布模型开发，需求规格说明书经用户签字确认后进入开发阶段。开发过程中用户多次提出需求变更，张某口头答应后让开发人员直接修改代码。同时，采购负责人李某在选择视频会议子系统供应商时，直接指定了其朋友所在公司，未进行公开招标。

第8个月，测试发现多个子系统间接口联调失败，追溯到各单位对接口规范理解不一致。此时项目总支出已达680万元，实际完成工作量55%。第三方审计发现采购合同价格高于市场均价30%，且未按合同约定对供应商延期交付进行违约索赔。

**问题**：
（1）计算项目的净现值（NPV），判断项目是否可行。（5分）
（2）分析第6个月末的挣值情况（PV、EV、AC、CV、SV、CPI、SPI），评价项目绩效。（6分）
（3）分析第8个月末的挣值情况，假设剩余工作仍按当前效率完成，计算EAC和TCPI（基于BAC）。（6分）
（4）指出该项目在变更管理、采购管理和质量管理方面存在哪些问题，并提出改进建议。（8分）

**参考答案**：

（1）NPV计算：
NPV = -800 + 200/(1+0.1)^1 + 350/(1+0.1)^2 + 400/(1+0.1)^3 + 300/(1+0.1)^4
= -800 + 200/1.1 + 350/1.21 + 400/1.331 + 300/1.4641
= -800 + 181.82 + 289.26 + 300.53 + 204.90
= -800 + 976.51 = 176.51万元
NPV = 176.51万元 > 0，项目在经济上可行。

（2）第6个月末挣值分析：
BAC = 800万元（总预算）
PV（计划价值）= BAC × 50% = 800 × 0.5 = 400万元
EV（挣值）= BAC × 40% = 800 × 0.4 = 320万元
AC（实际成本）= 450万元
CV = EV - AC = 320 - 450 = -130万元（成本超支）
SV = EV - PV = 320 - 400 = -80万元（进度滞后）
CPI = EV/AC = 320/450 = 0.711 < 1（成本效率低）
SPI = EV/PV = 320/400 = 0.8 < 1（进度效率低）
结论：成本超支130万元，进度滞后80万元，成本效率和进度效率均低于基准，项目绩效差。

（3）第8个月末挣值分析：
BAC = 800万元
EV = BAC × 55% = 800 × 0.55 = 440万元
AC = 680万元
CV = 440 - 680 = -240万元
CPI = 440/680 = 0.647
PC（计划完成百分比）= 8/12 = 66.67%
PV = BAC × 66.67% = 533.33万元
SV = 440 - 533.33 = -93.33万元
SPI = 440/533.33 = 0.825

典型偏差（按当前效率）：EAC = BAC/CPI = 800/0.647 = 1236.48万元
TCPI（基于BAC）= (BAC - EV)/(BAC - AC) = (800 - 440)/(800 - 680) = 360/120 = 3.0
TCPI = 3.0 > 1，表示需要将后续工作的成本效率提升到当前水平的约4.6倍(3.0/0.647)才能实现BAC目标，几乎不可能，建议调整预算基准。

（4）问题与改进建议：

【变更管理问题】
- 问题①：用户需求变更后，PM口头答应即让开发修改代码，未走正式变更流程
- 改进：建立CCB（变更控制委员会），所有变更必须提交书面变更申请，经CCB评估影响后审批，审批通过方可执行
- 问题②：代码修改后未同步更新需求文档和设计文档
- 改进：变更实施后必须更新需求规格说明书、设计文档、测试用例等配置项，保持文档与代码一致

【采购管理问题】
- 问题①：采购负责人直接指定朋友公司，未公开招标，涉嫌利益输送
- 改进：严格遵守招投标法规，超过规定金额的采购必须公开招标；建立供应商准入制度和利益冲突申报机制
- 问题②：合同价格高于市场价30%，且未对供应商延期索赔
- 改进：采购前进行市场询价和价格评审；合同中明确违约金条款并严格执行

【质量管理问题】
- 问题①：多子系统接口联调失败，根源是各单位对接口规范理解不一致
- 改进：制定统一的接口规范文档并评审通过；开发前进行接口规范的技术交底和培训；设置联调里程碑，提前发现接口问题
- 问题②：采用瀑布模型却频繁变更需求，导致质量风险累积
- 改进：根据项目特点选择合适生命周期模型，可采用增量模型或敏捷方法应对需求不确定性

**踩分要点**：
- NPV计算用折现率逐年折现，注意区分折现因子(1+i)^t，t从1开始
- 挣值计算用百分比法求EV和PV，注意区分计划百分比和实际百分比
- EAC典型偏差用BAC/CPI，非典型用AC+(BAC-EV)，两种情境判断要准确
- TCPI>1表示后续工作必须提高效率，达到基准难度大，应建议调整基准
- 变更管理踩分点：CCB、书面申请、影响评估、审批、文档同步
- 采购管理踩分点：公开招标、利益冲突申报、合同评审、违约索赔
- 质量管理踩分点：接口规范统一、联调里程碑、技术交底、生命周期模型选型

## 记忆口诀

### 立项财务四指标
"净现值看正负，内部率比基准，回收期算快慢，回报率看高低。"
——NPV>0可行/IRR>基准可行/回收期越短越好/ROI越高越优

### CMMI成熟度五级
"初级管理定，量化再优化"
——初始级（无序）→已管理级（项目管）→已定义级（组织标准）→量化管理级（统计控制）→优化级（持续改进）

### 六西格玛DMAIC五步
"定测析改控，步步数据撑"
——定义找问题→测量抓数据→分析找根因→改进出方案→控制防反弹

### 区块链四特征
"去中去不改，可追智能约"
——去中心化、不可篡改、可追溯、智能合约

### 采购管理红线
"超限不招是违规，指定唯一是寻租，合同评审不能少，违约索赔必须追"
——公开招标、竞争择优、合同评审、履约管理
