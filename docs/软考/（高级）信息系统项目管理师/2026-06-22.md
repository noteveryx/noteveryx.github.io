## 今日核心概念

### 1. 项目进度管理六过程与进度网络分析

定义：项目进度管理包括规划、定义、排列、估算、制定和控制六个过程，核心目标是在既定约束内完成项目，关键路径法（CPM）和关键链法（CCPM）是两大核心工具。

关键要点：

| 过程 | 核心工具与技术 | 关键输出 |
|------|--------------|---------|
| 规划进度管理 | 专家判断、会议 | 进度管理计划 |
| 定义活动 | 分解、滚动式规划 | 活动清单、活动属性 |
| 排列活动顺序 | PDM（紧前关系绘图法）、确定依赖关系 | 项目进度网络图 |
| 估算活动持续时间 | 类比估算、参数估算、三点估算、群体决策 | 持续时间估算、估算依据 |
| 制定进度计划 | 关键路径法、资源优化、进度压缩、敏捷发布规划 | 进度基准、项目进度计划 |
| 控制进度 | 数据分析（偏差/趋势/迭代燃尽图） | 工作绩效信息、进度预测 |

- PDM四种逻辑关系（优先记忆）：
  - FS（完成-开始）：最常见，A完成→B开始
  - SS（开始-开始）：A开始→B开始（如：挖地基开始→绑钢筋开始）
  - FF（完成-完成）：A完成→B完成（如：编写完成→测试完成）
  - SF（开始-完成）：极少用，A开始→B完成
- 三种依赖关系：强制性（硬逻辑）→ 选择性（软逻辑）→ 外部依赖
- 提前量与滞后量：
  - 提前量（Lead）：后续活动可提前的时间（负值表示，如FS-2表示前序完成前2天即可开始）
  - 滞后量（Lag）：后续活动需等待的时间（正值表示，如FS+3表示前序完成后等3天）
- 进度压缩两法对比：

| 方法 | 做法 | 影响 | 适用条件 |
|------|------|------|---------|
| 赶工 | 增加资源，加班 | 增加成本 | 关键路径上可加资源 |
| 快速跟进 | 并行原串行活动 | 增加返工风险 | 活动间有并行可能 |

### 2. 项目成本管理四过程与挣值全公式体系

定义：项目成本管理包括规划、估算、制定预算和控制四个过程，挣值管理（EVM）是监控成本与进度绩效的核心技术。

关键要点：

| 过程 | 核心工具与技术 | 关键输出 |
|------|--------------|---------|
| 规划成本管理 | 专家判断、会议 | 成本管理计划 |
| 估算成本 | 类比估算、参数估算、自下而上、三点估算 | 成本估算、估算依据 |
| 制定预算 | 成本汇总、储备分析、历史信息审核 | 成本基准、项目资金需求 |
| 控制成本 | 挣值管理、偏差分析、趋势分析 | 工作绩效信息、成本预测 |

- 挣值管理全公式体系：

| 指标 | 公式 | 含义 |
|------|------|------|
| CV（成本偏差） | EV - AC | >0节约，<0超支 |
| SV（进度偏差） | EV - PV | >0超前，<0滞后 |
| CPI（成本绩效指数） | EV / AC | >1节约，<1超支 |
| SPI（进度绩效指数） | EV / PV | >1超前，<1滞后 |
| EAC（完工估算）- 非典型 | AC + (BAC - EV) | 偏差不持续，后续回到计划 |
| EAC（完工估算）- 典型 | BAC / CPI | 偏差持续，按当前效率走完 |
| EAC（完工估算）- 双偏 | AC + (BAC - EV) / (CPI × SPI) | 成本进度均偏差且持续 |
| ETC（完工尚需估算） | EAC - AC | 还需花多少钱 |
| TCPI（完工绩效指数） | (BAC - EV) / (BAC - AC) | 剩余工作必须达到的效率 |
| VAC（完工偏差） | BAC - EAC | >0节约，<0超支 |

- 四种EAC适用场景速判：
  - "偶尔犯错，以后改了"→ 非典型（AC + BAC - EV）
  - "一直这样，改不了了"→ 典型（BAC / CPI）
  - "成本进度都偏，双管齐下"→ 双偏（AC + 剩余/(CPI×SPI)）
  - "原有计划全废，重新来"→ 重估（自下而上重估）
- TCPI解读：TCPI>1意味着后续必须比之前更高效才能在预算内完工；若TCPI远大于1则不现实

### 3. 项目质量管理三过程与质量工具深入应用

定义：项目质量管理包括规划质量管理、管理质量（质量保证）和控制质量三个过程，核心目标是"满足需求"和"适用性"，预防胜于检查。

关键要点：

| 过程 | 核心工具与技术 | 关键输出 |
|------|--------------|---------|
| 规划质量管理 | 成本效益分析、质量成本COQ、标杆对照 | 质量管理计划、质量测量指标 |
| 管理质量 | 质量审计、过程分析、质量工具 | 质量报告、测试与评估文件 |
| 控制质量 | 检查、测试/产品评估、七种基本质量工具 | 质量控制测量结果、核实的可交付成果 |

- 老七工具（数据驱动，找问题）：

| 工具 | 用途 | 适用场景 |
|------|------|---------|
| 因果图（鱼骨图） | 找根本原因 | 出了问题，找原因 |
| 流程图 | 展示过程步骤和决策点 | 分析哪个环节出问题 |
| 核查表（检查表） | 收集和分类数据 | 现场数据收集 |
| 帕累托图 | 识别主要原因（80/20法则） | 缺陷排序，抓主要矛盾 |
| 直方图 | 展示数据分布 | 看数据集中趋势和离散度 |
| 控制图 | 监控过程稳定性 | 判断过程是否受控 |
| 散点图 | 分析两个变量的相关性 | 判断因素间是否相关 |

- 新七工具（逻辑驱动，做规划）：
  - 亲和图：归类分组（需求分类）
  - 过程决策程序图PDPC：预测问题和对策
  - 关联图：分析复杂因果关系
  - 树形图（系统图）：层次分解
  - 矩阵图：分析因素间关系
  - 矩阵数据分析图：量化矩阵分析
  - 箭线图：活动排序
- 控制图判稳/判异规则：
  - 在控制线内：过程受控（稳定）
  - 超出控制上限/下限：过程失控
  - 连续7点在同一侧：过程失控（七点规则）
  - 连续7点递增或递减：趋势失控
  - 规格线vs控制线：规格线是客户要求，控制线是过程能力，两者不能混淆

### 4. 信息系统安全体系深度

定义：信息系统安全体系包括物理安全、网络安全、主机安全、应用安全和数据安全五个层面，采用纵深防御策略构建多层安全屏障。

关键要点：

- 防火墙三类型：

| 类型 | 工作层 | 特点 | 适用场景 |
|------|--------|------|---------|
| 包过滤防火墙 | 网络层 | 速度快、不检查内容 | 简单访问控制 |
| 状态检测防火墙 | 网络层+传输层 | 跟踪连接状态 | 大多数企业场景 |
| 应用层防火墙（WAF） | 应用层 | 深度包检测 | Web应用防护 |

- IDS vs IPS：

| 特征 | IDS（入侵检测） | IPS（入侵防御） |
|------|----------------|----------------|
| 部署方式 | 旁路监听 | 串接部署 |
| 响应方式 | 告警，不阻断 | 实时阻断 |
| 影响 | 不影响业务流量 | 可能误杀正常流量 |
| 定位 | 事后取证 | 实时防御 |

- VPN四种隧道协议：
  - IPsec：网络层，站点到站点，安全性高
  - SSL/TLS：传输层，远程接入，浏览器即可
  - L2TP：数据链路层，常与IPsec组合（L2TP/IPsec）
  - PPTP：早期协议，安全性低，已淘汰
- 零信任架构核心原则：
  - "永不信任，始终验证"——不再区分内外网
  - 三大支柱：身份验证、设备健康检查、最小权限访问
  - 取代传统"边界防御"思路，适合云原生和远程办公
- 安全审计四要素：审计策略→审计日志→审计分析→审计报告
- 应急响应六阶段：准备→检测→遏制→根除→恢复→总结（PDCA循环）

### 5. 项目管理师职业道德与法律法规

定义：信息系统项目管理师应遵守《项目管理师职业道德行为准则》，同时掌握著作权法、专利法、商标法、计算机软件保护条例等法律法规的核心条款。

关键要点：

- 职业道德五大核心价值：责任、尊重、公正、诚实、守法
- 著作权法核心要点：

| 维度 | 规则 |
|------|------|
| 保护对象 | 文学、艺术、科学作品（含软件） |
| 取得方式 | 自动取得（创作完成即享有，无需登记） |
| 保护期限 | 作者终身+死后50年；单位作品50年；软件50年 |
| 职务作品 | 一般归作者，单位优先使用2年；主要利用单位资源→单位享有 |
| 委托作品 | 约定优先；无约定归受托人 |
| 合作作品 | 共同享有，不可分割则协商行使 |

- 计算机软件保护条例特殊规则：
  - 软件著作权自开发完成之日起产生
  - 软件复制品的出版者、制作者不能证明其出版、制作有合法授权→承担法律责任
  - 合法持有软件复制品的单位可不经许可进行"非营利性使用"（装入计算机/备份/必要的修改）
- 专利法三类型：

| 类型 | 保护对象 | 保护期限 | 审查方式 |
|------|---------|---------|---------|
| 发明 | 技术方案 | 20年 | 实质审查 |
| 实用新型 | 形状/构造/其结合 | 10年 | 形式审查 |
| 外观设计 | 外观 | 15年 | 形式审查 |

- 商标法：注册有效期10年，可续展（每次10年）；先申请原则
- 不正当竞争法：商业秘密保护（秘密性+保密措施+商业价值三要素）

## 案例分析题

**题目**：

某省启动"数字政府一体化服务平台"项目，预算3000万元，工期24个月。项目内容包括：统一身份认证平台、政务服务中台、数据交换共享平台、智慧审批系统、电子证照系统。项目采用工料合同，由某大型IT公司承建。

项目执行到第12个月末，出现以下情况：
- 计划价值PV=1500万元，实际成本AC=1800万元，挣值EV=1200万元
- 项目进度计划仅用甘特图编制，未进行关键路径分析，导致多个并行任务出现资源冲突
- 项目团队采用类比估算进行成本估算，但本项目与历史项目差异较大，估算严重偏低
- 甲方在月度评审会提出新增"区块链证照存证"功能，PM认为属于范围蔓延但不知如何处理
- 质量审计发现：系统接口缺陷占全部缺陷的42%，数据库相关占25%，界面问题占15%，其他占18%
- 项目部署的Web应用防火墙仅配置了基础规则，SQL注入攻击测试未通过
- 系统日志以明文存储在服务器上，未进行安全审计，无审计策略
- 核心算法代码由一名高级工程师独立编写，无代码审查，该工程师提出调岗申请

**问题1**（8分）：计算CV、SV、CPI、SPI，预测典型偏差下的EAC、ETC、VAC，并判断TCPI是否可行。

**问题2**（8分）：从进度管理角度，分析该项目进度计划编制和执行中存在的问题，并提出改进措施。

**问题3**（8分）：从质量管理角度，分析缺陷分布特征，给出质量改进方案（需用质量工具）。

**问题4**（8分）：从信息安全角度，分析该项目的安全问题和改进方案。

**问题5**（8分）：甲方提出新增区块链功能，PM应如何处理？若该工程师调岗，涉及哪些法律和知识管理问题？

**参考答案**：

**问题1**：
- CV = EV - AC = 1200 - 1800 = -600万元（成本严重超支）
- SV = EV - PV = 1200 - 1500 = -300万元（进度滞后）
- CPI = EV/AC = 1200/1800 = 0.667（每花1元仅完成0.667元价值）
- SPI = EV/PV = 1200/1500 = 0.8（进度效率80%）
- 典型偏差EAC = BAC/CPI = 3000/0.667 = 4499.25万元（预计超支约1500万）
- ETC = EAC - AC = 4499.25 - 1800 = 2699.25万元
- VAC = BAC - EAC = 3000 - 4499.25 = -1499.25万元（超支约1500万）
- TCPI = (BAC - EV)/(BAC - AC) = (3000 - 1200)/(3000 - 1800) = 1800/1200 = 1.5

TCPI=1.5意味着后续工作效率需提升50%才能在预算内完工，可行性较低。建议申请追加预算或缩减范围。

**问题2**：
进度管理问题：
① 仅用甘特图编制计划——缺少关键路径分析，无法识别关键活动和非关键活动的浮动时间，资源冲突无法预判
② 类比估算不当——本项目与历史项目差异大，类比估算不适用，导致成本和工期估算偏低
③ 资源冲突未处理——并行任务出现资源冲突，说明未进行资源平衡或资源平滑
④ 未使用进度网络图——无法清晰展示活动间的逻辑关系和依赖
⑤ 缺乏进度基准——没有经过批准的进度基准，无法有效监控进度偏差

改进措施：
① 立即进行关键路径分析：绘制PDM网络图，计算ES/EF/LS/LF，找出关键路径和浮动时间
② 重新估算：采用三点估算（PERT）或自下而上估算替代类比估算，提高估算准确性
③ 资源优化：对资源冲突活动进行资源平衡（调整时序），在浮动时间内进行资源平滑
④ 建立进度基准：将修订后的进度计划提交审批，形成进度基准
⑤ 定期监控：使用进度偏差分析和趋势分析，及时发现偏差并纠正

**问题3**：
缺陷分布分析（帕累托图）：
- 接口缺陷：42%（累计42%）
- 数据库相关：25%（累计67%）
- 界面问题：15%（累计82%）
- 其他：18%（累计100%）

根据80/20法则，接口缺陷+数据库缺陷合计67%（接近80%），属于主要缺陷来源。

质量改进方案：
① 因果图分析根本原因——针对接口缺陷，从人（接口开发人员经验不足）、机（接口测试工具缺失）、料（接口规范不统一）、法（无接口设计标准）、环（联调环境不稳定）五方面分析
② 制定接口规范——统一接口设计标准（RESTful/API版本管理/错误码规范），制定接口文档模板
③ 加强接口测试——引入契约测试和自动化接口测试，每次构建自动运行接口测试用例
④ 数据库优化——引入数据库设计评审，建立SQL规范和索引优化指南
⑤ 持续监控——建立缺陷跟踪系统，每周生成帕累托图，追踪缺陷趋势

**问题4**：
安全问题分析：
① WAF配置不足——仅基础规则，SQL注入未防御，违反等保"安全区域边界"要求
② 日志明文存储——敏感审计日志未加密，存在泄露风险，违反"安全计算环境"要求
③ 无安全审计——缺乏审计策略和审计分析，无法满足等保三级安全审计要求
④ 单点依赖——核心代码仅一人掌握，存在安全风险和知识风险

改进方案：
① WAF强化：配置SQL注入、XSS、CSRF防护规则；部署IPS串接防御；定期漏洞扫描
② 日志安全：审计日志加密存储（AES-256/SM4）；日志传输使用TLS加密；设置日志访问权限（RBAC）
③ 安全审计：制定审计策略（审计哪些操作、保留多久、谁可查看）；部署安全信息和事件管理系统（SIEM）；定期审计分析
④ 代码安全：立即进行代码审查，识别安全漏洞；建立代码审查制度（所有提交必须经至少1人审查）
⑤ 参照等保三级"一中心三防护"架构全面整改

**问题5**：
新增功能处理流程：
① PM应要求甲方正式提交变更请求（书面形式）
② PM组织影响分析：评估对范围、进度、成本、质量的影响
③ 提交CCB审批——工料合同下新增功能涉及预算增加，必须经CCB（含甲方代表）审批
④ 审批通过后：更新范围基准、进度基准、成本基准
⑤ 审批不通过：记录并归档，与甲方沟通替代方案

工程师调岗涉及问题：
① 知识管理风险：核心代码仅一人掌握，属于"隐性知识"集中风险。应立即进行知识转移——代码审查、文档化、交叉培训
② 著作权归属：该工程师在职期间开发的代码属于职务作品，著作权归单位享有，但需确认劳动合同和保密协议
③ 商业秘密保护：核心算法属于商业秘密，需确认竞业限制协议和保密条款的有效性
④ 风险应对：立即安排至少1人接手该模块，进行代码走查和文档化，确保知识不随人走

## 记忆口诀

**进度管理六过程**：
"规定排估制控"
规（规划进度）→定（定义活动）→排（排列顺序）→估（估算持续时间）→制（制定进度计划）→控（控制进度）
PDM四关系"完开"组合，FS最常见，SF极少见。

**挣值全公式速记**：
"EV永远在前方"
CV=EV-AC，SV=EV-PV，CPI=EV/AC，SPI=EV/PV
EAC四法："非加典除双乘除重估重头来"
非典型=AC+(BAC-EV)；典型=BAC/CPI；双偏=AC+剩余/(CPI×SPI)；重估=自下而上
TCPI="(BAC-EV)/(BAC-AC)"，大于1要加速，远大于1不现实。

**质量老七工具**：
"因帕控流核直散"
因（因果图找原因）→帕（帕累托抓主因）→控（控制图判稳定）→流（流程图看过程）→核（核查表收数据）→直（直方图看分布）→散（散点图看相关）
新七工具"亲过关树矩矩箭"。

**信息安全纵深防御**：
"防火三墙包状用"
包（包过滤）→状（状态检测）→用（应用层WAF）
IDS旁路告警不阻，IPS串接实时阻断。
VPN四协议"ISLP"：IPsec安全高/SSL易接入/L2TP常组合/PPTP已淘汰。
零信任"永不信任始终验证"。

**法律法规速记**：
"著自专审商十年"
著（著作权自动取得）→自（创作完成即享有）→专（专利需审查）→审（发明实质审查/实用新型形式审查）→商（商标10年可续展）
职务作品归单位优先，委托无约归受托，商业秘密"秘密+保密+价值"三要素。
