## 今日核心概念
### 1. 云计算服务模型与部署模式
定义：云计算是一种通过网络按需提供可配置计算资源（网络、服务器、存储、应用和服务）的模式，核心特征是弹性伸缩、按需付费、资源池化。

关键要点：
- 三层服务模型：IaaS（基础设施即服务，提供虚拟机/存储/网络，如阿里云ECS）、PaaS（平台即服务，提供开发运行环境，如阿里云ACE）、SaaS（软件即服务，直接使用应用，如腾讯文档）
- 四种部署模式：公有云（面向公众，成本低）、私有云（单一组织专用，安全性高）、社区云（特定群体共享）、混合云（公+私组合，数据安全与弹性兼顾）
- 政务项目选型原则：核心数据放私有云或政务云，公共服务放公有云，形成混合部署架构
- 考试高频对比：用户对控制权的掌握程度 IaaS>PaaS>SaaS，用户对运维负担 SaaS<PaaS<IaaS

### 2. UML图分类（九类图三大类）
定义：统一建模语言（UML）是面向对象分析与设计的标准建模语言，使用图形化符号描述软件系统的静态结构和动态行为。共9种图，分为三大类。

关键要点：
- 结构图（静态）：类图（类及关系）、对象图（实例快照）、构件图（物理组件及接口）、部署图（硬件节点部署）、包图（包组织）——记忆关键"类对构部包"
- 行为图（动态-行为）：用例图（功能需求，用户与系统交互）、状态图（对象状态转移）、活动图（业务流程/算法流程）——记忆关键"用状活"
- 交互图（动态-交互）：顺序图（时间序消息传递）、协作图/通信图（对象组织关系）、时序图（时间约束）——后两者为考试低频
- 考试陷阱：用例图不属于"交互图"而属于"行为图"；部署图关注"硬件+软件的物理部署关系"

### 3. 标准分级体系
定义：根据《标准化法》，我国标准分为国家标准、行业标准、地方标准、团体标准和企业标准五级，其中国家标准和行业标准又分强制性标准与推荐性标准（GB为强制，GB/T为推荐）。

关键要点：
- 五级效力：国标（GB/GB/T）> 行标（如GA公安行标）> 地标（DB）> 团标（T）> 企标（Q）
- 强制性标准"必须执行"，推荐性标准"鼓励采用"
- 标准化指导性技术文件（GB/Z）为过渡性文件
- ISO/IEC国际标准对应关系：ISO对应工业产品，IEC对应电工电子，我国等同采用标记为GB/T XXXX idt ISO XXXX
- 信息系统项目常用标准：GB/T 16260（软件质量模型）、GB/T 8567（计算机软件文档编制规范）、GB/T 9385（计算机软件需求说明编制指南）

### 4. 知识产权——著作权归属三原则
定义：知识产权是权利人对其智力劳动成果所享有的专有权利，项目管理工作涉及最多的为计算机软件著作权。

关键要点：
- 职务作品归属：利用单位物质技术条件创作、与本职工作直接相关→著作权归单位，作者有署名权；非上述情况→著作权归个人，单位有优先使用权（2年）
- 委托开发归属：有合同约定按约定，无约定→著作权归受托方（开发方），委托方有使用权
- 合作开发归属：可分割使用的各部分各自享有，不可分割的→协商一致行使，不能协商且无正当理由→任一方不得阻止他方行使（转让权除外），收益应合理分配
- 软件著作权保护期：自然人终生+死后50年；法人或其他组织为首次发表后50年
- 考试陷阱：委托开发"无约定归受托方"≠归委托方（很多人记反）

### 5. 资源分解结构（RBS）与资源日历
定义：资源分解结构是按资源类别和类型对项目所需资源进行层级分解的工具，资源日历则记录每个资源在项目期间可用的具体时间段。

关键要点：
- RBS层级：资源分类（人力/设备/材料）→资源类型（开发人员/测试人员）→具体资源（张三/李四）
- 资源日历输出：可用时间段、休假安排、其他项目占用情况
- 与RACI矩阵的关系：RACI确定"谁做什么角色"，RBS确定"有哪些资源可用"，资源日历确定"资源何时可用"
- 考试陷阱："资源日历由HR提供"是错的，应由项目经理与职能经理协商确定
- 资源平滑（Resource Leveling/smoothing）对比：资源平衡可能改变关键路径，资源平滑在自由浮动时间内调整不改变关键路径——后者是考试高频辨析点

## 案例分析题
**题目**：

某省政府启动"智慧政务云平台"建设项目，预算800万元，工期12个月。项目采用混合云架构：核心政务数据部署在政务私有云，公共服务模块部署在公有云。项目经理小王带领团队完成了需求调研后，直接进入开发阶段。

第6个月末，项目管理委员会进行阶段评审，发现以下问题：
- 项目已完成6个迭代，累计PV=420万，AC=460万，EV=380万
- 公有云资源费用超出预期40%，原因是前期未进行云资源容量规划
- 两个核心模块间接口联调反复失败，原因是不同开发团队对接口规范理解不一致，仅凭口头沟通确认
- 甲方新任命的信息中心主任对项目提出大量变更需求，小王口头答应后直接通知开发人员修改
- 系统安全等级保护定级工作尚未启动，而合同约定交付前需通过等保三级测评

**问题**：
（1）请计算第6月末的CV、SV、CPI、SPI，并分析项目当前绩效状况。
（2）请分析项目在云资源管理、接口规范管理、变更管理、安全管理方面存在的问题，并给出改进建议。
（3）若后续绩效按当前典型偏差发展，计算EAC，并说明要达到BAC目标还需的TCPI。

**参考答案**：

（1）绩效计算：
- CV = EV - AC = 380 - 460 = -80万（成本超支80万）
- SV = EV - PV = 380 - 420 = -40万（进度滞后40万）
- CPI = EV/AC = 380/460 ≈ 0.826（每投入1元产出0.826元）
- SPI = EV/PV = 380/420 ≈ 0.905（进度完成率为90.5%）
- 结论：项目成本超支、进度滞后，CPI和SPI均低于1.0，绩效不理想，需立即采取纠正措施。

（2）问题分析与改进：

云资源管理问题：未进行云资源容量规划→导致公有云费用超预期40%。改进：①编制云资源容量规划文档，估算各模块所需计算/存储/网络资源；②建立云资源费用监控仪表板，设置费用告警阈值；③采用预留实例+按需实例混合计费策略降低成本。

接口规范管理问题：仅凭口头沟通确认接口规范→不同团队理解不一致。改进：①制定接口规范文档（API文档/SDK），明确输入输出参数/数据格式/错误码；②建立接口契约测试机制，每次迭代进行自动化接口兼容性验证；③将接口规范纳入配置管理，变更须经CCB审批。

变更管理问题：口头答应变更→缺少正式变更控制。改进：①建立整体变更控制流程（提交变更申请→评估影响→CCB审批→实施→验证→记录）；②所有变更必须书面记录，重大变更需甲方信息中心主任签字确认；③变更涉及的成本/进度影响需同步更新项目管理计划和基准。

安全管理问题：等保定级未启动→交付前可能无法通过测评。改进：①立即启动信息系统安全等级保护定级工作，委托具备资质的测评机构；②将安全需求融入开发过程（安全左移），同步开展代码安全审计和渗透测试；③编制等保三级整改方案，预留2-3个月等保测评与整改周期。

（3）EAC与TCPI：
- 典型偏差公式：EAC = BAC/CPI = 800/0.826 ≈ 968.5万
- TCPI（基于BAC）= (BAC - EV)/(BAC - AC) = (800-380)/(800-460) = 420/340 ≈ 1.235
- 含义：剩余工作每投入1元需产出1.235元才能不超BAC，按当前CPI仅0.826看几乎不可能，建议申请预算追加或调整范围

**踩分要点**：
- 计算类：公式正确得一半分，计算准确得满分，单位（万）必须带
- 分析类：每个问题点须先指出"问题"再给出"改进"，二者缺一扣一半分
- 云资源规划、接口文档、CCB变更流程、等保定级——这四个专业手段必须明确出现
- TCPI大于1.0时需结合CPI判断可行性，单纯算出数字不得满分

## 记忆口诀
- **云服务三层**："I盖房P租房S住酒店"——IaaS自己搭基础设施（盖房），PaaS给你平台（租房自带家具），SaaS直接用成品（住酒店拎包入住）
- **UML九图三类**："雇（构件）员（类）不（部署）对（对象）用（用例）活（活动）状（状态）顺（顺序）"——结构图前四字、行为图后三字、交互图最后字
- **著作权委托归属**："委托无约归受托，委托出钱有使权"——委托开发没签合同→著作权归开发方（受托方），委托方只有使用权
- **标准等级**："国家行业地团企，GB强制推加T"——等级从高到低，GB为强制性、GB/T为推荐性
- **EAC典型偏差**："典除用除法，预算除CPI"——典型偏差按当前效率继续→EAC=BAC/CPI
