## 今日核心概念
### 1. 知识管理（显性知识与隐性知识）
定义：知识管理是通过系统化的过程，确保项目团队成员能够获取、创造、分享和整合项目所需知识的活动。分为显性知识（可编码、可文档化，如规范/手册/代码）和隐性知识（个人经验、直觉、诀窍，难以文字化传递）。

关键要点：
- 知识管理三过程：知识规划（确定需要什么知识）→ 知识生成与沉淀（经验教训总结、事后回顾）→ 知识共享与整合（导师制、轮岗、社区实践）
- 显性知识传递靠文档化和培训，隐性知识传递靠面对面交流、师徒制、实践社区
- 经验教训登记册是知识管理核心输出，需在项目全生命周期持续更新
- 常见工具：知识地图、经验教训库、实践社区、事后回顾会
- 知识管理失败信号：关键人员离职后无人能接手、相同错误反复出现、专家经验无法复用

### 2. EAC完工估算的四种计算方法
定义：EAC（Estimate At Completion）是根据项目当前绩效预测的总成本，是挣值管理的核心预测指标。根据偏差性质不同，有四种计算方式。

关键要点：
- 类型一（非典型偏差、纠偏后恢复正常）：EAC = AC + (BAC - EV)，假设已发生的偏差不再重复
- 类型二（典型偏差、偏差持续）：EAC = BAC / CPI，假设CPI代表未来趋势
- 类型三（成本和进度均偏差）：EAC = AC + (BAC - EV) / (CPI × SPI)，同时考虑成本和进度影响
- 类型四（重估法）：EAC = AC + Bottom-up ETC，抛弃历史数据，彻底重新估算剩余工作
- 选择原则：看偏差是一次性特殊原因还是系统性原因；看管理层是否采取了纠偏措施
- TCPI（完工尚需绩效指数） = (BAC - EV) / (BAC - AC)，表示剩余工作必须达到的CPI水平

### 3. 信息系统生命周期与开发方法对比
定义：信息系统生命周期分为立项、开发、运维、消亡四个阶段。开发方法决定了项目如何从需求走向交付，不同方法适用于不同场景。

关键要点：
- 瀑布模型：需求→设计→编码→测试→运维，线性顺序，适用于需求明确、变更少的项目（如政务系统合规类项目）
- V模型：强调测试与开发阶段一一对应（单元测试对应详细设计、集成测试对应概要设计），适用于对质量要求极高的系统
- 螺旋模型：每个螺旋包含目标设定→风险分析→开发验证→评审规划，强调风险驱动，适用于大型高风险项目
- 原型法：快速构建可运行原型→用户反馈→迭代完善，适用于需求不明确的创新项目
- 敏捷开发：小步快跑、持续交付、拥抱变化，适用于需求多变、用户参与度高的项目
- DevOps：开发运维一体化，CI/CD持续集成与持续交付，适用于互联网产品快速迭代
- 选择方法的核心判断矩阵：需求确定性 × 技术成熟度 × 团队分布 × 交付紧迫度

### 4. IT服务管理（ITSM）与ITIL框架
定义：ITSM是以服务为中心的IT管理方法论，ITIL是ITSM的最佳实践框架。ITIL v4将IT服务生命周期分为服务价值链和四大维度模型。

关键要点：
- ITIL核心五阶段（v3）：服务战略 → 服务设计 → 服务转换 → 服务运营 → 持续服务改进
- 事件管理（突发事件处理，目标：尽快恢复服务）vs 问题管理（找根因，目标：防止重复发生）vs 变更管理（控制变更风险，目标：减少变更导致的事故）
- 服务台（Service Desk）是ITSM的单一联系点，分为呼叫中心型、非技能型和技能型
- SLA（服务级别协议）、OLA（运营级别协议）、UC（支持合同）三级服务级别体系
- ITIL v4引入服务价值链（SVC）：需求→参与→设计与转换→获取/构建→交付与支持→价值，六项活动编织成链
- 持续改进七步法：确定改进方向→收集数据→处理数据→分析数据→呈现信息→实施改进→评估改进

### 5. 招投标管理核心法规要点
定义：信息系统项目采购必须遵循《中华人民共和国招标投标法》和《中华人民共和国政府采购法》的强制性规定，违反可能导致中标无效、行政处罚甚至刑事责任。

关键要点：
- 必须招标的项目类型（三要素同时满足）：①大型基础设施/公用事业等关系公共利益安全的项目 ②全部或部分使用国有资金投资或国家融资的项目 ③使用国际组织或外国政府贷款的项目，且单项合同估算价达到规定标准
- 招标方式：公开招标（发布招标公告，不特定法人投标）和邀请招标（向三家以上特定法人发邀请），公开招标是默认首选
- 评标委员会：成员5人以上单数，技术经济专家不少于2/3，评标委员会名单中标前保密
- 废标情形：有效投标不足3家、所有投标均不符合要求、出现违法违规行为、投标报价均超过预算
- 中标通知书发出后30日内签订合同，不得再行订立背离合同实质性内容的其他协议
- 政府采购方式优先级：公开招标 > 邀请招标 > 竞争性谈判 > 单一来源采购 > 询价

## 案例分析题
**题目**：

某省政府启动"一网通办"政务服务平台升级项目，预算900万元，工期12个月，采用公开招标方式。甲公司以870万元中标。

项目启动后第4个月（计划完成30%），项目经理在月度评审中发现：实际支出AC=320万元，挣值EV=240万元。

此外，项目经理发现以下问题：
（1）原技术负责人因薪资问题在第3个月离职，接替者上手慢，部分核心模块代码注释缺失，理解需反复与原负责人微信沟通，效率下降约40%；
（2）第2个月因用户反馈增加了智能表单功能，开发团队直接做了修改，未走变更控制流程，也未更新文档，如今已找不到该功能的原始需求说明；
（3）甲方在第3个月口头要求增加与公安人口库的数据核验接口，开发团队口头答应但未实施，甲方向领导汇报时称"已安排开发"，引发信任危机。

问题：
1. 计算当前CV、SV、CPI、SPI，并分析项目绩效状态。
2. 若当前偏差为典型偏差，计算EAC。若剩余工作从第5个月开始必须按期完工（不超BAC），计算TCPI。
3. 分析知识管理方面存在哪些问题，并提出改进措施。
4. 结合变更管理和沟通管理，分析本项目中存在的问题及改进建议。

**参考答案**：

1. 挣值计算：
CV = EV - AC = 240 - 320 = -80万元（成本超支）
SV = EV - PV = 240 - (900×30%) = 240 - 270 = -30万元（进度滞后）
CPI = EV / AC = 240 / 320 = 0.75（每花1元只产出0.75元价值）
SPI = EV / PV = 240 / 270 ≈ 0.889（进度效率约89%）

绩效状态：成本超支、进度滞后，成本偏差较大需重点关注。

2. EAC和TCPI：
典型偏差 → EAC = BAC / CPI = 900 / 0.75 = 1200万元（超预算300万元）
TCPI = (BAC - EV) / (BAC - AC) = (900 - 240) / (900 - 320) = 660 / 580 ≈ 1.138

TCPI≈1.14 > 1，意味着剩余工作必须以114%的CPI效率执行才能不超BAC，难度大。

3. 知识管理问题及改进：
问题：①技术负责人离职后代码注释缺失，隐性知识（模块设计思路、编码习惯）未转化为显性知识（文档）；②核心知识集中在个别人身上，形成单点依赖风险；③信息传递依赖微信碎片化沟通，未进入经验教训登记册。

改进：①建立代码审查与注释规范，强制关键模块必须完整注释；②推行结对编程或导师制，每周轮换配对，确保知识分散；③每个迭代结束时举行事后回顾会，记录经验教训并入库；④建立知识地图，标注各模块的专家和备份人员。

4. 变更和沟通管理问题：
问题：①智能表单功能未走CCB流程直接修改，违反整体变更控制原则；②文档未同步更新导致需求追溯断裂；③公安接口口头沟通未经书面确认，项目经理未将口头要求转化为正式变更请求；④甲方汇报与开发现状信息不对等，暴露沟通管理缺失。

改进：①严格执行CCB变更控制流程，任何范围/需求变更必须提交书面变更请求→评估影响→CCB审批→执行并更新文档；②建立沟通管理计划，明确甲方汇报前必须先与乙方核对项目状态；③所有甲方口头要求24小时内书面化并发双方确认；④定期发布项目绩效报告（含EVM指标和变更日志），保持双方信息对称。

**踩分要点**：
- CV/SV/CPI/SPI计算各1分，绩效状态描述1分（共5分）
- EAC计算2分，TCPI计算1分，必须有判断TCPI>1的解读1分（共4分）
- 知识管理问题至少3点（3分），改进措施至少3点（3分）（共6分）
- 变更管理问题2点（2分），改进2点（2分）；沟通管理问题1点（2分），改进1点（2分）；信息对等分析1分（共9分）
- 合计约24分，属于案例分析题典型分值

## 记忆口诀

**知识管理"隐性难传，显性靠写"**：隐性知识靠师徒/面对面/实践社区传递；显性知识靠文档化/培训/知识库沉淀；经验教训登记册是知识管理核心输出。

**EAC四种算法速记**：非典偏差用加（BAC-EV加上AC）、典型偏差用除（BAC除以CPI）、双偏移用乘除（乘上SPI再除以CPI×SPI或直接用AC+(BAC-EV)/(CPI×SPI)）、彻底重估重来算——"非加典除双乘除，重估等于重头来"。

**开发方法"螺险原需瀑明确，敏捷变化DevOps快"**：螺旋模型重风险控制、原型法重需求探索、瀑布模型重需求明确、敏捷重拥抱变化、DevOps重快速交付运维一体。

**ITIL核心口诀**：服务价值链SVC六连环——"参设获交价值链"（参与→设计→获取→交付→价值）；事件问题变更三兄弟——"事件快恢复、问题找根因、变更控风险"；三级协议——"SLA对外OLA对内UC管事"。

**招投标核心数字**：评委会"5人以上单数，专家2/3以上"、签合同"中标后30日内"、废标"不足3家或有猫腻"、必须招标"国有+公益+外资，三者其一且够标准"。
