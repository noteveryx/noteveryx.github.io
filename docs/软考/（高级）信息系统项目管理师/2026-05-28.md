## 今日核心概念

### 1. PERT三点估算与进度压缩
定义：计划评审技术（PERT）通过乐观时间(O)、最可能时间(M)、悲观时间(P)三个值计算期望工期，公式为 Te = (O + 4M + P) / 6，标准差 σ = (P - O) / 6。

关键要点：
- 正态分布下，1σ概率68.26%、2σ概率95.46%、3σ概率99.73%
- 进度压缩两大手段：赶工（增加资源换时间，会增成本）、快速跟进（并行执行串行活动，会增风险）
- 赶工只对关键路径上的活动有效，且不是所有活动都能赶工
- 快速跟进可能导致返工，仅适用于可并行化的活动

### 2. 质量七工具
定义：质量管理中常用七种基本工具，用于规划质量、质量保证和质量控制过程。

关键要点：
- 因果图（石川图/鱼骨图）：追溯问题根源，从人机料法环测六个维度分析
- 帕累托图：80/20法则，少数原因造成多数问题，按发生频率排序的直方图
- 控制图：判断过程是否稳定，上控制界限UCL/下控制界限LCL，七点同侧=失控
- 流程图：展示过程的输入、步骤和输出关系，发现潜在质量问题
- 核查表（检查表）：用于收集数据的结构化表格
- 直方图：显示数据分布的条形图
- 散点图：显示两个变量之间关系的坐标图

### 3. 三大经典激励理论
定义：马斯洛需求层次、赫茨伯格双因素、麦格雷戈X/Y理论是人力资源管理中的三大经典激励理论。

关键要点：
- 马斯洛五层（从低到高）：生理→安全→社交→尊重→自我实现，低层满足后高层才起作用
- 赫茨伯格双因素：保健因素（工资/环境/政策）不满足会不满、满足不会满意；激励因素（成就/认可/成长）满足才会满意
- 麦格雷戈X理论（人性恶）：人天生懒惰、逃避责任、需要强制管控
- 麦格雷戈Y理论（人性善）：人愿意工作、追求责任、能够自我管理
- 实操建议：项目经理应多用Y理论，善用激励因素，保障保健因素

### 4. 项目管理计划及子计划
定义：项目管理计划是整合所有子计划和基准的综合文档，是指导项目执行、监控和收尾的纲领性文件。

关键要点：
- 十大知识领域对应十项子计划 + 三个基准（范围基准、进度基准、成本基准）
- 三大基准必须由CCB（变更控制委员会）审批变更
- 子计划可按需裁剪：小项目可合并简化，复杂项目需详细
- 项目管理计划 ≠ 项目文件：计划是"怎么做"，文件是"什么结果"
- 基准（Baseline）= 原始计划 + 已批准的变更，是实际绩效的参照标尺

### 5. 干系人参与评估矩阵
定义：用于对比干系人当前参与水平与期望参与水平，识别管理差距的工具。

关键要点：
- 参与度五级分类：不知晓→抵制→中立→支持→领导
- 干系人可能在不同阶段处于不同参与水平
- 矩阵输出驱动沟通策略：对抵制者加强沟通消除顾虑，对支持者保持关系争取升级为领导
- 需结合权力/利益方格制定差异化管理方案
- 干系人管理是持续过程，参与度需定期重评

## 案例分析题

**题目**：

某市政府委托A公司开发智慧政务一体化平台，合同金额1800万，工期18个月。项目经理张某制定了项目管理计划，其中进度计划如下：需求分析3个月→系统设计2个月→编码开发6个月→集成测试3个月→试运行2个月→正式上线，共16个月。

项目启动后第三个月，甲方突然要求在原有需求基础上新增"跨部门数据共享模块"，该模块尚未列入合同范围。经初步评估，新增模块需要额外3个月开发时间。项目经理提出以下方案：
- 将集成测试和编码开发部分并行执行（快速跟进）
- 增加3名高级开发人员（赶工）

同时，技术总监对代码质量表示担忧，要求项目经理制定质量改进措施。项目经理统计了最近一个月的缺陷数据：接口调用错误36个、数据库连接异常22个、页面渲染问题14个、权限校验漏洞7个、日志记录不规范5个、其他16个。

**问题1**：根据PERT三点估算，编码开发阶段的O=4个月、M=6个月、P=9个月，求期望工期和在2σ范围内完成的时间区间。（5分）

**问题2**：评价项目经理提出的应对新增需求的两种方案（快速跟进和赶工）的利弊，指出哪些策略有效、哪些策略该用哪条路径。（8分）

**问题3**：用帕累托图原理分析缺陷数据，指出应优先解决哪些问题，并结合质量七工具提出对应的分析手段。（7分）

**问题4**：从干系人参与评估矩阵的角度，分析甲方的参与水平变化以及项目经理应如何调整沟通策略。（5分）

**参考答案**：

**问题1**：
- Te = (4 + 4×6 + 9) / 6 = 37/6 ≈ 6.17个月
- σ = (9 - 4) / 6 = 5/6 ≈ 0.83个月
- 2σ范围 = [6.17 - 2×0.83, 6.17 + 2×0.83] = [4.51, 7.83]个月
- 2σ内完成的概率为95.46%

**问题2**：
- 赶工：有效。编码开发在关键路径上，增加高级开发人员可缩短工期。但成本会显著增加，且需要考虑新增人员的学习曲线和沟通成本
- 快速跟进：部分有效。编码开发和集成测试并行可压缩总工期，但风险在于：未完成编码就启动测试，可能导致大量返工；新增模块的编码和原系统集成测试之间的并行衔接不清晰
- 策略排序：优先赶工编码开发（关键路径），再考虑将试运行与集成测试部分并行
- 关键提醒：快速跟进只应在可并行化的活动上使用，且必须评估返工风险；赶工应优先用在关键路径活动

**问题3**：
- 按发生频率排序：接口调用错误36、数据库连接异常22、页面渲染问题14、权限校验漏洞7、日志记录不规范5、其他16
- 累计占比前两项（36+22=58个）占总缺陷（100个）的58%，前三项（72个）占72%
- 帕累托图优先解决：接口调用错误和数据库连接异常，这两项解决后缺陷可减少58%
- 分析手段：
  - 接口问题→因果图（鱼骨图）：从人员/方法/接口规范/测试环境四个维度分析根因
  - 数据库问题→流程图：梳理数据库连接的调用链路，找出断点
  - 整体监控→控制图：建立质量控制线，持续跟踪缺陷率是否受控
  - 数据收集→核查表：统一缺陷记录格式，确保数据完整

**问题4**：
- 甲方参与水平变化：发起时处于"领导"水平（主动推动项目）；新增需求时可能由"支持"滑向"抵制"（若需求被拒）或保持"支持"但仍需管控
- 沟通策略：
  - 对甲方高层：维持高水平沟通，定期汇报项目状态和变更影响分析，争取"领导"参与水平
  - 对甲方业务部门：详细解释新增需求的变更流程、工期影响和成本影响，引导其从"中立"转为"支持"
  - 定期重评干系人参与度，更新参与评估矩阵
  - 建立正式变更控制流程（CCB评审），让甲方参与决策过程

**踩分要点**：
- PERT公式写对得2分，标准差公式写对得1分，2σ概率值写对得1分，区间计算对得1分
- 赶工分析要提到"关键路径"才得分（1分），快速跟进必须提到"返工风险"（1分）
- 帕累托图要给出具体数字和占比（2分），质量工具与问题的对应关系要合理（2分）
- 干系人参与水平要标注五级分类（2分），沟通策略要具体到不同干系人群体（2分）

## 记忆口诀

**PERT估算**：
乐观到悲观、可能占四成；分母除以六、期望自然成。标准差P减O除以六、正态概率心中有。

**质量七工具**：
因帕控流 核直散 —— 因果图找根源、帕累托抓重点、控制图看稳定、流程图梳逻辑、核查表收数据、直方图看分布、散点图找关联。

**激励理论**：
马斯洛五层楼梯爬、生理安全社交尊重自我实现达；赫茨伯格双因素分开讲、保健不满足要骂娘、激励满足才有劲头闯；麦格雷戈X和Y、X是懒人Y是强、项目经理用Y方。

**进度压缩**：
赶工加人加钱、只对关键路径好使；快速跟进并行、返工风险不小看。

**干系人矩阵**：
不知晓、抵制、中立、支持、领导；C到D差距大、沟通升级跨鸿沟。
