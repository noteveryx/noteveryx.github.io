# （高级）信息系统项目管理师 · 2026-06-01

> 第8次学习 · 今日主题：信息系统规划方法 / 监控与控制 / 审计合规

---

## 今日核心概念

### 1. 信息系统规划方法（ISP四大方法）

**定义**：指导组织从战略层面系统性规划信息系统建设的方法论体系，确保IT投资与业务战略对齐。

**关键要点**：
- **关键成功因素法（CSF）**：识别组织成功必须满足的关键因素，优先保障对应信息需求；适用于高层管理决策支持系统
- **战略目标集转化法（SST）**：将组织战略目标转化为信息系统目标，自上而下分解；适用于战略导向型规划
- **企业系统规划法（BSP）**：IBM提出，自上而下规划、自下而上实施；通过业务流程→数据类→系统架构三步建立U/C矩阵；实施复杂、成本高
- **战略数据规划法（SDP）**：以数据为核心，建立企业主题数据库；强调数据共享和数据质量

**速判表**：
| 方法 | 核心驱动 | 适用场景 |
|------|----------|----------|
| CSF | 关键成功因素 | 高层决策/MIS |
| SST | 战略目标集 | 战略规划对齐 |
| BSP | 业务流程+U/C矩阵 | 全面系统规划 |
| SDP | 数据资源 | 数据中台/数据治理 |

### 2. 项目监控过程组（Monitor & Control）

**定义**：贯穿整个项目生命周期，持续测量项目绩效、识别偏差、发起变更的过程集合。

**关键要点**：
- **监控范围**：涵盖范围、进度、成本、质量、风险、采购、干系人参与等所有知识领域
- **核心输出**：工作绩效报告（Work Performance Reports）——整合分析后向干系人展示绩效状况
- **控制变更**：监控阶段发现偏差 → 提交变更请求 → 整体变更控制审批 → 实施批准的变更
- **挣值分析（EVM）在监控中的应用**：CV/SV/CPI/SPI是主要量化监控工具
- **趋势分析与预测**：用EAC（完工估算）预测最终成本；EAC = BAC/CPI（最常用）；EAC = AC + (BAC-EV)/CPI（绩效持续恶化时）

**关键公式速查**：
```
CV = EV - AC（负值=超支）
SV = EV - PV（负值=滞后）
CPI = EV/AC（<1=超支）
SPI = EV/PV（<1=滞后）
EAC = BAC/CPI（通用）
ETC = EAC - AC（剩余成本）
VAC = BAC - EAC（完工偏差）
TCPI = (BAC-EV)/(BAC-AC)（须达到绩效指数）
```

### 3. 配置管理与变更控制（Config & Change）

**定义**：通过系统化管理项目的可交付成果和文件，确保变更受控、基准可追溯的管理活动。

**关键要点**：
- **配置项**：项目中需要被正式管理的任何元素（需求文档、设计文档、代码、测试用例等）
- **基准（Baseline）**：经过正式审查和批准的规范或工作产品，只能通过正式变更程序修改；三大基准为**范围基准、进度基准、成本基准**，合称**绩效测量基准（PMB）**
- **配置管理活动**：① 配置识别 → ② 配置状态记录 → ③ 配置审计
- **变更控制委员会（CCB）**：审查、评估、批准/拒绝变更请求的正式机构；项目经理无权单独批准重大变更
- **变更请求类型**：纠正措施（已偏差）/ 预防措施（预防偏差）/ 缺陷修复（质量问题）/ 更新（文件变化）

---

### 4. 项目审计（Project Audit）

**定义**：对项目管理过程、成果物及合规性进行独立评估的质量保证活动。

**关键要点**：
- **质量审计 vs 过程审计**：质量审计关注是否遵守质量政策；过程审计检查管理过程合规性
- **审计时机**：可在项目生命周期任何阶段进行；采购审计在采购收尾时必须执行
- **采购审计目的**：识别合同准备、采购和合同管理过程中的成功与失败，形成教训供组织参考
- **内部审计 vs 外部审计**：内部由PMO或质量部门执行；外部由独立第三方执行（政府项目常见）
- **审计报告内容**：偏差发现 + 整改建议 + 最佳实践记录

---

### 5. 信息系统项目绩效报告

**定义**：整合项目监控数据后，向干系人沟通项目当前状态、趋势及预测的正式文件。

**关键要点**：
- **工作绩效数据（Raw Data）** → **工作绩效信息（分析后）** → **工作绩效报告（面向干系人）**
- **报告内容**：进度状态、成本状态、范围状态、质量度量结果、风险状态、问题日志更新
- **报告格式**：仪表盘报告（高层）/ 详细状态报告（团队）/ 里程碑报告（里程碑节点）
- **沟通管理计划**：规定向谁报告、报告什么、何时报告、以何种格式报告
- **关键原则**：绩效报告不等同于绩效测量；绩效测量是数据收集，绩效报告是信息传递

## 案例分析题

**题目**：

某省级政务数据共享平台项目，总预算BAC=500万元，计划工期12个月。项目执行到第8个月末，项目经理收集到以下数据：

- 计划完成工作量价值（PV）= 350万元
- 实际完成工作量价值（EV）= 300万元
- 实际发生成本（AC）= 360万元

在监控例会上，甲方代表提出：①系统核心模块需求发生了较大变化，要求增加3个月工期；②要求项目组提交一份完整的绩效报告；③对项目的审计安排表示关切，询问审计如何开展。

请回答以下问题：
1. 计算该项目当前的CV、SV、CPI、SPI及EAC，并判断项目目前的状态。
2. 针对甲方的需求变更请求，项目经理应如何处理？变更控制委员会（CCB）在其中扮演什么角色？
3. 项目绩效报告应包含哪些主要内容？（至少5项）
4. 简述项目审计的目的及采购审计应何时开展。

---

**参考答案**：

**第1问：挣值计算与状态判断**

```
CV = EV - AC = 300 - 360 = -60万元（成本超支60万）
SV = EV - PV = 300 - 350 = -50万元（进度滞后50万价值）
CPI = EV/AC = 300/360 ≈ 0.83（每花1元只完成0.83元价值）
SPI = EV/PV = 300/350 ≈ 0.86（进度仅完成计划的86%）
EAC = BAC/CPI = 500/0.83 ≈ 602万元（预计完工成本将超出约102万）
```

**状态结论**：项目当前处于**成本超支、进度滞后**的双重不利状态。CPI=0.83和SPI=0.86均小于1，按当前绩效趋势，预计完工成本将超出预算约102万元，需立即采取纠偏措施。

---

**第2问：需求变更处理流程**

变更处理步骤：
1. **记录变更请求**：项目经理将甲方变更请求正式记录到变更请求日志，明确变更内容（增加3个月工期、核心模块需求调整）
2. **影响评估**：项目团队评估变更对范围、进度、成本、质量、风险的影响，形成影响评估报告
3. **提交CCB审查**：将变更请求及影响评估报告提交变更控制委员会（CCB）
4. **CCB决策**：CCB审批通过、拒绝或要求修改变更方案；项目经理无权单独批准重大变更
5. **更新基准**：若批准，更新项目管理计划、进度基准、成本基准；若拒绝，通知甲方并记录原因

**CCB角色**：CCB是变更决策的唯一授权机构，负责独立评估变更的必要性和可行性，防止随意变更破坏项目基准，保护各方合法权益。

---

**第3问：项目绩效报告主要内容**

① 进度状态（当前进度与计划对比，里程碑完成情况）
② 成本状态（预算执行情况，EV/AC/PV挣值分析结果）
③ 范围状态（范围变更情况，已批准变更清单）
④ 质量度量结果（缺陷数量、测试通过率、质量偏差）
⑤ 风险状态（新识别风险、已有风险变化、应对措施执行情况）
⑥ 问题与障碍（当前未解决问题及责任人）
⑦ 下一阶段计划（下期工作目标及资源安排）

---

**第4问：审计目的与采购审计时机**

**项目审计目的**：对项目管理过程的合规性和有效性进行独立评估，识别偏差与问题，形成改进建议，积累组织过程资产（经验教训）。

**采购审计时机**：采购审计应在**采购收尾阶段**开展，目的是识别采购准备、采购执行和合同管理过程中的成功经验与失败教训，供组织在后续类似项目中参考，提升采购管理能力。

---

**踩分要点**：

1. **EAC公式选择**：EAC=BAC/CPI是默认公式（假设未来绩效与当前一致）；若题目特别说明后续工作按原计划执行，则用EAC=AC+(BAC-EV)，注意区分
2. **CCB权限边界**：必须强调"项目经理不能单独批准重大变更"，否则扣分
3. **绩效报告至少列5项**：仅写3-4项通常只给部分分，务必写齐
4. **采购审计时机**：必须写"收尾阶段"，写成"执行阶段"或"监控阶段"均不得分
5. **挣值计算步骤**：必须写出公式再代入数值，直接写结果不给步骤分

## 记忆口诀

**ISP四法速记**："关键战略企业数据"
- **关**键成功因素法（CSF）→ 关注关键要素
- **战**略目标集转化法（SST）→ 战略自上而下
- **企**业系统规划法（BSP）→ U/C矩阵全面
- **数**据战略规划法（SDP）→ 数据资产核心

---

**挣值四公式口诀**："差值看减法，效率看除法"
- **CV=EV-AC**（Cost Variance，负=超支）
- **SV=EV-PV**（Schedule Variance，负=滞后）
- **CPI=EV/AC**（小于1超支，大于1节省）
- **SPI=EV/PV**（小于1滞后，大于1超前）

记忆顺序：**EV永远在前**（EV-AC / EV-PV / EV÷AC / EV÷PV）

---

**配置管理三步法**："识别→记录→审计"
1. **识别**：确定哪些是配置项
2. **记录**：记录配置项状态和版本
3. **审计**：验证配置项与基准一致

---

**变更类型四字诀**："纠预缺更"
- **纠**：纠正措施（已发生偏差）
- **预**：预防措施（防止偏差发生）
- **缺**：缺陷修复（质量不合格）
- **更**：更新（文件/计划更新）

---

**CCB核心原则**："重大变更必过CCB，项目经理无独批权"
