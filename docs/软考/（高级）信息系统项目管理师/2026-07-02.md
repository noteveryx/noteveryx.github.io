## 今日核心概念

### 1. 人力资源管理六过程与ITTO体系
定义：项目人力资源管理通过规划、组建、建设、管理团队四个执行过程加上规划过程和控制过程，确保项目拥有合适的人员并保持团队高效运作。
关键要点：
- 规划人力资源管理：输出《人力资源管理计划》，包含角色与职责矩阵（RACI）、组织图（OBS）、人员配备管理计划、培训需求、认可与奖励计划、合规要求；
- 组建项目团队：输入包括人力资源管理计划，工具技术有预分配、谈判、招募、虚拟团队，输出为项目人员分配表与资源日历；
- 建设项目团队：核心工具为通用技能培训、团队建设活动、集中办公（war room）、认可与奖励，输出团队绩效评价；
- 管理项目团队：工具技术为观察与交谈、项目绩效评估、冲突管理、人际关系技能，输出变更请求与组织过程资产更新；
- 人力资源计划的"人员配备管理计划"必须包含：人员获取方式、培训计划、认可奖励制度、合规要求、遣散安排；
- 资源日历vs资源直方图：日历记录"某人何时可用"，直方图显示"某时段需要多少人"——两者必须对齐。

### 2. 团队发展阶段与冲突管理场景匹配
定义：团队按塔克曼模型必然经历形成→震荡→规范→成熟→解散五阶段，不同阶段需要不同的领导风格与冲突处理策略。
关键要点：
- 形成期（Forming）：成员互相试探，PM应采用"指令型"领导，冲突少但默契更少——重点在明确目标与角色；
- 震荡期（Storming）：意见分歧、权力争夺，冲突爆发——PM采用"影响型"领导，必须直面冲突，首选"合作/解决问题"方法；
- 规范期（Norming）：规则建立、互信增强——PM转向"参与型"领导，冲突管理可用"妥协/调解"；
- 成熟期（Performing）：自组织、高效运作——PM采用"授权型"领导，冲突自行化解，PM只做方向把控；
- 解散期（Adjourning）：项目收尾，成员心理波动——PM需关注遣散安排、知识转移、绩效总结；
- 冲突管理六方法适用排序：合作/解决问题（最优）> 协作/妥协 > 缓和/包容 > 回避/撤退 > 强制/命令（最后手段）——考试常考"哪种方法何时用"。

### 3. 领导力风格选择矩阵与权力运用
定义：项目经理根据团队成熟度、项目阶段、组织文化选择不同领导风格，并运用六种权力推动项目目标实现。
关键要点：
- 领导力三风格：专制型（快速决策/危机场景）、民主型（团队成熟/创新场景）、放任型（专家团队/探索场景）——切忌"一刀切"；
- 权力六类型及运用策略：
  - 合法权力（职位赋予）：项目章程授权，启动阶段最有效；
  - 奖励权力（正向激励）：建设团队阶段，认可与奖励计划支撑；
  - 惩罚权力（负向约束）：仅用于严重违规，频繁使用导致士气崩盘；
  - 专家权力（技能权威）：技术决策、质量评审时最有力；
  - 参照权力（人格魅力）：长期项目、跨部门协调时的软实力；
  - 感召权力（愿景驱动）：变革项目、创新型项目的核心驱动力；
- 考试高频考点：PM在弱矩阵组织中权力最小，必须更多依靠专家权力和参照权力；在强矩阵中有合法权力和奖励权力加持；
- 权力组合原则：合法权力打底+专家权力立信+参照权力固本，惩罚权力慎用、奖励权力常态化。

### 4. 项目变更控制与CCB深度运作
定义：整体变更控制确保所有变更经正式审批、记录、执行和追踪，变更控制委员会（CCB）负责评审和批准/否决变更请求。
关键要点：
- 变更控制七步流程：提出变更请求→影响分析（范围/进度/成本/质量/风险全维度）→CCB评审与决策→更新项目管理计划与基准→通知受影响干系人→执行变更→验证变更结果并归档；
- CCB组成与职责：由项目发起人、客户代表、PM、技术专家、质量代表组成；CCB只做"审批"不做"执行"，PM负责执行和追踪；
- 变更请求四类型：
  - 纠正措施：把偏离拉回基准（实际偏差→纠正）；
  - 预防措施：防患于未然（潜在偏差→预防）；
  - 缺陷补救：修复已发现的产品缺陷（质量不达标→修复）；
  - 更新：因法规/标准变化对基准的主动调整；
- 变更控制的"黄金原则"：任何变更必须书面提出、任何变更必须影响分析、任何变更必须CCB审批、任何变更必须记录归档；
- 紧急变更处理：PM可在紧急情况下先行执行，事后补办CCB审批手续——但"紧急"必须有明确定义和时限。

### 5. 配置管理与变更管理联动机制
定义：配置管理识别、记录、控制项目产品的功能与物理特征，变更管理控制对这些特征的任何改动，两者联动确保产品一致性。
关键要点：
- 配置管理三核心活动：
  - 配置识别：确定配置项、分配标识号、建立配置基线；
  - 配置状态记录：追踪每项配置的当前状态、变更历史、版本号；
  - 配置审计：物理审计（产品是否与设计一致）+ 功能审计（产品是否满足需求规格）；
- 配置项分层：一级配置项（项目章程、项目管理计划）→二级配置项（需求规格、设计文档）→三级配置项（代码、测试脚本）→底层配置项（数据字典、接口定义）；
- 变更与配置联动流程：变更请求→影响分析（含配置影响）→CCB审批→更新配置项→更新配置基线→配置状态记录→配置审计验证；
- 版本控制三规则：主干稳定、分支开发、合并回归；基线一旦建立只通过变更流程才能修改；
- 配置管理的关键输出：配置管理计划、配置项清单、配置基线、配置状态报告、配置审计报告——缺任何一项都会导致"版本混乱""变更无追溯"。

## 案例分析题

**题目**：某市"智慧医保"信息平台项目，预算1500万元，计划工期14个月，团队25人。项目采用弱矩阵组织结构，PM仅1年项目管理经验。项目进行到第6个月时发现以下问题：
① PM在组建团队时仅按部门分配任务，未编制人力资源管理计划，未建立RACI矩阵，导致3个模块出现职责重叠、2个模块无人负责；
② 团队处于震荡期，开发组与测试组在接口规范上持续争吵，PM回避冲突，说"大家都是成年人，自己协商解决"；
③ PM对核心架构师的离职请求直接用惩罚权力威胁"你走就是违约"，导致架构师更加抵触，其他团队成员也产生信任危机；
④ 甲方提出新增"异地就医结算"功能需求，PM口头答应并直接安排开发，未走变更流程，未更新范围基准；
⑤ 配置管理混乱——3个版本的接口文档同时流通，开发人员不知道该参照哪个版本，导致联调失败6次。

请从人力资源管理、变更控制、配置管理三个维度分析问题并给出改进措施。同时计算：项目第6月末PV=600万，EV=480万，AC=560万，求CV、SV、CPI、SPI，并判断项目状态。

**参考答案**：

1. 挣值计算
   - CV = EV - AC = 480 - 560 = -80万（成本超支80万）
   - SV = EV - PV = 480 - 600 = -120万（进度落后120万）
   - CPI = EV/AC = 480/560 = 0.857（每花1元只值0.857元）
   - SPI = EV/PV = 480/600 = 0.80（进度效率仅80%）
   - 项目状态：成本超支+进度落后，双重红灯，必须立即干预。

2. 人力资源管理维度
   - 问题①：未编制人力资源管理计划——缺失角色职责定义（RACI）、人员配备管理计划、培训需求与认可奖励制度，导致职责重叠与空缺；
     改进：立即编制人力资源管理计划，建立RACI矩阵明确每项任务的负责(A)/执行(R)/咨询(C)/知会(I)角色；对无人负责模块预分配或谈判获取资源；
   - 问题②：PM回避冲突——震荡期冲突必须正面处理，"自己协商"是回避/撤退策略（最差选择）；
     改进：PM应采用"合作/解决问题"策略，组织开发组与测试组面对面会议，以接口规范为焦点，用数据（联调失败次数）驱动共识；震荡期PM需采用"影响型"领导风格，主动介入而非回避；
   - 问题③：滥用惩罚权力——对核心人员使用惩罚权力导致信任崩盘，这是弱矩阵组织中的致命错误（PM本就合法权力不足）；
     改进：PM应依靠专家权力和参照权力挽留核心架构师——了解离职原因（可能薪资/发展空间/工作压力），提供留人方案（薪资谈判/技术决策权/培训机会）；如确实无法挽留，立即启动AB角机制与知识转移。

3. 变更控制维度
   - 问题④：口头答应需求、未走变更流程——违反变更控制"黄金原则"（任何变更必须书面提出、必须影响分析、必须CCB审批、必须记录归档）；
     改进：
     ① 补办变更请求，对"异地就医结算"做全面影响分析（范围/进度/成本/质量/风险），估算增量成本与工期影响；
     ② 提交CCB评审，由发起人、甲方代表、技术专家共同决策是否接受；
     ③ 无论审批结果，均更新项目管理计划、范围基准、WBS，通知所有受影响干系人；
     ④ 建立变更管理流程文档，确保后续所有变更走正式流程。

4. 配置管理维度
   - 问题⑤：多版本接口文档同时流通、联调失败——缺少配置识别（未确定配置项与版本号）、配置状态记录（未追踪当前有效版本）、配置审计（未验证产品与文档一致性）；
     改进：
     ① 立即执行配置识别——确定接口文档为二级配置项，分配唯一标识号，冻结有效版本为基线版本；
     ② 回收所有非基线版本文档，统一发布基线版本并通知全部开发与测试人员；
     ③ 建立配置状态记录机制——每项配置变更必须通过变更流程审批后更新，版本号递增有规则（V1.0→V1.1→V2.0）；
     ④ 定期进行配置审计（至少每月一次），验证代码与接口文档一致、产品与需求规格一致；
     ⑤ 编制配置管理计划，纳入项目管理计划体系。

5. 综合改进方案
   - PM需提升权力组合策略：弱矩阵中合法权力不足，必须加强专家权力（参与技术评审）和参照权力（建立信任关系）；
   - 编制完整的项目管理计划体系，包括人力资源管理计划、变更管理计划、配置管理计划；
   - 建立CCB，明确变更审批流程与紧急变更处理规则；
   - 启动团队建设活动，从震荡期尽快过渡到规范期。

**踩分要点**：
- 必须算出CV=-80/SV=-120/CPI=0.857/SPI=0.80四项挣值指标并判断双重红灯状态；
- RACI矩阵是人力资源管理必答关键词，必须指出"职责重叠与空缺"的根源是缺少RACI；
- 震荡期冲突管理必须指出PM选用了最差的"回避"策略，应改用"合作/解决问题"；
- 惩罚权力滥用+弱矩阵组织是核心考点，必须指出弱矩阵中PM应依靠专家权力和参照权力；
- 变更控制"黄金原则"四条必须全写，缺一条扣分；
- 配置管理三核心活动（识别/记录/审计）必须全部提到，且需说明与变更管理的联动流程。

## 记忆口诀

- 人力资源六过程："规组建管两控制"（规划→组建→建设→管理+变更控制→组织过程资产更新）
- RACI四角色："负责执行咨询知会"——A负责拍板R动手干活C提供意见I事后知晓
- 塔克曼五段领导匹配："指影参授遣"——形成指令/震荡影响/规范参与/成熟授权/解散遣散
- 冲突六法优先级："合协缓回强"——合作最优/妥协次选/缓和包容/回避最差/强制最后
- 权力六型运用："合奖惩专参感"——合法打底/奖励常态/惩罚慎用/专家立信/参照固本/感召驱动
- 变更黄金四原则："书提分审全归档"——书面提出/影响分析/CCB审批/记录归档
- 变更四类型："纠预缺更"——纠正拉回/预防防患/缺陷修复/更新调整
- 配置三步法："识记审"——识别配置项/记录状态/审计一致性
- 配置与变更联动："变更先分析配置影响→CCB审批→更新配置项→重建基线→状态记录→审计验证"
