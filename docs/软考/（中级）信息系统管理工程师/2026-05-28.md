## 今日核心概念
### 1. 信息系统需求工程
定义：需求工程是系统开发中获取、分析、定义和验证用户需求的结构化过程，是连接用户与开发团队的关键桥梁，决定系统最终质量的核心环节。

关键要点：
- 需求获取方法：访谈、问卷调查、头脑风暴、原型法、观察法、文档分析法
- 需求分类：功能需求（系统做什么）、非功能需求（性能/安全/可靠性等约束）、设计约束
- 需求规格说明书（SRS）是需求工程的最终产物，必须清晰、完整、一致、可验证
- 需求变更管理流程：变更申请 → 影响分析 → 评审决策 → 实施变更 → 验证确认
- 需求验证方法：评审、原型确认、测试用例验证、可追溯性矩阵

### 2. 系统测试方法
定义：系统测试是验证信息系统是否满足需求规格说明书的系统性工程活动，目的是发现缺陷、验证功能和评估系统质量。

关键要点：
- 黑盒测试：关注输入/输出，不关心内部结构，包括等价类划分、边界值分析、因果图法、场景测试
- 白盒测试：关注内部逻辑结构，包括语句覆盖、判定覆盖、条件覆盖、路径覆盖
- 测试层级：单元测试（开发人员）→ 集成测试（模块间接口）→ 系统测试（整体功能）→ 验收测试（用户确认）
- 回归测试：每次修改后重新执行相关测试用例，确保原有功能不受影响
- α测试：开发方内部模拟实际使用环境进行测试；β测试：用户在实际环境中试用

### 3. IT资产管理
定义：IT资产管理是对组织内所有信息技术资产（硬件、软件、许可、服务合同）的全生命周期管理，包括采购、部署、维护、报废各阶段。

关键要点：
- 资产分类：硬件资产（服务器/网络设备/终端）、软件资产（操作系统/应用软件/中间件）、许可证资产、数据资产
- 资产全生命周期：采购申请 → 验收入库 → 分配使用 → 维护保养 → 盘点核查 → 报废处置
- 软件许可证管理：跟踪许可证类型（永久/订阅/并发）、采购数量、实际使用量，避免侵权和浪费
- 配置管理数据库（CMDB）是IT资产管理的核心工具，记录资产属性及关联关系
- 盘点制度：定期盘点（年度/季度）与动态更新结合，确保账实一致

### 4. IT财务管理
定义：IT财务管理是对IT服务成本和效益的系统化管理活动，目标是实现IT投资的价值最大化，确保IT预算的合理分配和有效控制。

关键要点：
- TCO（总拥有成本）：包含硬件采购、软件许可、运维人力、培训、能耗、场地等全周期成本
- ROI（投资回报率）=（IT收益 - IT投入）/ IT投入 × 100%，用于评估IT项目的经济效益
- IT预算编制方法：增量预算法、零基预算法、滚动预算法
- IT计费管理：建立服务目录和定价模型，实现IT服务成本的内部核算与分摊
- 成本效益分析：不仅是财务指标，还包括效率提升、服务质量改善、风险降低等间接效益

### 5. 知识管理
定义：知识管理是对组织内知识和经验进行系统化获取、组织、共享和应用的管理活动，防止知识因人员流动而流失，将个人知识转化为组织能力。

关键要点：
- 知识的分类：显性知识（可文档化、可编码）与隐性知识（个人经验、技能、直觉）
- 知识管理流程：知识获取 → 知识组织 → 知识存储 → 知识共享 → 知识应用
- 常见工具：知识库系统、FAQ库、经验教训库、Wiki协作平台、专家地图
- 知识共享机制：导师制、技术分享会、轮岗培训、实践社区（CoP）
- 知识管理的难点在于隐性知识的挖掘与转化，核心是营造"乐于分享"的组织文化

## 案例分析题
**题目**：某市政务服务中心上线运行三年，现有信息系统包括行政审批系统、电子监察系统、统一门户等共8个业务子系统。近期审计发现以下问题：

（1）需求阶段仅有会议纪要，未形成正式的需求规格说明书，导致多期开发中出现大量需求变更无法追溯；
（2）系统上线前的测试记录缺失，仅保留了一份"用户试用报告"，测试主管已离职；
（3）IT资产台账中服务器数量与实际部署相差6台，软件许可证采购记录与安装数量不符；
（4）核心业务系统的运维知识高度集中在2名老员工手中，其中1人上月离职，故障排查效率下降60%；
（5）IT经费没有独立核算，历年IT投入总额、各系统运维成本均无法统计。

问题1：针对问题（1）和（2），从系统开发规范角度，指出该中心存在哪些管理缺陷？（8分）
问题2：针对问题（3），提出IT资产管理的改进措施，至少列出4条。（8分）
问题3：针对问题（4）和（5），结合知识管理和IT财务管理，提出改进建议。（9分）

**参考答案**：

问题1：
（1）需求管理缺陷：未建立需求工程制度，缺少正式的需求规格说明书（SRS），导致需求变更无法追溯、审批流程缺失，需求管理处于无序状态。（4分）
（2）测试管理缺陷：缺少系统的测试规划和测试用例库，未按层级（单元测试→集成测试→系统测试→验收测试）执行测试，测试文档不完整，知识资产随人员流失。（4分）

问题2：
（1）建立统一的IT资产台账/CMDB系统，将硬件、软件、许可证纳入统一管理；
（2）实施定期盘点制度（如每季度全面盘点），核对账实是否一致；
（3）实行资产全生命周期管理，从采购、入库、分配、使用到报废全程留痕；
（4）建立软件许可证管理台账，跟踪许可证类型和实际使用量，防范合规风险；
（5）将IT资产管理责任落实到人，明确资产使用人和管理人的职责。（答出4条即满分）

问题3：
知识管理改进建议：（5分）
（1）建立知识库系统，将运维流程、故障处理方案、系统架构文档等显性知识集中管理；
（2）推行导师制和岗位AB角制度，新老搭配，实施轮岗培训，加速隐性知识传承；
（3）将知识贡献纳入绩效考核，鼓励员工主动输出文档和经验总结。

IT财务管理改进建议：（4分）
（1）建立IT独立核算科目体系，实现IT预算、支出、成本的统一管理；
（2）按系统/项目维度进行成本归集，计算各系统TCO；
（3）实施IT服务计费管理，将IT成本合理分摊到各业务部门。

**踩分要点**：
- 问题1必须同时指出"需求管理"和"测试管理"两个维度，只答一个最多得4分
- "SRS（需求规格说明书）"和"测试层级"是核心关键词，漏写扣分
- 问题2的改进措施必须具体可操作，泛泛而谈"加强管理"不得分
- 问题3的知识管理部分必须提到"显性/隐性知识"或"知识传承"，IT财务部分必须提到"TCO"或"独立核算"

## 记忆口诀
需求工程五步走——获取、分析、定义、验证、变更有
测试四层往上推——单元、集成、系统、验收不会亏
资产管好记四字——采、配、维、废（采购→分配→维护→报废）
TCO算总账不要漏——硬件软件人员都得有
显性知识文档存，隐性要靠师徒传
