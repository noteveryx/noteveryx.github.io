# 申论每日一练 — 2026年05月28日

## 给定材料

材料一：近年来，各地大力推进数字政府建设，政务服务平台覆盖面不断扩大。截至2025年底，全国一体化政务服务平台实名注册用户超过10亿，省级政府一体化政务服务能力显著提升。然而，在数字政府建设快速推进的同时，部分地区出现了"指尖上的形式主义"——政务APP过多过滥、功能重复建设、数据多头填报等问题，基层干部承担了大量的"指尖"负担。

材料二：东部某县一位社区工作者反映，自己手机上安装了47个政务APP，每天需要登录签到、填报数据的就有23个，部分APP功能几乎不重叠却来自不同条线部门。"上午人社系统要数据，下午民政系统也要差不多的数据，我们只能在各个系统里反复填报。"据统计，该社区工作人员平均每天花在各类政务APP上的时间超过2.5小时。

材料三：西部某市2023年启动了"一体统管"改革，将原有的32个分散的政务APP整合为3个统一平台，打通了12个部门的数据壁垒，实现"一次填报、多方共享"。改革后，基层填报工作量下降约65%，群众办事平均等待时间从45分钟缩短至12分钟。市长表示："数字化不是目的，让群众和基层干部真正有获得感才是目的。"

材料四：专家指出，数字政府建设要从"有没有"转向"好不好"，关键在于破解"数据烟囱"和"信息孤岛"。有学者指出，一些地方将数字化简单等同于"上系统""装APP"，缺乏顶层设计和统筹规划，导致"数据瀑布"变成"数字负担"。要实现数字治理的提质增效，必须在体制机制、标准规范、数据共享等方面进行系统性改革。

## 作答要求

1. 根据给定材料，概括主要问题并提出对策建议。（不超过400字）

2. 结合材料，自选角度，自拟题目，写一篇议论文。（800-1000字）

## 参考答案

### 第一题：概括问题与对策建议

**一、主要问题**

（一）政务APP过多过滥，功能重复建设。不同条线部门各自开发系统，导致基层干部需要安装大量APP，功能重叠严重。

（二）数据壁垒突出，多头填报现象普遍。部门间数据不共享、不互通，"数据烟囱"和"信息孤岛"问题严重，造成基层人员反复填报相同数据。

（三）顶层设计缺失，存在"指尖上的形式主义"。部分地方将数字化简单等同于"上系统"，缺乏统筹规划，数字治理未能实现提质增效的初衷。

（四）基层负担沉重，严重影响服务效能。大量时间消耗在系统操作上，挤占了为民服务的时间和精力。

**二、对策建议**

（一）强化顶层设计。由省级层面统筹规划数字政府建设，建立统一的技术标准和数据规范，避免各自为政、重复建设。

（二）推进系统整合。对现有政务APP进行清理整合，将功能相近的平台合并为统一入口，实现"一网通办"。

（三）打通数据壁垒。建立跨部门数据共享机制，推行"一次填报、多方共享"，从源头上减少基层填报负担。

（四）完善考核机制。将数字政府建设成效的考核重心从"建了多少系统"转向"基层减了多少负、群众办事方不方便"，杜绝形式主义。

### 第二题：议论文

**以系统思维破解数字治理的"指尖之困"**

数字政府建设是推进国家治理体系和治理能力现代化的重要抓手。近年来，从东部沿海到西部边陲，"一网通办""最多跑一次"等改革措施，让人民群众切实感受到了数字化带来的便利。然而，一些地方政务APP过多过滥、基层干部"指尖"负担沉重的现象，也暴露了数字治理中"重建设轻整合、重数量轻质量"的深层次问题。破解这一困局，必须以系统思维推进数字治理改革，真正实现"减负增效"的目标。

系统思维要求统筹规划，从顶层破解"各自为政"的碎片化问题。数字政府建设是一项系统工程，涉及众多条线部门，任何一个部门单兵突进都可能导致新的"数据烟囱"。当前，一些地方之所以出现47个政务APP并存的怪象，根源在于缺乏统筹协调。各部门从自身需求出发开发系统，忽视横向协同与纵向贯通，结果便是基层成了"数据漏斗"——所有系统都向基层要数据，却彼此不相往来。因此，必须在省级乃至国家层面建立统一的数字政府建设规划，明确建设标准、数据规范和时间表，用"一盘棋"思维取代"各自为战"。

系统思维要求数据贯通，以信息共享打破"信息孤岛"。数据是数字政府建设的核心资源，其价值在于流动与共享。东部某县社区工作者"上午人社要数据、下午民政也要差不多数据"的尴尬，折射的正是数据不流动的困境。应当建立跨层级、跨地域、跨系统、跨部门、跨业务的数据共享交换平台，推行"一次采集、多方共享"机制。西部某市整合32个APP为3个统一平台后，填报量下降65%，这充分说明：系统整合不是简单的"做减法"，而是通过流程再造实现数据的高效流通，从根子上减少基层的重复劳动。

系统思维要求以人为本，用群众和基层干部的获得感检验改革成效。数字政府的最终目的是服务人民。如果数字化改革反而增加了基层的负担、降低了服务效率，那就背离了改革的初衷。因此，必须建立以实效为导向的考核评价体系，将"基层减了多少负""群众办事方不方便"作为衡量数字政府建设成效的核心指标。同时，要建立基层反馈机制，让一线工作人员参与系统设计和优化改进，避免"上面建系统、下面用不好"的尴尬。

习近平总书记强调："要运用大数据提升国家治理现代化水平。"数字治理不是简单的技术叠加，而是一场深刻的治理变革。必须以系统思维为引领，统筹规划、数据贯通、以人为本，才能让数字政府建设真正成为提升治理效能、减轻基层负担的有力抓手，让数字化的"指尖"红利惠及每一位基层干部和人民群众。（约980字）
